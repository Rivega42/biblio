## Назначение и модель (БД VUZ, дисциплины, специальности, контингент, связь с IBIS/RDR)

**Назначение.** Решение задачи книгообеспеченности (КО) — ведение данных о КО учебного процесса и получение коэффициентов книгообеспеченности (ККО) в просмотровых и выходных формах (txt:320).

**Три базы данных** участвуют в задаче (txt:321–324):
1. **БД электронного каталога (ЭК)** — её часть с учебниками; общая БД фонда (имя по умолчанию IBIS) либо отдельная БД только учебной литературы. Если имя не IBIS — задаётся в настройке (txt:322, 457, 888).
2. **БД студентов = БД читателей RDR** — для подсчёта обеспеченности разных контингентов. Возможна работа без БД студентов — численность вводится в записи VUZ (txt:323, 459).
3. **Спец. БД учебных дисциплин и контингентов — VUZ** — связующая между каталогом и студентами; с её формирования начинается работа (txt:324, 412).

**БД VUZ — типы записей** (поле 920 = код РЛ) (txt:412–422, 570–572):
- **Дисциплина — DISC**: наименование, идентификатор (уникальный, короткий, лат. символы), кафедра читающая, цикл, уровень компонента; поле 83 — все контингенты, где читается (txt:414, 571, 586–598). Наименование НЕ уникально, идентификатор уникален (txt:587–592).
- **Контингент — VUZ**: формируется автоматически из полей 83 дисциплины. Контингент = связка: форма обучения, вид обучения/уровень подготовки, направление, специальность/профиль, факультет, семестр (txt:415–421, 617–619). Содержит поле 69 — перечень читаемых дисциплин (txt:430, 483).
- **Факультет — FAK**: необязательная, автоматическая, только в модуле «Учебный план» (txt:422, 626).
- **Рабочая программа — WPD**: версия, даты согласования/введения, разработчики, привязка к дисциплине по идентификатору (txt:625, ~1135–1137).

**Связка (связь IBIS-VUZ-RDR).** Набор «факультет-направление-специальность/профиль-форма обучения-вид обучения-семестр», длина не более 120 символов (макс. длина термина словаря) (txt:426, 463, 617). Через связку: учебники каталога привязаны к дисциплинам/контингентам (поле 691); студенты RDR отнесены к контингентам (поле 90) (txt:423–425).

**Специальные элементы описания по БД** (txt:427–430):
- **Запись каталога** — страница «KO», основное поле 691; вручную заполняются 691, 61, 694, 699, остальное автоматически (txt:428, 892).
- **Запись студента (RDR)** — поле 90 (контингент), поле 69 (изучаемые дисциплины, авто по VUZ) (txt:429, 785). При смене семестра поле 69 перестраивается (~1444, 1581).
- **Запись дисциплины (VUZ)** — поле 83 (контингенты), порождает записи контингентов, каждая связана со студентами аналогичного контингента (txt:430). Связка контингента VUZ — поле 68 (txt:1484); кол-во студентов без RDR — поле 68 подполе Z (TabNumbTag=68, TabNumbDelim=Z) (~876–877).

При корректном вводе связи поддерживаются автоматически; есть контроль связей и режим «Связать» для переустановки (txt:431).

**Справочники (MNU)** — кодируют текстовые параметры; едины для всех трёх БД либо в DEPOSIT (txt:466–474, 490). Обязательные: FO.mnu, VO.mnu, FAK.mnu, KAFCH.mnu, CIKLD.mnu (txt:492–496). Желательные для «Учебного плана»: spec, napr, disc (txt:498–501). Выделенные фонды — Naznac.mnu + NaznCatDelim (A-факультет, C-спец., N-направл., O-форма, V-вид, F-семестр) (txt:502–514). Филиалы — Fili.mnu (FilialMnu), код филиала = место хранения mhr.mnu (txt:518, 1367–1370).

## Расчёт книгообеспеченности (формула/коэффициенты)

**Общая идея.** ККО вычисляется для текущего семестра (txt:1252). Варианты (txt:1253–1256):
- **ККО отдельного учебника** — по полю 693: общее число студентов разных контингентов, использующих учебник, и число экземпляров.
- **ККО для параметра** (дисциплина/специальность/факультет) — двумя способами: (1) отобрать учебники и студентов по параметрам; экз. / студенты / кол-во учебников; (2) среднее арифметическое значений ККО каждого учебника (из поля 693).

**Формула ККО для одной книги** (интерфейсная таблица, txt:1316):
ККО = экземпляры (с учётом КМИ) / Студенты всего (все студенты, использующие книгу в текущем семестре).

Где (txt:1264, 1310–1320):
- **Экз.** — поле 910, статус U или статус не из множества C,2,4,6,7; учитывается спец. назначение фонда; при наличии КМИ (коэффициент многократного использования) экз. умножается на КМИ.
- **Студенты** — по БД RDR (поле 90) либо по VUZ (без RDR); форматы запросов SCH_STUD_RDR / SCH_STUD_VUZ (txt:1313, ~2904).
- **Приведённое ККО** — min(ККО, 1), потолок 1 (txt:1320).

**Полугодия / текущий семестр** (txt:1419–1424, 1273–1275):
- весеннее — 01.01–31.08, чётные семестры; осеннее — 01.09–31.12, нечётные.
- Текущий семестр — по календарю либо по «Дате окончания осеннего семестра» из настройки.
- Осенние данные (ОС) — месяц больше 8; весенние (ВС) — месяц меньше 9.
- При весеннем семестре среднее ККО = среднее арифметическое ОС и ВС; осенние берутся из архивного поля 692. Если одного значения нет — берётся другое ненулевое (txt:1274, 1322).
- Осенью формируются прогнозируемые данные по весне (студенты по осенним) (txt:1275).

**Выделенные фонды** (txt:1271–1272, 1324–1335): при подполе «спец. назначение фонда» в 910 ККО считается отдельно по каждому фонду; непомеченные экз. — общего пользования. Строки таблицы: выделенный фонд, Остальные (1 фонд), Общие экз-ры (2 и более фондов), Без учёта фондов. Колонка ККО+общ.экз. — ККО фонда + ККО от общих экземпляров.

**Электронные учебники (ЭУ)** — ККО = 1 независимо от бумажных экз. (txt:1195). Признак определяет формат KoElectro.pft (PftElectroName=KoElectro): код J02 в характере; 951^K + J; тип 900^t=компьютерные файлы (L) + поле 230 без 910; либо аналог-ЭУ при AnalogElRes (txt:1196–1202). Пометка — TextElecRes (по умолч. (ЭЛ)) (txt:1203).

**Точность ККО** (txt:1457–1472): DefaultModeSpec управляет учётом специализации: 1/пусто — спец.=код, специализация=код+.NN; 3 — точное совпадение; 2 — 6-значный код с двумя нулями = общий + варианты (плюс условие п.2).

**Поле 693 — структура** (Приложение 10, txt:2868–2937). Формируется заново (с удалением прежних) режимом «Связать», заданием «Формирование ККО для текущего семестра», после переноса, при возврате из корректировки. Подполя повторения:
B=годы обуч.; C=семестр (O/V); A=обозначение фонда; D=число экз.; K=экз. с КМИ; Q=экз. аналогов; E=число студентов (модельные поля 1001-1014); F=ККО (=K/E); N=ККО без учёта фондов; M=ККО остальных/общих; P=ККО с аналогами; S=среднее ОС+ВС (весной, из 692); X=служебное (NOFOND/COM); G=дата. Для весеннего при пустом ОС-ККО среднее = половина осеннего архивного (txt:2876–2878). Выбор ККО в списках 12.2: приоритет S, P, N, F (~1996–1999).

## Выходные формы и отчёты

**Генератор отчётов (раздел 12).** Форма из 3 частей: список отчётов, Отобрать по (WSS), Сформировать заголовки; параметры подаются в форматы как поле 991 (txt:1905–1911).
