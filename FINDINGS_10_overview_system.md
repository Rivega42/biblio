## Состав системы и архитектура

Архитектура клиент-сервер поверх TCP/IP, в локальных и глобальных сетях (txt:539). ПО написано на Delphi, платформа — Windows 2000/XP/NT и выше (txt:540-541).

Модули и АРМы (txt:529-537):
- TCP/IP Сервер БД ИРБИС64 — мониторинг процессов, авторизация пользователей, протоколирование (txt:530).
- АРМ "Администратор" — клиентский и серверный варианты; серверный работает в архитектуре файл-сервер с единым INI (txt:531, 571).
- АРМ "Каталогизатор", "Комплектатор", "Читатель", "Книговыдача", "Книгообеспеченность" (txt:532-536).
- Web-ИРБИС, J-ИРБИС — Web-доступ (отд. документация) (txt:537).

Установка двухэтапная: SETUP64.EXE → СЕРВЕР ИРБИС64 (порт по умолч. 6666, выделенный IP; вместе ставится серверный АРМ "Администратор") → КЛИЕНТЫ ИРБИС64 (txt:560-564). Локальный тест — IP 127.0.0.1 (txt:565). Тестовый вход MASTER/MASTERKEY (txt:568).

Профили пользователей — пара INI-файлов: клиентский (на машине, напр. CIRBISC.INI) + серверный (передаётся при авторизации, напр. IRBISC.INI); основные параметры — в серверных INI (txt:569-570, 3354-3361). Альтернативный клиентский INI задаётся первым параметром командной строки (txt:572-574). Какой серверный INI получит клиент — определяется таблицей на сервере (txt:576).

TCP/IP сервер (гл.7, txt:3294-3351): запуск IRBIS_SERVER.EXE; диспетчерские режимы — СПИСОК КЛИЕНТОВ ДЛЯ ДОСТУПА (ведение профилей, шифрование CLIENT_M.MNU), СПИСОК ЗАРЕГИСТРИРОВАННЫХ КЛИЕНТОВ (IP, имя, идентификатор, АРМ, время регистрации/подтверждения), СПИСОК ЗАПУЩЕННЫХ ПРОЦЕССОВ (активный/пассивный), ЗАПУСТИТЬ/ОСТАНОВИТЬ/ПЕРЕЗАПУСТИТЬ/ОТЛАДКА/ВЫХОД. Настройка — IRBIS_SERVER.INI; параметры CLIENT_TIME_LIVE, PROCESS_TIME_LIVE для авто-разрегистрации/прерывания (txt:3327, 3332, 3344, 3350).

Информационное обеспечение (БД, txt:543-557): CMPL (комплектование), IBIS (первая/единственная БД ЭК), RDR (читатели), RQST (заказы), HELP (Рубрикатор ГРНТИ), авторитетные файлы ATHRC/ATHRA/ATHRS/ATHRU/ATHRB, RSUDC/RSBBK (УДК/ББК), TEZ (тезаурус), POST, PODB, VUZ, KZD, MORPH, модельные IMAGE/ARCH/GUAR. Внутрисистемный формат базируется на UNIMARC/RUSMARC (txt:558).

Файловая структура БД (txt:578-604): каждая БД — каталог в \IRBIS64\DATAI\. Изменяемые данные = 5 файлов — .MST + .XRF (файл документов) и .IFP + .L01 + .N01 (словарь/инвертированный файл). Неизменяемые параметрические: .FST, .PFT, .STW, .MNU, .OPT, .TAB, .SRT, .HDR, .WS, .WSS, .SRW, .GBL, .TRE и др. Списки доступных БД: DBNAM1/2/3.MNU. Файл DBNAME.PAR — физическое размещение файлов; 11-я строка задаёт путь к внешним объектам с относительной адресацией (поле 951, подполе A) (txt:603).

## Модель данных и структура записи

Запись (документ) состоит из полей: структурная единица с числовой меткой (МММ), содержащая один элемент описания (txt:4327). Поле может делиться на подполя — двумя символами: знак "крышка" (^) + разделитель подполя (латинская буква или цифра, регистр не различается) (txt:4328).

Правила (Прил.2, текстовый формат, txt:4273-4288):
- Документ начинается с новой строки, разделитель документов — строка ***** в первых позициях.
- Поле: "#МММ: данные" (лидирующие нули метки можно опускать).
- Поля могут идти в произвольном порядке; поля с одинаковыми метками повторяются (повторяющиеся поля).
- Подполя: ^A данные ^B данные ...; подполя с одинаковыми разделителями внутри поля не повторяются.

MFN — внутренний номер записи в файле документов; строго говоря, не поле (метки не имеет) (txt:4358, 4422-4425). Команда MFN / MFN(d) выводит d цифр (по умолчанию 6) (txt:4424). Пример учебной записи — MFN: 34 из БД IBIS (txt:4336-4355).

Статусы записи (АРМ Администратор, txt:1407, 2989, 2995, 3008, 3060):
- логически удалённые (могут иметь предыдущие копии) vs физически удалённые («дырки», копий не имеют);
- заблокированные (в т.ч. «ложная» блокировка после аварии АРМ Книговыдача — снимается по MFN или списком);
- неактуализированные — записи, для которых не выполнена авто-актуализация словаря.
В обмене статус передаётся конструкцией "mfn # статус записи" (при возврате не учитывается) (txt:1979-1985).

## Технология поиска (Прил.5, 6, 17)

Инвертирование (словарь). Словарь БД формируется автоматически из ТВП (таблица выбора полей, файл .FST) (txt:5366-5378). Строка ТВП = идентификатор поля (ИП) + метод индексирования (МИ) + формат выборки. По каждому документу: отработать формат выборки -> применить метод индексирования -> присвоить ИП. Методы индексирования 0–8 (txt:5388-5421): 0 — элемент из строки (всё поле/подполе); 1 — из подполя/строки (нужен режим mpl); 2 — фразы в угловых скобках; 3 — фразы в косых чертах; 4 — слова (по ISISACW.TAB, со стоп-словами .STW); 5–8 = 1–4 с присоединением префикса. Термин словаря = префикс инверсии + собственно термин (напр. T=, A=, K=, U=, V=).

Ссылка термина (трёхкомпонентная): MFN документа, метка поля (TAG / вторая часть ссылки), номер повторения (OCC / третья часть) (txt:5129, 5599, 6877, 6883). На этом строятся квалификация /(tag,...) и контекстные операторы.

Сценарий поиска (Прил.6, секция [SEARCH] INI; для БД — одноимённый INI) (txt:5483-5492). Один вид поиска по словарю задают параметры ItemNameNN, ItemPrefNN (префикс инверсии), ItemDictionTypeNN, ItemMenuNN, ItemLogicNN, ItemTrancNN (усечение), ItemAdvNN (расширение по авторитетному файлу/тезаурусу), ItemPftNN, ItemExactlyNN и др.; число — ItemNumb (txt:5495-5581). Типы словаря (ItemDictionType): 0 — стандартный; 1 — с раскодировкой (через .MNU); 2 — тематический рубрикатор; 3 — на технологии ИРБИС-Навигатора (txt:5520-5524).

Логика (ItemLogicNN, txt:5563-5568): 0 — только ИЛИ; 1 — ИЛИ+И; 2 — ИЛИ+И+НЕТ (по умолч.); 3 — +И(в поле); 4 — +И(фраза).

Язык запросов (Прил.17, txt:6859-6916). Два вида: прямой (по словарю/инвертированному файлу, быстрый) и последовательный (перебор записей, медленный, с БОЛЬШЕ/МЕНЬШЕ, НАЛИЧИЕ/ОТСУТСТВИЕ).

Операнд прямого поиска: "префикс термин $" /(tag1,...tagN) — где $ = правое усечение (без $ — точное значение); кавычки обязательны при пробеле/скобках/символе # /символах операторов; /(...) — квалификация по меткам полей (txt:6868-6877).

Логические операторы и приоритеты (по убыванию) (txt:6878-6891):
- точка (в пробелах) — контекстное И «друг за другом» (совпадают 2-я и 3-я части ссылок, 3-и отличаются на 1) — высший приоритет;
- (F) — контекстное И в одном повторении поля;
- (G) — контекстное И в одном поле;
- символ * (И) и символ ^ (НЕ) — один уровень;
- символ + (ИЛИ) — низший.
Скобки меняют порядок; выражения в скобках объединяются только операторами + * ^ (txt:6892). Примеры — txt:6896-6900.

Последовательный поиск — запрос есть формат вида "if лог.выражение then '1' else '0' fi"; запись отбирается, если результат содержит '1' (txt:6903-6915). В режиме СВОБОДНЫЙ ПОИСК задаётся только лог.выражение, опционально с префиксом |МММ (метка поля). Регулярный последовательный поиск -> пересмотреть .fst (txt:6916).

Дополнительно: контекстные уточнения для «Ключевые слова» (CvalifNameNN/CvalifValueNN, txt:5583-5605); поиски по связи (ScntNameNN/ScntFormatNN/ScntPrefNN/ScntSuffixNN/ScntLogicNN — термины формируются форматом из исходного документа, txt:5607-5640); поиски «Для чайников» (префикс К=/DS=) и «Для умников» (интеллектуальные: авторитетные файлы, УДК/ББК, тезаурус) (txt:5646-5654).

## Язык форматирования PFT (Прил.4)

Назначение: показ/печать, создание словарей, импорт/экспорт, ФЛК и др. Базируется на языке форматирования CDS/ISIS (ЮНЕСКО) (txt:4323-4324). Формат = набор команд и функций, разделённых запятыми/пробелами; сохраняется в .PFT; комментарий начинается с "/*" до конца строки; регистр команд произволен (txt:4329-4333). Система обрабатывает три объекта: запись, формат, рабочую область (txt:4330).

Основные конструкции:
- Вывод поля: V метка (напр. v200); подполя: v200^a, v600^* (первое подполе) (txt:4359-4378).
- Фрагмент: *смещение.длина (нумерация с 0), напр. v1*3.3 (txt:4393-4420).
- MFN: MFN / MFN(d) (txt:4422).
- Режимы вывода Mmc: P — проверка (поля как есть), H — заголовок (разделители подполей -> пунктуация), D — данные (+ авто-точка и два пробела); регистр c — преобразование в прописные (txt:4440-4455). Стандартная замена разделителей: ^a -> "; ", ^b..^i -> ", ", прочие -> ". " (txt:4443-4446).
- Команды размещения (вертикаль/горизонталь): / (новая строка), #, %, c/x (горизонталь) (txt:5099).
- Литералы (безусловные/условные), фиктивный вывод поля, выражения (числовые/строковые/логические), функции: числовые VAL, RSUM, RMIN, RMAX, RAVR, L; строковые F, REF, S; логические P (наличие поля) и A (отсутствие); форматные выходы UNIFOR &uf(...)/&unifor(...) (txt:331-376 оглавл.; txt:6724, 6756, 6774).

Условные/ветвящиеся форматы (txt:5006-5021): IF условие THEN формат-1 ELSE формат-2 FI (ELSE/формат-1 опциональны; IF/THEN/FI обязательны; вложенность любая, FI закрывает каждый IF как скобка). Ветвление по типу записи: для интегрированных БД с разными типами записей в одном формате через IF, проверяя поле-код типа документа (txt:5021).

Повторяющиеся группы (...) (txt:5022-5079): обрабатывают повторения поля поэкземплярно (счётчик повторений с 1). Не могут быть вложенными, кроме как в аргументе REF. Обязательны при использовании повторяющегося поля в L, в первом аргументе REF.

RTF и HTML-тэги в форматах (txt:5080-5116): RTF-формат — весь в фигурных скобках, команды начинаются с обратного слэша; HTML-формат помечается пустым тэгом в начале; в обоих неприменима вертикальная верстка / # %. Предусмотрены ИРБИС-ссылки в тэгах A HREF и IMG SRC, показ штрих-кода через спец-ссылку IRBIS:!!ШТРИХ_КОД!!.

Форматы ФЛК (формально-логический контроль, txt:5117-5125): первый символ результата — 0 (ок), 1 (непреодолимая ошибка), 2 (преодолимая); остальное — сообщение пользователю. Имена начинаются с "!" (напр. !200, !907).

Форматы с переменными метками *** (для переноса данных при вводе через словарь; исходное — термин словаря, метки заменяются TAG из ссылки, берётся строка OCC) (txt:5126-5132).

Список форматов системы (Прил.4 п.14, txt:5133-5143): десятки именованных форматов для БД IBIS/CMPL/RDR — ФЛК (!NNN), вводы через словарь (amov*, mmov*, tmov*), корректировка по словарю (dmod*), поиски по связи (scnt*), просмотр (brief, ibisw, rmarcw — RUSMARC, umarcw — UNIMARC, usmrcw — USMARC), каталожные карточки (kkk*), табличные формы (tab*).

Оптимизированный формат (txt:5144-5163): авто-переключение формата показа по виду документа (файл .OPT, параметр PFTOPT); ключ — метка/формат/@формат; маскирование +. Для IBIS — PFTW.OPT (RTF) и PFTW_H.OPT (HTML, по умолчанию).

## Рабочие листы и ввод (Прил.3, 8)

Справочники (txt:4293-4318): экранные меню и раскодировка. Простые (неиерархические) — текстовые .MNU: пары «элемент / пояснение», логический конец *****, число строк 2N+1. Иерархические — .TRE (древовидные, как в Windows). Списки доступных БД — DBNAM1/2/3.MNU; коды стран STR.MNU, языков JZ.MNU, кодовые наборы UNICODE.MNU и др. Редактор справочников и РЛ запускается из АРМа Администратор (Прил.8).

Рабочие листы (РЛ) ввода — .WS (РЛ полей) и вложенные .WSS (РЛ подполей); список доступных РЛ — FMT31.MNU (txt:4309-4310, txt:582). Редактор РЛ и справочников — Прил.8 (txt:449-476): редактирование строк/страниц, столбцы РЛ полей (метки, методы ввода), вложенные РЛ, ТВП. Способы ввода в АРМ Каталогизатор: меню-справочник, словарь, вложенный РЛ, многостроковое окно, Рубрикатор ГРНТИ, авторитетные файлы/АПУ/тезаурус, групповой ввод повторений, оперативные меню, ИРБИС-Навигатор (txt:134-143).

## Форматы обмена и MARC

- Система полностью совместима с RUSMARC/UNIMARC/MARC21 через средства двухсторонней конверсии; внутрисистемный формат базируется на UNIMARC/RUSMARC (txt:516, 558).
- Текстовый файл документов (Прил.2) — формат импорта/экспорта в режимах ИМПОРТ/ЭКСПОРТ АРМов Каталогизатор и Администратор: "#МММ: данные", разделитель документов *****, подполя через ^X (txt:4271-4290).
- Просмотр/конверсия в MARC обеспечивается форматами: rmarcw (RUSMARC), umarcw (UNIMARC), usmrcw (USMARC) (txt:5136).
- Переформатирование при обмене — через ТВП (.FST): списки IMPORTW/EXPORTW/FSTW.MNU (txt:4309-4310, Прил.5).
- Режим АРМа Администратор СОЗДАТЬ ISO-ФАЙЛ по последним копиям MST (txt:305) — указывает на поддержку ISO 2709 при обмене. TODO ниже.

## Глоссарий терминов и сокращений

- АРМ — автоматизированное рабочее место (txt:529).
- ЭК — Электронный каталог; БД — база данных (txt:517).
- MFN — внутренний (machine) номер записи в файле документов (txt:4358).
- Поле / метка (TAG) — структурная единица записи с числовой меткой (txt:4327).
- Подполе / разделитель подполя — символ ^ + латинская буква/цифра (txt:4328).
- OCC — номер повторения поля (txt:5129).
- Ссылка термина — (MFN, TAG, OCC) (txt:5129, 6877).
- Словарь / инвертированный файл — .IFP/.L01/.N01 (txt:583-590).
- ТВП — таблица выбора полей, .FST (txt:5366).
- РЛ — рабочий лист ввода, .WS/.WSS (txt:582).
