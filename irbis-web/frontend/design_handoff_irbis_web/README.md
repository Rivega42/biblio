# Передача в Claude Code — Веб-каталог библиотеки (`irbis-web`)

## TL;DR (что сделать в Claude Code)
1. Скачайте проект целиком (zip) и распакуйте.
2. Положите папку как **Agent Skill**: `~/.claude/skills/library-catalog-design/` (или
   `<repo>/.claude/skills/library-catalog-design/`). Точка входа — `SKILL.md` в корне.
3. В Claude Code: «Используй скилл **library-catalog-design**, реализуй экраны на нашем стеке
   React + TypeScript + Tailwind по дизайн-референсам».
4. Claude Code прочитает `SKILL.md` → `readme.md` → токены/компоненты/UI-кит и будет
   проектировать в вашей кодовой базе как эксперт по этому бренду.

## Что это за файлы (важно)
Файлы в этом пакете — **дизайн-референсы на HTML/JSX**: прототипы, показывающие задуманный вид и
поведение, **а не продакшн-код для копирования один-в-один**. JSX-компоненты реальны, но стилизованы
через **CSS-переменные**, а не Tailwind. Задача в Claude Code — **воссоздать эти экраны в вашем
окружении** (React + TS + Tailwind по ТЗ §10) вашими паттернами и библиотеками: перенести токены в
`tailwind.config`, пересобрать компоненты, подключить данные к реальному REST/WebSocket-бэкенду
вместо моков.

## Фиделити
**High-fidelity.** Цвета, типографика, отступы, радиусы, состояния — финальные. Воссоздавайте
пиксель-в-пиксель, опираясь на токены ниже. Логика поиска/заказа/очереди — на **моках** (контракт в
`ui_kits/irbis-web/data.js`); это намеренно: фронт собирает параметры и зовёт бэкенд.

## Стек и архитектурные правила (ТЗ §10–§12)
- **React + TypeScript + Tailwind** (зафиксировано проектом).
- **Тонкий клиент:** никакой ИРБИС-логики/разбора протокола/построения поисковых выражений на
  клиенте — это бэкенд. Фронт собирает параметры и зовёт REST/WebSocket.
- **Конфиг базами:** поля поиска, фильтры, формат карточки, спецформы — читаются из конфигурации
  базы (см. `data.js`), не хардкодятся. Добавление базы = добавление конфига.
- **Навигация по грантам, а не по АРМам:** меню задач собирается из грантов учётки
  (`data.js → staff.grants`). Показывать только разрешённое.
- **Безопасность UI:** без реальных ПДн; минимум ПДн в ЛК; служебные ошибки не показывать
  пользователю (только человеческий текст); файлы — view-only через прокси, без прямых ссылок; без
  внешних CDN/трекеров; шрифты локально; статусы — цвет + иконка + текст.
- **Тематизация токенами:** светлые/тёмная/праздничные/a11y + **скин на библиотеку** — наборы
  CSS-переменных, переключаются через `data-theme` без перезагрузки. В Tailwind — через
  `darkMode: ['selector', '[data-theme="dark"]']` и плагин с CSS-переменными (см. ниже).

## Дизайн-токены → Tailwind
Все значения — в `styles.css` → `tokens/*.css`. Рекомендуемый способ: завести CSS-переменные как
есть и сослаться на них из `tailwind.config` (тогда смена `data-theme` работает «бесплатно»).

```js
// tailwind.config.js (фрагмент)
export default {
  darkMode: ['selector', '[data-theme="dark"]'],
  theme: {
    extend: {
      colors: {
        // семантические алиасы → var(), чтобы темы переключались автоматически
        page: 'var(--bg-page)', card: 'var(--surface-card)', sunken: 'var(--surface-sunken)',
        strong: 'var(--text-strong)', body: 'var(--text-body)', muted: 'var(--text-muted)',
        subtle: 'var(--text-subtle)', link: 'var(--text-link)',
        accent: { DEFAULT: 'var(--accent)', fg: 'var(--accent-fg)', weak: 'var(--accent-weak)', press: 'var(--accent-press)' },
        line: { subtle: 'var(--border-subtle)', DEFAULT: 'var(--border-default)', strong: 'var(--border-strong)' },
        status: { available: 'var(--status-available)', issued: 'var(--status-issued)', unknown: 'var(--status-unknown)' },
      },
      borderRadius: { xs:'3px', sm:'5px', md:'8px', lg:'12px', xl:'16px', pill:'999px' },
      spacing: { /* 4px-сетка: 1=4 2=8 3=12 4=16 5=20 6=24 7=28 8=32 10=40 12=48 16=64 20=80 */ },
      boxShadow: {
        xs:'0 1px 2px rgba(28,27,25,.06)',
        sm:'0 1px 3px rgba(28,27,25,.09),0 1px 2px rgba(28,27,25,.05)',
        md:'0 4px 14px rgba(28,27,25,.10),0 2px 4px rgba(28,27,25,.05)',
        lg:'0 14px 40px rgba(28,27,25,.16),0 4px 10px rgba(28,27,25,.06)',
      },
      maxWidth: { container:'1240px', content:'760px' },
      fontFamily: {
        ui: ['PT Sans','Segoe UI','system-ui','sans-serif'],
        display: ['PT Serif','Georgia','serif'],
        mono: ['PT Mono','ui-monospace','monospace'],
      },
    },
  },
};
```
Импортируйте `styles.css` (он `@import`-ит `tokens/*.css` + `tokens/fonts.css`) один раз в корне
приложения — он объявляет переменные и `@font-face` (локальные PT-шрифты, без сети).

### Шрифты
PT Sans / PT Serif / PT Mono. В пакете объявлены через `@font-face` с `local()` (используют
установленные в ОС начертания; PT Astra присутствует в Astra Linux). Для веба положите `.woff2` в
`/fonts` и добавьте `src: url(...)` перед `local()` в `tokens/fonts.css`. Внешние фонт-хостинги
запрещены (контур).

## Темы (8 скинов)
`data-theme` на корневом контейнере. Только семантические алиасы меняются между темами.
- Светлые: `working` (бумага, по умолчанию), `azure` (Лазурь), `pine` (Хвоя), `theatrical` (Театр).
- `dark` (тёмная). Праздничные: `newyear`, `march8`. Доступность: `a11y` (высокий контраст,
  крупнее кегль, толще фокус — ГОСТ Р 52872-2019).
- **Скин на библиотеку:** `data.js → libraries[].theme`. Выбор библиотеки задаёт бренд (название,
  монограмма) и тему по умолчанию; пользователь может переопределить тему вручную.

## Компоненты (см. `components/<group>/`)
У каждого — `.jsx` + `.d.ts` (контракт пропсов) + `.prompt.md` (как/когда использовать). Иконки —
только `<Icon name="…">` (локальный набор, без сырых SVG/эмодзи).
- **forms/**: Button, IconButton, Input, Select, Checkbox, Radio, Switch, FilterChip.
- **catalog/**: DatabaseSelector (иерарх. мультиселектор), SearchBar, SearchModes, TreeNav
  (ГРНТИ/УДК/ББК), ResultCard (+ тег базы), StatusBadge, HoldingsTable, PftBlock, Pagination,
  SubjectTag.
- **viewer/**: FileViewer (view-only: без скачивания, контекстное меню заблокировано, приоритет
  955>951, подсветка терминов + релевантные страницы).
- **cataloging/**: DynamicField (тип поля → контрол: текст / меню .mnu / словарь / дерево .tre /
  да-нет / авторитет / дата; повторяемые поля и подполя; ФЛК).
- **feedback/**: Alert, Toast (+ ToastViewport), Dialog, Skeleton (+ SkeletonResult), EmptyState,
  Badge. **navigation/**: Tabs.

## Экраны / потоки (см. `ui_kits/irbis-web/`)
Контекст переключается в шапке: **Читатель / Сотрудник**. Конфиг и моки — `data.js`.
- **Читатель:** `HomeScreen` (выбор баз, «Я ищу:»), `ResultsScreen` (режимы простой/расширенный/
  комплексный/спецформа, фасеты с кол-вом, навигатор-деревья, мультибаза, пагинация сверху+снизу,
  отмеченные, состояния загрузка/пусто/ошибка), `RecordScreen` (сворачиваемые блоки, закладки
  Экземпляры/Эл.версии/Сиглы, связи 488/390, PFT, заказ), `SpecialForm` (PLAY «Роли», TUAR/IMAGE
  «Дата события», GUAR Фонд/Опись), `AccountScreens` (вход по билету; Формуляр с продлением,
  Корзина заказов, Закладки, Постоянные запросы, Уведомления, Оплата штрафов), `OrderModal`
  (заказ в 3 шага).
- **Сотрудник:** `StaffDesktop` (рабочий стол по грантам), `CatalogingWorksheet` (рабочий лист из
  профиля базы), `Circulation` (выдача/возврат/очередь/бронеполка/ячейки), `Inventory` (сверка с
  ТСД, офлайн-индикатор), `Dashboard` (BI: KPI, выдачи по месяцам, доля по базам).

## Контракт данных (моки → будущий REST)
Точная форма — в `ui_kits/irbis-web/data.js`: `databases` (конфиг баз + спецформы + навигаторы),
`results` (с `sourceDb`), `records` (PFT, `holdings` 910, `files` 951/955, `links` 488/390, `sigla`),
`account` (минимум ПДн), `staff`/`staffData` (гранты, очередь, бронеполка, инвентаризация,
дашборд), `catalogingProfiles` (рабочие листы), `libraries` (бренд + скин). Поля MARC: 200^a,
700^a/^b, 210^c/^d, 910^a/^b/^d/^h, 600/601, 951/955, 122^a/^b.

## Файлы в пакете
- `readme.md` — полное руководство (контекст, контент, визуальные основы, иконография, индекс).
- `styles.css`, `tokens/*.css` — токены и шрифты (источник правды).
- `components/<group>/` — компоненты (`.jsx` + `.d.ts` + `.prompt.md` + карточка `@dsCard`).
- `ui_kits/irbis-web/` — кликабельный прототип (`index.html`) + экраны (`*.jsx`) + `data.js`.
- `guidelines/*.card.html` — специмены (цвет/тип/отступы/бренд).
- `assets/` — иконки/логотипы/изображения.
- `SKILL.md` — точка входа Agent Skill для Claude Code.
