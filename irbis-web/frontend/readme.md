# ИРБИС-Веб — Дизайн-система

Дизайн-система и фронтенд-кит для **публичного веб-каталога библиотеки** (`irbis-web`) —
импортозамещающей замены вендорскому J-ИРБИС. Каталог работает поверх ИРБИС64, в защищённом
сетевом контуре, на Astra Linux. Библиотека профильная — **театрально-художественная**
(пьесы, либретто, эскизный фонд, драматическая цензура, календарь премьер), что задаёт
сдержанный «библиотечно-архивный» характер с театральным акцентом.

Эта система покрывает **визуальный слой и фронтенд**: токены, типографику, цвет, темы,
библиотеку компонентов на React и кликабельный прототип каталога на моках. Логика протокола
ИРБИС, бэкенд и форматирование PFT — вне зоны дизайн-системы (фронт только отображает данные).

> **Источники.** Система построена по ТЗ «Веб-каталог ИРБИС (`irbis-web`)», версия 1.0 от
> 16.06.2026 (документы проекта: `PROJECT_BRIEF.md`, `ARCHITECTURE.md`, `IRBIS_PROTOCOL.md`,
> `FUNCTIONAL_SCOPE.md`, `SECURITY.md`). Кодовая база или Figma не передавались — визуальное
> направление выбрано с нуля по разделу 4 ТЗ и согласовано с заказчиком (см. «Принятые решения»).

## Принятые решения (по открытым вопросам ТЗ, разд. 14)

- **Две темы оформления + тема доступности.** Тема по умолчанию — **«рабочая»**: спокойная,
  читаемая, с чернильно-синим акцентом — «то, с чем приятно работать любому». Для читательской
  части предусмотрен **«театральный» скин** (пергамент + винный/гранатовый акцент + засечки в
  заглавиях). Обе строятся на одном слое токенов; переключение мгновенное и не ломает вёрстку.
- **Акцент:** рабочая тема — чернильно-синий `#2C5578` (AA на белом); театральная — гранат `#7A2E3A`.
- **Шрифты — локальные/системно-безопасные стеки** (`PT Sans` / `PT Serif` / `PT Mono` с
  фолбэком). Без внешних шрифт-хостингов (разд. 11). PT-семейство присутствует в Astra Linux.
- **Пилотные базы (демонстрация конфиг-управляемости):** «Электронный каталог» (эталонная,
  список), «Эскизный фонд» (изобразительная, галерея/превью), «Архивные документы GUAR»
  (фонд/опись/диапазон дат), «Календарь премьер». Экраны рендерятся из конфига базы.

## CONTENT FUNDAMENTALS — как написан текст

- **Язык — русский**, обращение нейтрально-уважительное, без «вы»/«ты» там, где можно обойтись
  безличной формой: «Найдено 42 записи», «Войдите по номеру билета», «Заказ принят».
- **Тон** — спокойный, точный, учрежденческий, но не казённый. Без восклицаний, без маркетинга.
  Подписи короткие и предметные: «База поиска», «Расширенный поиск», «Экземпляры», «Вернуть до».
- **Регистр.** Заголовки — обычным предложным регистром («Личный кабинет», «Карточка записи»).
  ВЕРХНИЙ РЕГИСТР только для мелких служебных меток (eyebrow): «ВЕБ-КАТАЛОГ», «БАЗА ПОИСКА».
- **Ошибки — человеческим языком, без служебных деталей** (никаких стеков, кодов протокола,
  адресов узлов): «Каталог временно недоступен. Повторите попытку позже», «Билет не найден».
- **Статусы дублируются текстом**, не только цветом: «Доступен», «Выдан», «Нет данных».
- **Эмодзи не используются.** Числа — без «слопа»: только осмысленные (количество найденного,
  частоты словаря, сроки возврата, инвентарные номера). Тысячи — с разделителем (`214 530`).
- Примеры микротекста: «Заказ принят и поставлен в очередь», «Доступно в читальном зале или
  после входа по читательскому билету», «Демонстрационные данные обезличены».

## VISUAL FOUNDATIONS — визуальные основы

- **Контент важнее хрома.** Главный объект — запись каталога; интерфейс уступает место данным.
  Минимум декора, спокойные поверхности, акцент строго по делу.
- **Цвет.** Тёплая бумажная основа (`#FBFAF7`) и графитовый текст (`#1C1B19`). Один сдержанный
  акцент на тему. **Единственное место яркого цвета — статусы экземпляров**: зелёный
  «доступен», янтарный «выдан», серый «нет данных». Театральный скин уводит палитру в пергамент
  и гранат; a11y-тема — в чистый чёрно-белый высокий контраст.
- **Типографика.** Гротеск для интерфейса (`--font-sans`), засечки для заглавий карточек в
  театральном скине (`--font-serif`), моноширинный для инвентарных номеров/MFN/кодов полей
  (`--font-mono`). Шкала на множителе `--fs` (16px), a11y-тема поднимает базовый кегль до 18px
  и интерлиньяж — масштабирование без переписывания значений. Минимум для интерфейса — 13px.
- **Сетка.** Контентная ширина до 1240px. Desktop — двухколоночная схема «фильтры/словарь слева
  — результаты справа» (`--rail-filters: 288px`). Mobile — рейл сворачивается. Шаг сетки — 4px.
- **Фон.** Сплошные тёплые поверхности, без градиентов, без фоновых изображений и текстур.
  Изображения — только контентные (превью эскизного фонда), их можно полностью отключить (a11y).
- **Скругления.** Поля/кнопки `8px` (md), карточки `12px` (lg), модалки `16px` (xl), чипы/бейджи —
  pill. Единообразно по всем темам.
- **Границы и тени.** Опора на тонкие тёплые границы (`#E6E0D4` / `#D6CEBE`), а не на тяжёлые
  тени. Тени сдержанные, тёплый чёрный, четыре ступени (xs→lg); крупные — только для модалок,
  тостов и поповеров. В a11y-теме тени заменяются жёсткими границами.
- **Карточки** — поверхность `--surface-card`, тонкая граница, на hover слегка усиливается
  граница + появляется `shadow-sm`. Отмеченная карточка результата — акцентная подложка/граница.
- **Движение — сдержанное.** Длительности 120/180/280мс, плавный `ease-standard`/`ease-out`.
  Фейды и мягкие смещения (тосты, модалки), без отскоков и бесконечных анимаций. Полностью
  уважается `prefers-reduced-motion`.
- **Состояния наведения/нажатия.** Hover — смена фона на `surface-hover`/`accent-weak` или
  затемнение акцента (`accent-hover`); нажатие — `accent-press` + микросдвиг кнопки на 0.5px.
- **Фокус.** Видимый всегда: кольцо `3px` цвета акцента (a11y — `4px` сплошное). Логический
  порядок табуляции, цели ≥40px (минимум касания 44px).
- **Прозрачность/блюр.** Только подложка модалки (полупрозрачный тёплый чёрный + лёгкий blur).
  В контенте прозрачность не используется.

## ICONOGRAPHY — иконографика

- **Локальный линейный набор** (`components/icon/Icon.jsx`) — единый источник иконок. 24×24,
  обводка `currentColor`, толщина 1.75, скруглённые концы; стиль Lucide/Feather. ~55 иконок
  (`search`, `book`, `images`, `archive`, `calendar-star`, `filter`, `sliders`, `check-circle`,
  `clock`, `x-circle`, `accessibility`, `tag`, `bookmark` и т. д.). Полный список — `ICON_NAMES`.
- **Почему локально, а не CDN.** Каталог работает в защищённом контуре без внешних CDN/трекеров
  (разд. 11). Набор встроен в код; при необходимости заменяется на self-hosted Lucide один-в-один
  (тот же стиль и веса).
- **Правила.** Иконки ставятся ТОЛЬКО через `<Icon>` — никаких сырых SVG и эмодзи в интерфейсе.
  Декоративные иконки скрыты от скринридера (`aria-hidden`); значимые получают `label`.
  Статусы экземпляров — всегда цвет + иконка + текст.
- **Логотип** — плейсхолдер (`assets/logo-irbis.svg`, `assets/mark-irbis.svg`): монограмма
  «раскрытая книга» + словесный знак, наследует цвет темы через `currentColor`. ⚠️ Заменить на
  настоящий знак учреждения, когда он будет предоставлен.

## Передача (handoff)

- **Темы** переключаются атрибутом `data-theme` на корневом контейнере: без значения — рабочая,
  `data-theme="theatrical"` — театральный скин, `data-theme="a11y"` — высокий контраст. Меняются
  только семантические алиасы → переключение без перезагрузки.
- **Конфиг базы — данными** (`data.js` → `databases[]`): поля поиска, фильтры, layout (`list` /
  `gallery`), флаг `dateRange`. Экраны читают конфиг; добавление базы = добавление конфига.
- **PFT-блок** (`PftBlock`) принимает серверный HTML; в проде вставлять только через санитайзинг
  (передать `sanitize`). В моках html обезличен и безопасен.
- **Точки подключения бэкенда** в прототипе: `runQuery()` (поиск), `recordFor()` (карточка),
  `login()` / `confirmOrder()` (ЛК/заказ) — заменяются на REST/WebSocket. Состояние — в React
  (без localStorage в артефакте, как требует разд. 10).
- **Стек:** React + TypeScript + Tailwind (целевой). В дизайн-системе компоненты на JSX с
  токенами-переменными; контракты — в `.d.ts`. Tailwind-конфиг строится поверх этих же токенов.

См. также `SECURITY` ниже.

### Безопасность и приватность (отражено в UI)
Только обезличенные данные; в ЛК — минимум ПДн (вход по номеру билета); RFID и служебные id
читателю не показываются; служебные ошибки не выводятся; ссылки «поделиться» — внутри контура;
никаких внешних CDN/трекеров/шрифт-хостингов.
