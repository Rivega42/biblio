# Веб-ИРБИС — Индекс комплекта документов (нить исследования)

> Карта всех материалов проекта: восстановление спецификации ИРБИС64+ и проектирование веб-версии
> всех функций (организованной по учёткам/грантам). Читать сверху вниз. Обновлено 20.06.2026.

---

## 1. С чего начать (порядок чтения)

1. **`IRBIS_CAPABILITY_MAP_v2.md`** — сводная мастер-карта возможностей продукта. Точка входа.
2. **`ACCESS_MODEL_web-irbis.md`** — модель доступа: учётка = набор грантов (функция × база × уровень),
   роли = пресеты. Организующий стержень веб-версии.
3. **`FUNCTION_PROTOCOL_MAP.md`** — функция → команды протокола `IC_*` + требуемый АРМ-контекст.
4. **`TZ_CLAUDE_CODE_irbis-web.md`** + **`TZ_CLAUDE_CODE_ADDENDUM_access-suite.md`** — инженерное ТЗ на
   сборку (ядро + слой доступа + модули-функции по грантам; фазы P0…P4).

## 3. Первоисточники и методика

- **`irbis64_client_dll_TEXT.txt`** — расшифрованная официальная дока API (полный протокол).
- **`ReadMe_jirbis2_text.txt`** — текст мануала J-ИРБИС 2.0 (поведенческий референс).
- **`TZ_IRBIS_RECON.md`** — методика 3-проходной картографии (статика / живой протокол / UI клиента).
- **`PROMPT_CLAUDECODE_RECON.md`** — промт для НЕЗАВИСИМОЙ перепроверки Клод Кодом (без моих ответов,
  с реестром полноты) — для сравнения двух карт.

## 4. Статус

**Сделано (из архива):** полный каталог протокола; реестр 51 базы; механика конфигов (префиксы,
словарь, формы, форматы, связи, пути); GUAR-488; структура всех 6 настольных клиентов (источник функций); веб-модель вендора; модель
доступа; привязка функций к протоколу; инженерное ТЗ + аддендум доступа/полного набора функций.

**Полнота:** сильная карта на уровне структуры/механики; **не** исчерпывающая до каждого поля
(PFT/WSS/CHM сэмплированы). Достаточно для архитектуры и каркаса, не для прямой генерации боевого
кода по каждому полю.

**Осталось (вне архива — требует действий):**
1. **Проход Б — живой сервер `:6666`:** форматы пакетов/коды результата, права `RIGHT`, протокол очереди заказов. Запускать локально (WSL/Windows).
2. **Конфиги боевых баз СПб ГТБ** (PLAY/TUAR/EK/ESKIZ/HPO) — с сервера библиотеки; механику применить ту же.
3. **Независимая перепроверка** Клод Кодом по `PROMPT_CLAUDECODE_RECON.md` → сверка с этой картой.
4. **Уточнения под СПб ГТБ:** уровни/пресеты-роли; точные правила полей Каталогизатора (irbiscat.doc разд.4).

## 5. Следующий шаг (рекомендация)

Каркас P0: ядро (протокол-клиент + поиск + серверный PFT) + модуль Access (учётки/гранты/резолвер/
пул сессий/аудит) + веб-Читатель (каталог СПб ГТБ). Параллельно — Проход Б на живом сервере для
подтверждения протокола и снятия боевых баз.
