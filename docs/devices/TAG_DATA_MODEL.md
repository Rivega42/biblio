# TAG_DATA_MODEL — байтовая раскладка метки (ISO 28560 / Danish), из декомпиляции ядра

> **Закрывает** [OPEN_QUESTIONS.md](OPEN_QUESTIONS.md) §8 (побайтовая раскладка ISO 28560/Danish) и углубляет [DEVICE_CONTROL_MAP.md](DEVICE_CONTROL_MAP.md) §4.
> **Источник:** декомпиляция нативного `EasyBook_RFid.dll` (Ghidra headless, `_re_tools/easybook_decomp.c`). Ссылки — `FUN_адрес`. Уровень достоверности помечен: **[декомп]** подтверждено кодом / **[стандарт]** соответствует ISO 28560-2 / **[инференс]**.
> **Главный вывод:** IDlogic пишет на метку **стандартный ISO 28560-2** (без проприетарного формата). Значит для совместимости с уже промаркированным фондом Biblio достаточно **стандартного кодека ISO 28560-2** — отдельный «секретный» формат реверсить не нужно.

## 1. Физический уровень — ISO 15693, блоки

HF-метка = **ISO 15693** (NXP ICODE SLI/SLIX). Доступ к данным — блочный:
- Чтение: `ISO15693_ReadMultiBlocks` (экспорт), FEIG-путь `CLSCRF_ReadMultipleBlocks_15693`, RH-путь `RH_Reader_ISO15693_*`. **[декомп]**
- Запись: `ISO15693_WriteMultipleBlocks` / `CLSCRF_WriteSingleBlock_15693` / `RH_Reader_ISO15693_WriteBlock`. **[декомп]**
- Системная информация (число блоков, размер блока, UID): `ISO15693_GetSystemInfo` / `CLSCRF_GetSystemInfo_15693`. Размер блока у ICODE SLI/SLIX — **4 байта**, блоков 28 (112 байт user memory). **[стандарт]**

Данные пользователя = **непрерывный поток «объектов данных» ISO 28560-2**, упакованный в блоки. **[декомп/стандарт]**

## 2. Структура «объекта данных» (ISO 28560-2) — из декодера `FUN_0043c520`

Декодер `FUN_0043c520(buf, len, →out)` разбирает поток байт метки. Каждый объект:

```
байт 0 (precursor):
   бит 7      = флаг «offset present» (расширенный заголовок)
   биты 6..4  = КОД КОМПАКЦИИ (1..7) — выбирает способ декодирования содержимого
   биты 3..0  = относительный OID (тип данных); для валидного первого объекта = 1
если бит7=1:  байт1 = offset (local_9), байт2 = длина (local_a), данные с offset 3
если бит7=0:  байт1 = длина (local_a),  данные с offset 2 ; offset(local_9)=0
```
**[декомп]** (`FUN_0043c520`: `bVar1=buf[0]; comp=(bVar1>>4)&7; ext=bVar1&0x80; relOID=bVar1&0x0f`; проверка валидности `relOID==1 && comp!=0` иначе «Input data is corrupt»; длины `local_9/local_a`).

Поток объектов читается подряд до конца user memory; ошибки — `"Input data is missing or corrupt"` / `"Input data is corrupt"`.

## 3. Коды компакции (подтверждены кодом)

| Код (биты 6-4) | Метод | Функция | Что делает |
|---|---|---|---|
| **1** | Integer | `FUN_0043c478` | копирует октеты payload **в обратном порядке** (целое big↔little) → числовой первичный идентификатор **[декомп]** |
| **2** | Hex-string | `FUN_0043c350` | каждый байт → 2 hex-символа по таблице `"0123456789ABCDEF"` (`DAT_004afe6c`) **[декомп]** |
| **3** | 5-бит текст | inline-цикл в `FUN_0043c520` | распаковка по 5 бит, **+0x40** к коду → подмножество ISO 646 (заглавные/цифры) **[декомп]** |
| **4** | 6-бит текст | inline-цикл | по 6 бит, **+0x40** **[декомп]** |
| **5** | 7-бит текст | inline-цикл | по 7 бит, без смещения → полный ISO 646 IRV **[декомп]** |
| **6 / 7** | Octet / integer string | `FUN_0043c294` | копирует октеты payload как есть (строка) **[декомп]** |

Это **в точности таблица компакции ISO 28560-2** (1=integer, 2=…, 3=5-bit, 4=6-bit, 5=7-bit, 6=octet). Смещение `+0x40` для 5/6-бит и таблица hex — стандартные. **[стандарт]**

## 4. Логическая модель (энкодер `FUN_0043bb70`/`FUN_0043be54`)

Энкодер собирает из объектов промежуточную строку формата:
```
%s=%d;%d;%d;%s;%s;%s,
```
(литерал в `FUN_0043bb70`:1879, `FUN_0043be54`:2044/2072). **[декомп]** Поля (порядок IDlogic, сопоставление с ISO 28560 — **[инференс]** по контексту и `TRFIDDanish`):
1. `%s` — **первичный идентификатор экземпляра** (штрихкод/инв. номер) — Primary Item Identifier (relative-OID 1).
2. `%d` — content parameter / тип (часть кода объекта).
3. `%d` — **«из скольких частей»** (Number of parts in item).
4. `%d` — **«номер части»** (Ordinal part number) — комплект/том.
5. `%s` — **владелец / библиотека (ISIL)** — Owner library (ISIL).
6. `%s` — **set information / тип использования** (Set information / Type of usage).
7. `%s` — доп. строка (терминатор/служебное).

Класс модели — `TRFIDDanish` / unit `IDLogic_Danish` (из RTTI в дампе строк). Соответствует профилю **«Danish Data Model» (DS/INF 163) поверх ISO 28560-2**. **[стандарт/инференс]**

## 5. EAS / AFI (противокражка) — командный уровень

- **EAS-бит** (анти-вор) — команды NXP ICODE SLIX: `NXPICODESLIX_EableEAS` / `NXPICODESLIX_DisableEAS` / `NXPICODESLIX_EASCheck` (ISO 15693 custom, код производителя NXP 0x04). FEIG-путь: `CLSCRF_SetEAS_15693` / `CLSCRF_ResetEAS_15693` / `CLSCRF_EASAlarm_15693`; RH-путь: `RH_Reader_ISO15693_AlarmEAS`. **[декомп]**
- **AFI** (Application Family Identifier) — `ISO15693_WriteAFI` / `CLSCRF_WriteAFI_15693` (команда ISO 15693 `0x27`). **[декомп/стандарт]**
- Значения **AFI ON/OFF и пароль EAS — конфигурируемые** (поля `edAFION`/`edAFIOFF`/`edEASPass` в UniDllConfigurator, дефолт `00000000`), не зашиты в DLL — подтверждает закрытие [OPEN_QUESTIONS.md](OPEN_QUESTIONS.md) §9 (значения берутся из конфига стенда, реверсить нечего). Экспорт `RfidEASAFIPasswordSet` (vtable +0x38) задаёт пароль. **[декомп]**
- Экспорты-обёртки (диспетчеризация в vtable считывателя): `RfidEASGet`(+0x18), `RfidEASSet`(+0x1c), `RfidDM15Init`(+0x20), `RfidDM15Write`(+0x24), `RfidDM15Read`(+0x28), `RfidDM15RawRead`(+0x2c), `RfidLockSet`(+0x30), `RfidDM15Erase`(+0x34), `RfidEASAFIPasswordSet`(+0x38). **[декомп]**
- Тест-режим: `RfidEASGet` при флаге `DAT_004b314c` читает EAS из файла `EasyBook_rfid.txt` (строки `key=value;`) — отладочная заглушка, в проде не используется. **[декомп]**

## 6. Что это значит для Biblio

- **Совместимость с промаркированным фондом достигается стандартным кодеком ISO 28560-2** — не нужен проприетарный реверс. Reader Agent должен:
  1. читать/писать блоки ISO 15693 (ICODE SLI/SLIX, блок 4 байта);
  2. (де)кодировать поток объектов ISO 28560-2 с компакцией 1/2/3/4/5/6 (см. §3);
  3. маппить элементы: Primary Item Identifier ↔ инв/штрихкод (= ключ выдачи `910^B`/поле 40^H в ИРБИС), Owner library (ISIL), Set information (часть/комплект).
- **EAS/AFI:** реализовать NXP ICODE SLIX EnableEAS/DisableEAS + WriteAFI; значения AFI ON/OFF и пароль EAS взять из конфигурации (как у IDlogic), а не зашивать.
- **Проверка:** §4 (сопоставление полей строки `%s=%d;%d;%d;%s;%s;%s` ↔ конкретные relative-OID ISO 28560) частично инференс — рекомендуется сверить на **дампе реальной метки** со стенда (одно чтение `RfidDM15RawRead` даёт сырые байты + разобранную строку).

## 7. Достоверность и остаток
- **[декомп] подтверждено:** блочный ISO 15693, структура precursor-байта, все 6 методов компакции, формат промежуточной строки, EAS/AFI команды и vtable-офсеты.
- **[инференс] осталось:** точное соответствие конкретного relative-OID ↔ конкретное поле (штрихкод/owner/set) — закрывается одним дампом реальной метки. На объём работ Biblio не влияет (кодек стандартный).
- Аппаратные блокеры [OPEN_QUESTIONS.md](OPEN_QUESTIONS.md) §1–3 (привод ячеек/механика станций/счётчик) декомпиляцией этого DLL **не закрываются** — их нет в этом бинаре, нужен доступ к прошивке станций.
