## 2. Роли типов файлов (подтверждено по содержимому)

| Тип | Роль |
| --- | --- |
| `*.PAR` (в корне DATAI, по базе) | Пути базы: п.1–10 — каталог данных; **п.11 — папка полных текстов** (напр. `IBIS.PAR`: `11=.\DATAI\ibis\texts\`) |
| `*.FST` | Таблица инвертирования = **генератор словаря/префиксов** (поле→термин с префиксом) |
| `*.INI` секция `[SEARCH]` | **Карта «вид поиска (подпись) → префикс»** для форм поиска |
| `*.PFT` | Форматы отображения (вывод записи, ссылки, иконки); цепочки `@формат` |
| `*.MNU` | Меню/справочники (значения комбобоксов, виды поиска, списки баз) |
| `*.WSS` | Рабочие листы (структура форм ввода Каталогизатора) |
| `*.TAB`/`*.OPT` | Табличные формы / опции отображения форматов |

## 3. Префиксы — как устроены (закрывает #1/#2)

### 3.1. Секция `[SEARCH]` = подпись → префикс (авторитетно)
В INI базы: `ItemNumb=N`, затем по каждому виду поиска набор ключей. Пример GUAR (ItemNumb=24):
| Подпись (`ItemNameK`) | Префикс (`ItemPrefK`) | Доп. |
| --- | --- | --- |
| Ключевые слова | `K=` | Logic=4, ModByDic=!DMODK |
| Персона | `A=` | F8For=!F8avt, **Adv=ATHRA,A=** (авторитетный файл) |
| Заглавие/Название | `T=` | — |
| … (всего 24) | … | ItemMenu/ItemHint/ItemTranc/ItemAdv и т.д. |

Прочие ключи элемента: `ItemDictionType` (тип словаря), `ItemLogic` (логика по умолч.),
`ItemMenu` (справочник значений), `ItemModByDic` (модификация по словарю), `ItemTranc` (усечение),
`ItemHint` (подсказка), `ItemAdv` (авторитетный файл для расширения), `ItemIdent/IdentUp`.
> **Вывод:** для любой базы веб-форма поиска строится прямо из `[SEARCH]`: подписи, префиксы, словари,
> авторитетные файлы, логика, усечение. Для PLAY/TUAR — читать их `[SEARCH]` на сервере СПб ГТБ.

### 3.2. FST = генератор терминов словаря (механика префикса)
Формат строки: `<метка> <метод> <формат>`. Формат печатает термин с префиксом. Примеры из `ibis.fst`:
- `K=` (ключевые): десятки строк `1200 8 MHL,'/K=/'(v200^a…)` — из заглавий, серий, рубрик 600/601, 922/923/925 и т.д.
- `T=` (заглавие): `200 0 …'T=',&unifor("9"v200^a…)`.
- `KT=` (полнотекст): `1112 5 …'/KT=/'&uf('++…поля…')` — по всем элементам записи.
- Электронная версия: `955 0 …'TXT=',v955^a` и `'V=FT'` — маркеры наличия файла/полного текста.
> Методы: `0` обычный, `8` — со ссылками подполей (`%`), `5`/`9` — спец. Префикс зашит в формат строки.

## 4. GUAR Фонд↔Опись — подтверждено: поле 488

Из `Guar.fst` и `rmarce_gua.fst`:
- **Связь — через поле 488:** `^t` — идентификатор цели связи; `^1` — уровень цели
  (фонд/опись/дело/документ); `^c` — название; `^j`/`^g` — части шифра.
- Собственный уровень записи — в **`v920`** (`fond`/`opis`/`delo`/`guar`/`list`).
- Индекс межзаписевых связей: `488 0 …'II',<уровень>,'='v488^t` → строит `II=` для перехода.
- Поля 461/462/463 формируют **ссылки вверх по иерархии** (документ→дело→опись→фонд), используя 488.
- Шифр: 852 (`NF=` для фонда, `Nehr=` для документа), схема систематизации описи — поле 545/327.
> Это конкретно подтверждает проектное «Фонд↔Опись через 488»: двухуровневый переход — частный
> случай общей 488-иерархии. Точные PFT-ссылки просмотра — в `*.pft` GUAR (есть в DATAI/GUAR).

## 5. Серверные конфиги (`IRBIS64.rar`)

`irbis_server.ini` — секции `[MAIN]`, `[REDIRECT]`, `[FullText]`. Ключи:
- `DataPath=C:\IRBIS64\DATAI\`, `workdir=C:\IRBIS64\workdir`, `IP_PORT=6666`, `WebIrbisPort=…`.
- **Реестры баз:** `DBNNAMECAT=dbnam1.mnu` (список баз Каталогизатора), `DBNNAMECATR=dbnam3.mnu`
  (список баз, видимых Читателю). `CLIENTLIST=client_m.mnu` — **список клиентов = логины/пароли/права
  АРМ** (⚠️ креды; в боевой репо не кладём).
- АРМ-INI: `Irbisr.ini` (Читатель) — секции `[Main] [SEARCH] [Rubricator] [Display] [Reader]
  [Request] [NAVIGATOR]`; аналогично IRBISA/B/c/k/m/P. `Irbisr.ini [SEARCH]` = эталон форм поиска веб-Читателя.
- `irbis64_client.dll` — бинарь клиента (реализация протокола из находки #2; бинарь не разбираем).
- Справочники: `unifor.mnu`, `client_ini.mnu`, `GenPFT64.ini`.

## 6. Что закрыто и что осталось

**Закрыто механически (демо даёт «как»):**
- #1/#2 префиксы — `[SEARCH]` (подпись→префикс) + FST (генерация). Для боевых PLAY/TUAR — те же файлы на сервере СПб ГТБ.
- GUAR Фонд↔Опись — поле **488** (`^t` цель, `^1` уровень) + `v920` уровень записи. Подтверждено.
- Полные тексты — путь из `*.PAR` п.11; маркеры `V=FT`/`TXT=` в FST; база `RIGHT` присутствует (#6).
- Реестр баз и видимость — `dbnam1.mnu`/`dbnam3.mnu`; права/логины — `client_m.mnu`.

**Осталось:**
1. Точные **PFT-цепочки и логика областей** для конкретных баз (GUAR-ссылки просмотра, IMAGE имидж-карточки) — разобрать `*.pft` выбранных баз DATAI как образцы.
2. **Рабочие листы (WSS)** Каталогизатора по образцу IBIS — структура форм ввода.
3. Боевые конфиги **PLAY/TUAR/EK/ESKIZ/HPO** — только с сервера СПб ГТБ (в демо их нет).
4. Документные батчи: `irbiscat.doc` разд.4 (правила полей), `IrbisCom.doc`, `irbisKO64.doc`, `irbisa.chm`, `irbis64.chm` точечно, `RELEASE_OVERALL.doc`.
