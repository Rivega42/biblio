## 1. GUAR — Гуманитарный архив (профильная база СПб ГТБ)

**Иерархия объектов (5 уровней), рабочие листы (РЛ):**
`Фонд (РЛ Fond) → Опись (РЛ Opis) → Дело (РЛ Delo) → Документ/ед.хр. (РЛ Guar) → Лист (РЛ List,
аналитическое описание)`. Допускается «перепрыгивание» уровней (Документ может быть создан прямо из
Фонда/Описи).

**Связи (ключевое для «Фонд↔Опись»):**
- Связи между записями **устанавливаются автоматически** при вводе «сверху вниз». Каждая нижняя
  запись хранит ссылки на **все вышестоящие** уровни; в просмотре показываются отсылки **вверх и
  вниз** — «путешествие по фонду» с прямыми переходами на связанные записи.
- В вебе вендора (АРМ Читатель) это видно как ссылки: «Просмотреть/Перейти к описям фонда»,
  «Фонд N» (возврат), «Просмотреть дела/документы описи», «введено в ЭК», «Аннотация полностью»,
  «подробнее», «Схема систематизации описи (в новом окне)».
> Для `irbis-web`: двухуровневый переход Фонд↔Опись — частный случай этой общей иерархии связей.
> Точное поле связи (проект называл 488) — подтвердить по `guar_PFT.doc`/`.pft` (форматы GUAR;
> в этом архиве нет, лежат в `DATAI\GUAR`).

**Поля, отмеченные в описании:** 327 (Схема систематизации описи, многоуровневая), 910 (экземпляры —
на уровнях Опись/Дело/Документ, могут выдаваться), 852 (шифр/номер; `???` если номер не задан),
392 (примечание об особенностях листов).

**Поисковый аппарат GUAR:** отдельные БД-рубрикаторы **GUSK** (Схема классификации архива) и
**GURF** (Рубрикатор фондов) — по структуре как HELP (ГРНТИ), иерархические. Словари: Вид/тип
документа, персоны, названия, крайние даты, тематическая классификация, географические рубрики,
схема классификации, ключевые слова.

## 2. Web-ИРБИС64+ — вендорская веб-надстройка (модель веб-API)

> Это эталон того, **как сам вендор делал веб поверх протокола**. Прямо релевантно нашему веб-клиенту.

**Механика:** CGI-шлюз + форм-файлы `*.frm`; настройка в `irbis_server_ft.ini`; на каждую команду —
своя последовательность фреймов, собирающих HTML. Форматы/справочники — в `Deposit_plus` (`*.pft`,`*.mnu`).

**Основные команды (параметр `C21COM=`):**
| Код | Команда | Параметр INI (список фреймов) |
| --- | --- | --- |
| `F` | Показ набора фреймов | ShowFrames |
| `S` | Поиск | SearchFrames |
| `T` | Показ терминов словаря | DictionryFrames |
| `Z` | Заказ | ZakazFrames |
| `R` | Запись в БД | RecUpdateFrames |

**Параметры запроса (выявлены из боевого CGI-запроса):**
- `C21COM` — команда; `I21DBN` — секция INI (профиль); `P21DBN` — имя БД; `Z21ID`/`Z21FAMILY` —
  логин/пароль читателя; `S21FMT` — формат вывода (напр. `briefHTML_ft`); `S21REF` — размер страницы;
  `S21STN` — старт; `S21CNR` — кол-во.
- Поисковое поле = тройка: `<x>_S21STR` (значение) + `<x>_S21P03` (**префикс словаря**) +
  `<x>_S21LOG` (логика И/ИЛИ/НЕ) + `<x>_S21P01/P02` (тип/усечение).

**Подтверждённые префиксы словаря (из параметров формы):**
| Поле формы | Префикс | Значение |
| --- | --- | --- |
| AVT | `A=` | автор |
| NAME | `T=` | заглавие |
| DAT | `G=` | год |
| A34 | `V=` | вид документа |
| VAR/COLLT | `COLLT=` | коллекция |
| FT | `K=` / `KT=` | ключевые слова / полнотекст |
| DP | `DP=` | (доп. точка доступа — уточнить) |
| A1 | `R=` | (уточнить) |
> Частично закрывает открытые вопросы проекта #1/#2 (префиксы); финал — сверить с секцией SEARCH INI базы.

**ЛК/формуляр читателя (`author_3.frm`):** содержит Историю книговыдач (две: «бумажные» выдачи и
«электронные» = обращения к полным текстам; интервал засчитывания — `EbookAccessInterval`, по умолч.
24 ч; лимит элементов `MAXHISTORYCOUNT`=10; обратная хронология), **Закладки** на страницах полных
текстов, **Корзину заказов** «бумажной» книговыдачи. Данные истории формирует `show_history.pft`.

**Заказ:** подтверждение через `order_form.frm` (из `order.frm`); библиоописание — формат `zakaz.pft`;
параметры: MFN заказываемого, примечания к заказу, MFN для удаления из корзины. Корзина использует `C21COM=S`.

**Гость:** ссылка «ВОЙТИ КАК ГОСТЬ» работает при наличии в RDR условного читателя с полями
**30 (идентификатор)** и **50 (категория)** = `GUEST`.
> Для `irbis-web` это вариант; наш проект требует, чтобы у Гостя НЕ было ЛК/корзины (замечание J-ИРБИС) —
> используем механику GUEST только для гостевого поиска, без персональной корзины.

**Просмотр полного текста:** всплывающее окно «Постраничный просмотр полного текста» (по ссылке или
по клику на фрагмент первой релевантной страницы).

**Загрузка ВКР (пример write через веб):** `reg.frm`, обязательные поля помечены `*`, команда
`R` (RecUpdateFrames) + передача PDF: `<input type="file" name="PDFTEXT" accept="application/pdf">`.

## 3. АРМ «ЧИТАТЕЛЬ» — структура интерфейса (из общего описания)

Это карта экранов веб-Читателя (наш каталог), как у вендора:

**Рабочие области (4):** СЛОВАРЬ · ТЕКУЩИЙ ЗАПРОС · КОРОТКИЕ ОПИСАНИЯ · ПОЛНОЕ ОПИСАНИЕ.
(СЛОВАРЬ имеет вариант для поиска по Тематическому рубрикатору.)

**Меню БАЗА ДАННЫХ:** Открыть · Регистрация · Выход.

**Меню ПОИСК:** Вид основного поиска · **«Для чайников»** (DS=, упрощённый) · **«Для умников»**
(экспертный) · **Комплексный поиск** · **Последовательный поиск** · **Поиск по связи** · Выполненные
запросы · Постоянные запросы.
> Подтверждает наименования режимов (открытый вопрос #1): у вендора это «Для чайников»/«Для умников»/
> «Комплексный»/«Последовательный»/«По связи». Наш проект переименовывает (см. FINAL_TZ §6/§16).

**Меню ПРОСМОТР:** Текущий результат · Формат просмотра · Внешний объект · Заказ · Сортировка ·
Переход · Переход по отмеченным · Снять все отметки · Связанные документы.

**Меню СЕРВИС:** Печать · Экспорт · Заказ по ретрофонду · Рекомендованная литература · Настройка
интерфейса. **Меню ПОМОЩЬ.**

## 4. АРМ «КАТАЛОГИЗАТОР» — структура и рабочие листы

**Рабочие области (2):** СЛОВАРЬ · РАБОЧИЙ ЛИСТ.

**Способы ввода в поле:** меню-справочник · словарь · вложенный РЛ · многостроковое окно ·
Рубрикатор ГРНТИ · Авторитетные файлы/АПУ к УДК-ББК/Тезаурус · групповой ввод повторяющихся ·
оперативные меню-справочники · ИРБИС-Навигатор.

**Типология рабочих листов (РЛ) ввода** (из `irbiscat.doc`): монографическое описание книги,
сводное описание (спецификация тома), аналитические описания, описания/корректура журналов,
корректура номеров журналов, упрощённый ввод книг, описание музейного предмета.

**Страницы РЛ (вкладки формы ввода)** — это структура форм веб-Каталогизатора:
Дублетность · Коды (ISBN/ISSN, кодир. информация) · Основное БО · Расширенное БО · Специфические ·
**Экземпляры** · Технология · Систематизация · Содержание (оглавление сборника) · КО
(книгообеспеченность) · Редкие · Краеведение.

**Порядок работы:** ввод однотомных/многотомных, корректура, ввод заимствованием, ввод через АРМ
Книговыдача, обработка периодики (новые издания, регистрация поступлений, корректура номера).

> Это и есть «формы и поля» веб-Каталогизатора — переносятся постранично 1:1 по составу вкладок/полей.
> Детальные правила полей (раздел 4 `irbiscat.doc`) разберу отдельным проходом.
