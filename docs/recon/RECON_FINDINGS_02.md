## 1. ⚠️ Безопасность — сразу

INI-файлы клиентов АРМ (`CIrbisa.ini`, `Cirbisc.ini`, `cirbisb.ini`, `cirbisk.INI`, `cirbisP.ini`,
`Irbis_Navigator.ini`) **содержат боевые креды**: ключи `UserName` / `UserPassWord` / `USERPASSWORD`,
а также `ServerIP`, `ServerPort`, `DBN`, `WebServer`.

- **В репозиторий эти INI не класть.** Это ровно класс прошлой утечки.
- Значения здесь не приводятся. В веб-клиент берём только **схему ключей** (см. §5), значения — в env.
- При шаринге конфигов — сначала вычистка `*PassWord*`/`*Password*`.

## 4. Карта протокола — каталог функций клиентской DLL

> Это **авторитетная карта нативного протокола** (официальная дока). Снимает необходимость угадывать
> опкоды: видно, что сервер умеет и с какими параметрами. Полный текст — в `irbis64_client_dll_TEXT.txt`.

**2. Общие / сессия:** `IC_reg` (регистрация: host, АРМ-тип, user) · `IC_unreg` · `IC_set_client_time_live`
(интервал подтверждения) · `IC_set_show_waiting` · `IC_set_webserver` / `IC_set_webcgi` (работа через
web-шлюз) · `IC_set_blocksocket` · `IC_isbusy` · `IC_nooperation` (keepalive).

**3. Ресурсы (файлы на сервере):** `IC_update_ini` (профиль на сервере) · `IC_getresourse` (текстовый
ресурс/файл) · `IC_clearresourse` (очистка кэша) · `IC_getresoursegroup` (групповое чтение) ·
`IC_getbinaryresourse` (**двоичный ресурс — чтение файлов 951/955**) · `IC_putresourse` (**запись
текстового** ресурса — двоичные не пишутся, согласуется с ограничением «сервер не пишет бинарь»).

**4. Мастер-файл (записи):** `IC_read` (чтение по MFN с блокировкой) · `IC_readformat` (**чтение +
расформатирование @форматом** — ключ к серверному PFT) · `IC_update` / `IC_updategroup` /
`IC_updategroup_sinhronize` (запись/обновление, в т.ч. связанная — для Фонд↔Опись) · `IC_runlock`
(разблокировать) · `IC_ifupdate` (актуализировать) · `IC_maxmfn`.

**5. Работа с записью (в памяти клиента):** `IC_fieldn`/`IC_field` (поле/подполе) · `IC_fldadd`/
`IC_fldrep` (добавить/заменить поле) · `IC_nfields`/`IC_nocc`/`IC_fldtag` · `IC_fldempty` ·
`IC_changemfn` · `IC_recdel`/`IC_recundel` (логич. удаление/восстановление) · `IC_recunlock` ·
`IC_getmfn` · `IC_recdummy` (пустая запись) · `IC_isActualized`/`IC_isLocked`/`IC_isDeleted`.

**6. Словарь (для поиска/автодополнения/умного поиска):** `IC_nexttrm`/`IC_prevtrm` (термины вперёд/
назад от заданного) · `IC_nexttrmgroup`/`IC_prevtrmgroup` (+ расформатирование) · `IC_posting`
(ссылки термина) · `IC_postinggroup` (первые ссылки списка) · `IC_postingformat` · `IC_getposting`.

**7. Поиск:** `IC_search` (прямой по словарю по поисковому выражению + формат + пагинация first/numb) ·
`IC_searchscan` (последовательный по результату/диапазону, с min/max и доп. выражением).

**8. Форматирование:** `IC_sformat` (по MFN) · `IC_record_sformat` (запись в клиентском представлении) ·
`IC_sformatgroup` (группа). Это и есть серверный PFT для веб-вывода.

**9. Пакетная обработка:** `IC_print` (**табличная форма** — Word/RTF-подобные таблицы) · `IC_stat`
(статистические распределения) · `IC_gbl` (**глобальная корректировка** — пакетные правки записей).

**10. Администрирование:** `IC_adm_restartserver` · `IC_adm_getdeletedlist`/`getalldeletedlists` ·
`IC_adm_dbempty`/`dbdelete`/`newdb` (опустошить/удалить/создать БД) · `IC_adm_dbunlock`/`dbunlockmfn` ·
`IC_adm_dbstartcreatedictionry`/`dbstartreorgdictionry`/`dbstartreorgmaster` (словарь/реорг) ·
`IC_adm_getclientlist`/`getclientslist`/`getprocesslist`/`setclientslist` (клиенты/процессы/доступ).

**11. Вспомогательные:** `IC_reset_delim`/`IC_delim_reset` (работа с разделителями полей).

## 5. Что это даёт веб-АРМам (привязка API → модули)

- **Ядро (один раз):** сессия (`IC_reg/unreg/nooperation`) + поиск (`IC_search/searchscan`) +
  словарь (`IC_nexttrm/posting…`) + чтение/формат (`IC_read/readformat/sformat…`) + ресурсы/файлы
  (`IC_getresourse/getbinaryresourse`).
- **Веб-Читатель (каталог СПб ГТБ):** ядро (read) + заказы. Конфиг подключения — по схеме INI
  (`ServerIP/ServerPort/UserName/UserPassWord/DBN`), значения из env.
- **Веб-Каталогизатор:** ядро + запись (`IC_update/updategroup`, работа с полями §5, блокировки,
  `IC_gbl` глобальная корректировка). Загрузка бинарных файлов — НЕ через протокол (нужен файловый доступ).
- **Веб-Книговыдача:** заказы/выдача поверх RQST/RDR (модель из находок #1) + статусы записей.
- **Веб-Комплектатор / Книгообеспеченность:** `IC_print`/`IC_stat` (табличные/стат. формы) + запись.
- **Веб-Администратор:** функции `IC_adm_*` — отдельный модуль с самым строгим доступом (контур, роли, аудит).
