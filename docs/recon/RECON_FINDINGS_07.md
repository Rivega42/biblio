# Recon-находки #7 — RELEASE_OVERALL (релиз-ноуты по версиям)

> Источник: `RELEASE_OVERALL.doc` (накопительные изменения). Дата 20.06.2026.

## Суть
Документ — кумулятивные релиз-ноуты версий **2012.1 → 2021.1** (на 2023-сервере отдельной секции
нет; верх — 2021.1). Содержание преимущественно **инкрементально-эксплуатационное**, без новой
архитектурной поверхности сверх уже снятой в #1–#6:
- Много правок по **Книгообеспеченности (VUZ)**: пакетные задания (RepItem/RepMark, замена кафедры/
  уровня компонента, изменение по доп. БД каталога через MNU-список), отчёты, контингенты.
- **Комплектатор/КСУ**: формы движения фонда, инвентарные списки.
- **Администратор**: режимы актуализации/словаря; «Создать словарь заново — полностью только ЭК»;
  многопроцессорная пересборка словаря (`IrbisMultiLoad.exe`, с 2012.1).
- Точечные форматные правки: распаковка повторяющегося подполя издателя в **488**, изменения
  `book_h.pft`, обработка кода языка в 922/330.

## Ключевые вехи версий
- **2018.1** — включён весь функционал ИРБИС64+; исполняемые модули **несовместимы** с предыдущими;
  введена авторизация клиента **по учётной записи Windows** (см. серверный Администратор).
- **2012.1** — многопроцессорная пересборка словаря; новый механизм актуализации.

## Вывод для проекта
Подтверждает версионный контекст (наш сервер — линейка ИРБИС64+ 2023, наследует 2018.1+). На карту
возможностей не влияет: всё существенное уже отражено. Точечные форматные правки (488, book_h.pft)
учитывать при разборе конкретных PFT боевых баз.
