# Справочник АВТОРИТЕТНЫХ/НОРМАТИВНЫХ файлов САБ ИРБИС64 (reconstruction-grade)

> Фаза 2. Источник — реальные конфиги БД (CP1251 → UTF-8) в `C:\IRBIS64\_recon\_work\cfg_txt\<DB>\`.
> Все факты — со ссылкой на файл. ПДн опущены. Неясное помечено `TODO(recon #ATHR-NN)`.
>
> Охват: **ATHRA** (индивидуальные авторы), **ATHRC** (коллективы/мероприятия/издательства), **ATHRS** (предметные заголовки), **ATHRG** (географические названия), **ATHRU** (АПУ к УДК), **ATHRB** (АПУ к ББК).

---

## 0. Обзор и общая архитектура

Шесть авторитетных/нормативных БД RUSMARC-Authorities-подобной структуры. Все построены на единой модели: **поле 210 = принятый (унифицированный) заголовок**, **410 = ссылки «см.» (вариантные/непринятые формы)**, **510 = ссылки «см. также» (связанные принятые формы)**, **710 = параллельные формы на других языках/графиках**. ATHRU/ATHRB — упрощённые БД АПУ (только 210/510/300).

| БД | Назначение | Заголовок (поле 210) | Поля-ссылки | Полей в `.WS` |
|----|-----------|----------------------|-------------|---------------|
| **ATHRA** | Индивидуальные авторы (имена лиц) | Имя лица (ИЛ) | 410, 510, 710 | 23 (полн. `DEFAULT.WS` — 29) |
| **ATHRC** | Коллективы / мероприятия / издательства | Наименование организации (НО) | 410, 510, 710 | 21 |
| **ATHRS** | Предметные заголовки/подрубрики | Предметная рубрика | 410, 500, 510, 515, 520, 550 | 22 |
| **ATHRG** | Географические названия | Географ. наименование | 410, 510 | 18 |
| **ATHRU** | АПУ к УДК | Заголовок рубрики АПУ + код | 510 | 3 |
| **ATHRB** | АПУ к ББК | Заголовок рубрики АПУ + код | 510 | 4 |

Источники: `athra.WS`, `ATHRC.WS`, `ATHRS.WS`, `athrg.WS`, `ATHRU.WS`, `ATHRB.WS` (строка 3 каждого = число полей).

Общая раскладка блоков (RUSMARC-Authorities):
- блок 1 — управляющие (152 — правила каталогизации/предметизации);
- блок 2 — принятый заголовок (2xx: 210, плюс 215 геогр., 230 загл. — в этих БД 210);
- блок 3 — примечания (300, 305, 310, 320, 330);
- блок 4 — ссылки «см.» (4xx: 410);
- блок 5 — ссылки «см. также» (5xx: 500, 510, 515, 520, 550, 710);
- блок 6 — классификация (675 УДК, 689 ББК — в ATHRS);
- блок 8 — источники/история (801, 810, 815, 820, 825, 830, 835, 836);
- блок 9 — локальные технологические/идентификаторы (900, 905, 907, 910, 920, 921–924, 951, 953, 10=ORCID).

## 5. Модель авторитетного контроля

### 5.1 Структура отсылок (унифицированный/вариантный/связанный заголовок)

Единая для всех БД 4-блочная модель:

| Тип формы | Поле | Семантика | Самост. запись? |
|-----------|------|-----------|------------------|
| **Унифицированный (принятый)** заголовок | 210 | Единственная нормированная форма имени/наименования/рубрики | — (это сама запись) |
| **Вариантная (непринятая)** форма «см.» | 410 | От неё ведёт ссылка «см.» → к 210. | **НЕТ** (по `.WS`: «самостоятельные авторитетные записи НЕ создавать») |
| **Связанная принятая** форма «см. также» | 510 (+ 500/515/520/550 в ATHRS) | Ссылка «см. также» от 210 → к другой принятой записи | **ДА** (требуется отдельная авторитетная запись) |
| **Параллельная** форма (др. язык/графика) | 710 | Принятый заголовок на другом языке/в иной графике | **ДА** |

Тип ссылки «см. также» уточняется подполем `^5` (`kod.mnu`):
- ATHRA `kod.mnu`: `a` прежнее имя, `b` последующее, `e` псевдоним, `f` подлинное имя, `z` другое; с суффиксом `0` — то же с **блокировкой вывода ссылки на экран** (`a0/b0/e0/f0/z0`). FST 410 явно исключает `f0/a0/b0/e0/z0` из индексации.
- ATHRC/ATHRG: `a` прежнее наим., `b` последующее, `d` сокращённое, `z` другое.
- ATHRS: `g` более широкое понятие, `h` более узкое понятие (иерархия предметных рубрик).

### 5.2 Связь с библиографической БД IBIS — подполе `^3`

**Подполе `^3` = номер (шифр) авторитетной записи** — ключ связи:
- В авторитетных БД `^3` присутствует в полях-ссылках **510 и 710** (ATHRA: «Шифр связанной записи»; ATHRC: «Номер заимствованной записи из другой БД»; ATHRS 510/510k: то же). Связывает принятую запись со связанной/заимствованной авторитетной записью.
- Поле **903** («Шифр документа в БД») хранит собственный шифр записи; формируется из `v210^a.1 + &unifor('R9')` (см. `autoin.gbl`).
- В библиографической БД IBIS подполе `^3` соответствующего поля (700/701/710/606…) хранит **номер авторитетной записи** из ATHRx — это и есть точка соединения каталога с авторитетным файлом.

### 5.3 Автоматическое формирование связанных записей — `autoin.gbl`

`ATHRA/autoin.gbl` (global-correction, выполняется при сохранении) реализует автоматику авторитетного контроля:
1. При наличии 510 или 710 **без** `^3` и при разрешающем `905^7` — собирает все 510/710 во временное поле 1002 (`REPEAT`).
2. Для каждого — через `&unifor('JATHRA,…')` проверяет наличие записи в ATHRA по алкоду; если **нет** — `NEWMFN` создаёт новую авторитетную запись:
   - поле **210** (переносит ^A/^B/^G/^1/^C/^D/^8/^9/^F/^</^3/^:);
   - поле **903** = `^3` (шифр-связь);
   - поле **801** (страна/организация/дата из `org.mnu` + `&unifor('3')`);
   - поле **920** = `ATHRA`.
3. Также `autoin.gbl` досоздаёт поле **907** (каталогизатор/дата), **801**, и нормализует `^<` (число слов ФИО) в 210/410/510/710 через формат `aaa`.

> Это означает: связанная форма «см. также» (510/710) автоматически материализуется в самостоятельную авторитетную запись — реализация правила из `.WS` «требуются самостоятельные записи». В ATHRS поле 510 формируется автоматически из 500/520/550/515 (см. `ATHRS.WS`, комментарий поля 510).

## 6. PFT — форматы показа

Основной формат показа полной авторитетной записи (RTF, `{…}`):

| БД | Формат | Заголовок вывода | Ключевые ветви |
|----|--------|------------------|----------------|
| ATHRA | `athra.pft` | «Авторитетная запись на ИНДИВИДУАЛЬНОГО АВТОРА» | принятая форма (210) → «см.также на параллельном языке» (710) → «см.также под псевдонимом/подлинным/предыдущим/последующим именем» (510 по `^5`) → «Непринятые формы имени лица» (410) → примечания (300/330/340/815/820/825/835/836/830/810) → место работы (910) → ORCID (10, с гиперссылкой `http://orcid.org/`). Ветки REF (310) и EXP (320). |
| ATHRC | `athrc.pft` | «Авторитетная запись на КОЛЛЕКТИВ/МЕРОПРИЯТИЕ/ИЗДАТЕЛЬСТВО» | 210 (с городами/датами/номером) → 710 параллельные → 510 «см.также под прежним/последующим/сокращённым наим.» → 410 «см. (непринятая форма)» → примечание 300 → 951. |
| ATHRS | `ATHRS.PFT` | «Авторитетная запись на ПРЕДМЕТНЫЕ РУБРИКИ» | 210 → 500/520/550 «см.также» → 510 «см.также под более узким/широким понятием» → 410 «см. под более широким/узким понятием» → ББК (689) / УДК (675) → примечание 300 → 951. Мультиплекс по 920 (ATHRS/REF/EXP). |
| ATHRG | `Athrg.pft` | «Авторитетная запись на ГЕОГРАФИЧЕСКОЕ НАИМЕНОВАНИЕ» | 210 (свёртка ^g) → 710 → 510 «см.также под прежним/последующим/сокращённым наим.» → 410 «см.» → источник 810 → примечание 300 → 951. |
| ATHRU | `ATHRU.PFT` | «АВТОРИТЕТНАЯ ЗАПИСЬ НА РУБРИКИ АПУ» | 210 (`^a -- ^b`) → методические указания (300). |
| ATHRB | `ATHRB.PFT` | «АВТОРИТЕТНАЯ ЗАПИСЬ НА РУБРИКИ АПУ» | 210 (`^a -- ^b`) → методические указания (300). |

Дополнительные форматы (по каждой БД): `BRIEF.PFT` (краткое описание — заголовок одной строкой), `INFOW.PFT`/`infow_html.pft` (инфо для веб), `DBNFLC.PFT` (имя БД), `<DB>GRR.PFT` (групповой), веб-форматы `athr_wn.pft`, `athr_top_wn.pft`, `athr_bottom_wn.pft`, `athr_search_wn.pft`, `athr_select_wn.pft` (Ж-ИРБИС/веб-каталог авторитетных файлов). ATHRA имеет спец-форматы `!AMOV*.PFT` (свёртка/разворот имени для редактора заголовка), ATHRC — `!CMOV*.PFT`, ATHRS — `!PMOV/!SMOV/!500/!510/!515/!520/!550`, ATHRG — `!CMOV/!PMOV/!DMOV/!BMOV/!GMOV/!DMODK/!DMODS`.

> `TODO(recon #ATHR-05)`: формат `!F8AVT/!F8COL/!F8SUB/!F8UDK/!F8BBK` (F8-словари) и `!amov`/`!cmov`/`!pmov`/`!gmov`/`!bmov`/`!dmov` — это форматы свёртки/разворота заголовков, вызываемые редактором при `,ПРЕФИКС=,@!формат`. Полную семантику разобрать отдельно (объёмный код свёртки ФИО/инверсии).
