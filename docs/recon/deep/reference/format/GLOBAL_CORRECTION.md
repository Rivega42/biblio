# Язык глобальной (пакетной) корректировки ИРБИС64 (`.gbl` / `IC_gbl`)

Справочник по языку пакетной корректировки записей (Приложение 7 «Язык пакетной
корректировки» и Приложение 16 «Глобальная корректировка в АРМе Администратор»
руководства `irbis64_2022.doc`) и по реальному каталогу заданий `.gbl` из
БД IBIS, CMPL, RDR, VUZ дистрибутива.

Назначение: единовременная корректировка группы записей на уровне записей,
полей и подполей (`irbis64_2022.doc txt:5894-5895`). В вебе исполняется серверной
функцией `IC_gbl` (раздел 4).

Источники фактов: документация — `irbis64_2022.doc (txt:N)`; реальные задания —
имя `.gbl`-файла; протокол — `protocol/PROTOCOL_REFERENCE.md`; форматные выходы —
`format/PFT_LANGUAGE.md`.

---

## 1. Язык глобальной корректировки

### 1.0 Общая модель исполнения

- Задание — последовательность **операторов**; операторы выполняются в порядке
  следования, и каждый оператор видит запись в состоянии, в которое её привели
  предыдущие операторы (`txt:5920`).
- Корректировка применяется к **фрагменту БД**: диапазон MFN; либо записи по
  запросу; либо запрос + диапазон MFN (`txt:6783-6786`).
- Каждый оператор работает на **текущей записи**; внутри оператора способ
  корректировки задаётся **форматами** (ФОРМАТ 1, ФОРМАТ 2) — это обычные
  PFT-форматы, расформатируемые на текущей записи (`txt:5916-5917`).

### 1.1 Ссылка на поля / подполя / повторения

Поле/подполе адресуется во **второй строке** оператора: `<метка>[<разд><подполе>]`
— число метки, далее (необязательно) разделитель подполя `^` и буква подполя
(`txt:5925`). Примеры из заданий: `910`, `910^D`, `200^F`, `701^!`.

**Повторение поля** (третья строка оператора) (`txt:5926-5931`):

| Значение | Смысл |
|---|---|
| `*` | корректируются ВСЕ повторения |
| `N` (число) | N-е повторение |
| `L` | последнее повторение |
| `L-N` | N-е с конца повторение |
| `F` | **корректировка по формату**: номер строки результата ФОРМАТА 1 ↔ номер повторения поля |

Режим `F` — ключевой механизм: ФОРМАТ 1 выдаёт многострочный результат
(обычно через групповой формат `(...)/`), и i-я строка управляет i-м повторением
поля. Если повторений больше, чем строк, — лишние не трогаются; если строк
больше — лишние игнорируются (`txt:5947-5955`).

**Доступ к данным в форматах**: стандартный PFT — `v910`, `v910^D`, `v910^A`,
условные функции `if…then…else…fi`, `p()/a()` (присутствие/отсутствие),
`s()` (склейка), `rsum()` (сумма по группе), групповой повтор `(...)/`.

### 1.2 Форматные выходы UNIFOR в заданиях

Задания массово используют `&unifor(...)` / синоним `&uf(...)` (см.
`format/PFT_LANGUAGE.md`). Ключевые применения, наблюдаемые в `.gbl`:

- **Глобальные переменные как регистры цикла/буфера** —
  `&uf('+7W…')` запись, `&uf('+7R…')`/`Gnnn` чтение, `&uf('+7U…')` добавление,
  `&uf('+7S…')` сортировка/арифметика, `&uf('+7G…')` исключение неоригинальных
  (`autoin.gbl`: `&uf('+7W1#'…)`, `&uf('Av1330#1')`; `PFT_LANGUAGE.md` строка про `+7`).
- **`&uf('AvNNN^x#k')`** — взять k-е повторение поля NNN (подполя ^x) из
  *модельного* поля как значение (массово в `Move691.gbl`, `SumEkzToSZ.gbl`).
- **Поиск/чтение из другой БД на лету** — `&uf('J<db>,<выражение>')`,
  `&uf('D<db>,…')`, `&uf('7,?…?,формат')` (поиск + форматирование результата):
  `SumEkzToSZ.gbl` строит список MFN и поля через `&uf('7,?BORZ=…?,mfn/')`;
  `Move691.gbl` тянет данные `&uf('DVUZ,?IDD=…?,v3^A')`.
- **Доступ к INI-параметрам сервера** — `&uf('IMAIN,<param>,<default>')`,
  `&uf('IPRIVATE,FIO,')`, `&uf('IPRIVATE,ETR,')` (логин/этап текущего оператора)
  — `autoin.gbl`, `Move691.gbl`.
- **Справочники** — `&uf('K<mnu>|<key>')` (`autoin.gbl`: `Korg.mnu`, `K106t.mnu`).
- **Сервисные** — `&uf('3')` текущая дата; `&uf('+97',…)` нормализация регистра
  для сравнения; `&unifor('Z')` размножение экземпляров (работает ТОЛЬКО в
  глобальной корректировке, `txt:4810`); `&uf('4…')` форматирование предыдущей
  копии записи (для `UNDOR`, `txt:6016`); глобальные счётчики `&uf('++C<idx>#<val>')`
  (БД COUNT, `txt:7087-7097`) — «практически имеет смысл использовать ТОЛЬКО в
  операндах команд глобальной корректировки» (`txt:7097`).

### 1.3 Полный перечень операторов

Документированный набор (первая строка группы — имя оператора): `ADD, REP, CHA,
CHAC, DEL, DELR, UNDEL, CORREC, NEWMFN, END, IF, FI, ALL, EMPTY, REPEAT, UNTIL,
//`, плюс описанные отдельными пунктами `UNDOR`, `PUTLOG` (`txt:5923`, §4.16-4.17).
Операторы `ADD/REP/CHA/CHAC/DEL` описываются **пятью строками** (имя, метка,
повторение, ФОРМАТ 1, ФОРМАТ 2); прочие — своим числом строк (`txt:5923-5935`).
Незадействованные строки заполняются **символом-заполнителем** (в реальных файлах —
`XXXXXXXXXXXXXXXXXXX`) либо пустой строкой (`txt:5934`).

#### Корректировка полей/подполей

**ADD** — добавление нового повторения поля/подполя (`txt:5938-5950`).
- 5 строк: `ADD` / метка[^подполе] / повторение / ФОРМАТ 1 / ФОРМАТ 2(не исп.).
- Метка без подполя: каждая строка ФОРМАТА 1 → новое повторение поля.
- Метка с подполем: 1-я строка ФОРМАТА 1 → подполе в заданное повторение
  (если повторения нет — создаётся новое).
- Повторение `F`: номер строки ФОРМАТА 1 = номер повторения, куда дописывается
  подполе. Дописать в КОНЕЦ поля нельзя — для этого `CHA`.
- Пример: `autoin.gbl` — `ADD 907 / … / if &uf('+6')='0' then '^CDEL'…` (новый 907).

**REP** — замена ЦЕЛИКОМ поля/подполя значением ФОРМАТА 1 (`txt:5951-5957`).
- Повторение ≠ `F`: поле/подполе → 1-я строка ФОРМАТА 1 (остальные игнор.).
- Повторение `F`: i-я строка ФОРМАТА 1 заменяет i-е повторение.
- Пустая строка ФОРМАТА 1 → поле/подполе **удаляется** (пустую строку получать
  оператором пропуска `#`).
- Пример: `Movekp.gbl` — `REP 591 * / if … then v591 else &unifor('DPOST…')`.

**CHA / CHAC** — замена ПОДСТРОКИ внутри поля/подполя (`txt:5958-5971`).
- 5 строк: имя / метка[^подполе] / повторение / ФОРМАТ 1 (искомое A) / ФОРМАТ 2 (замена B).
- `CHAC` — с учётом регистра; `CHA` — без учёта (сравнение в верхнем регистре,
  B пишется как задан).
- A пусто → B дописывается в конец поля/подполя; B пусто → A удаляется.
- Повторение `F`: строки Ai/Bi сопоставляются по номеру повторения.
- Все вхождения A заменяются; все одноимённые подполя корректируются.
- Пример: `autoin.gbl` — `CHA 926^B * / '.' / '. '` (нормализация инициалов).

**DEL** — удаление поля или подполя (`txt:5972-5976`).
- Удаляется заданное повторение поля или подполе (если подполей несколько — все).
- Повторение ≠ `F`: ФОРМАТ 1/2 не используются.
- Повторение `F`: ФОРМАТ 1 даёт по строке на повторение; строка `'1'` → удалить
  соответствующее повторение, иначе оставить.
- Пример: `autoin.gbl` — `DEL 910 F / (if … then '1' else '0' fi/)`.

#### Корректировка записей

**DELR** — логическое удаление поданных на корректировку записей; доп. данных нет
(`txt:5977-5978`). Пример: `ClearCmpl.gbl`, последняя группа `Move...` — финальный `DELR`.

**UNDEL** — восстановление удалённых записей в диапазоне MFN из формы ГЛОБАЛЬНОЙ;
доп. данных нет; последующие операторы выполняются на восстановленных (`txt:5979-5980`).
(В наблюдаемых заданиях дистрибутива не встречается.)

**EMPTY** — очистка (опустошение) текущей записи (`txt:6000-6001`).

**ALL** — дополнить запись ВСЕМИ полями текущей записи; применяется после
`NEWMFN`/`CORREC` (создать новую/чужую запись и наполнить её копией текущей)
(`txt:5998-5999`). Пример: `InsteadLost.gbl`-стиль; в документе — Пример 5
(`NEWMFN 'CMPL' / ALL / REP 920 …`, `txt:6198-6200`).

**UNDOR** — откат к одной из предыдущих копий записи (`txt:6006-6031`).
- 2-я строка (колонка «Параметр1/Поле-подполе») — формат, дающий число шагов:
  `N` (к N-й копии назад, N=1 — предыдущая), `*` (к исходной копии), пусто (нет действий).
- Совместно с `&uf('4…')` (форматирование предыдущей копии) позволяет искать копию
  по условию в цикле `REPEAT…UNTIL` (`txt:6017-6031`).

#### Работа с другой БД / новыми записями

**NEWMFN** — создать новую запись в текущей или другой БД (`txt:5988-5991`).
- 2 строки: `NEWMFN` / формат → имя БД (`'*'` = текущая).
- Далее идут операторы `ADD` (наполнение новой записи; ФОРМАТ 1 в них считается
  на ИСХОДНОЙ записи исходной БД), группа завершается `END`.
- Пример: `CreateSZ.gbl` — `NEWMFN '*' / ADD 62 … / ADD 920 'SZ' / END`.

**NEWREC** *(недокументирован в Прил.7)* — условное создание новой записи: 2-я строка
— формат-условие; при результате `'1'` создаётся новая запись, наполняемая
последующими `ADD`, группа завершается `END`. Аналог `NEWMFN` с встроенным
условием. Пример: `IBIS/autoin.gbl:2255` — `NEWREC / if v920='J' and p(v936) … then '1'…`
(создание записи регистрируемого выпуска журнала). `TODO(recon #GBL-01)` — точная
семантика результата формата (БД назначения vs. флаг) по доке не подтверждена.

**CORREC** — вызвать на корректировку ДРУГИЕ записи (из текущей или иной БД),
отобранные по терминам (`txt:5981-5987`).
- 4 (опц. 5) строки: `CORREC` / формат→имя БД (`'*'`=текущая) / формат→строка,
  передаваемая в корректируемые записи как **модельное поле 1001** / формат→строки-
  термины словаря (с `$` = усечение) / (опц.) число корректируемых записей.
- Группа операторов корректировки чужих записей завершается `END`. Не забывать
  удалять поле 1001 в последнем операторе группы.
- Пример из доки: `txt:6141-6155` (перенос экземпляров 910 в записи по УДК).

**END** — завершение группы `CORREC`/`NEWMFN`/`NEWREC`; возврат к исходной БД и
записи. Одна строка (`txt:5992-5993`).

#### Управление потоком

**IF … FI** — условное выполнение группы операторов (`txt:5994-5997`).
- `IF` = 2 строки: имя + формат; результат `'1'` → выполнять до `FI`, иначе пропустить.
- `FI` = 1 строка. Конструкции вложенные (массово в `autoin.gbl`).

**REPEAT … UNTIL** — цикл по группе операторов (`txt:6002-6005`).
- Группа между ними повторяется, пока формат в `UNTIL` (2-я строка) даёт `'1'`.
- Типовой паттерн: цикл по повторениям через модельное поле + `DEL <model> 1`
  (срезать обработанное повторение), `UNTIL if p(v<model>) then '1' fi`
  (`SumEkzToSZ.gbl`, `Move691.gbl`).
- Внутри форматов также используется выход из группового повтора `break`
  (`autoin.gbl:6030`-стиль).

#### Протокол и комментарии

**PUTLOG** — формирование пользовательского протокола; 2-я строка — формат, дающий
очередную строку протокола (`txt:6032-6033`). Опция интерфейса «В протокол сообщения
задания» оставляет в протоколе только строки PUTLOG. Массово в `Move691.gbl`,
`CHKrdr.gbl`, `MoveDisc.gbl` (349 вхождений по каталогу).

**PUTFLD** *(устарел)* — предшественник `PUTLOG`. Релиз-нота: «Во всех пакетных
заданиях заменён оператор PUTFLD на PUTLOG, что не приводит к появлению
неактуализированных записей» (`RELEASE_OVERALL.doc txt:3405`). Сохранился в старых
заданиях: `RDR/CHKrdr.gbl`, `CMPL/VDEL691.gbl`, `IBIS/LinkIbis.gbl` (11 вхождений).
Синтаксис идентичен PUTLOG (имя + формат-строка протокола).

**// (комментарий)** — может стоять между операторами; до 4 строк текста после себя
(`txt:6034-6035`). В реальных файлах часто `//` с одной строкой текста, а также
встречается `/*…` и `/* …` как строка-комментарий внутри групп.

#### Недокументированные операторы работы с полями (Прил.7 их не описывает)

**DEFFLD** — объявление/инициализация рабочего (модельного) поля. Структура как у
5-строчного оператора: `DEFFLD` / метка / + 3 строки-заполнителя. Наблюдается в
начале заданий, оперирующих модельными полями: `CMPL/Movekp.gbl` (`DEFFLD 3000`),
`CMPL/VDEL691.gbl` (`DEFFLD 3000`). Назначение по контексту — резервирование
служебной метки перед циклами. `TODO(recon #GBL-02)` — формальный синтаксис и
семантика DEFFLD не описаны в доступной доке.

**GETFLD** — чтение поля (метка во 2-й строке). Парен с DEFFLD/PUTFLD в группах
работы с модельными полями: `CMPL/Movekp.gbl:112`, `CMPL/VDEL691.gbl:112`,
`RDR/CHKrdr.gbl:467`, `VUZ/MoveVuzRdr.gbl:208`, `VUZ/OTVRDR.gbl:194` (вид
`GETFLD / 4000`). `TODO(recon #GBL-03)` — назначение (вероятно загрузка значения
служебного поля для дальнейшей обработки) по доке не подтверждено.

### 1.4 Параметры задания (для режима ГЛОБАЛЬНАЯ)

В задании можно объявить до 9 **параметров**, значения которых пользователь задаёт
перед запуском; в форматах они обозначаются `%1…%9` и подставляются перед
исполнением (`txt:5918`, `txt:6173-6187`). Структура — см. раздел 2.

## 2. Формат файла `.gbl`

Текстовый файл с расширением `.GBL`, кодировка CP1251 (`txt:5919-5922`).

```
<строка 1>      ЧИСЛО ПАРАМЕТРОВ (0…9)
[пара строк на каждый параметр, число пар = числу параметров:]
   <значение параметра ИЛИ имя .MNU/.WSS для выбора ИЛИ пусто>
   <наименование параметра (заголовок столбца) ИЛИ текст-подсказка>
[далее — группы строк операторов:]
   <ИМЯ ОПЕРАТОРА>
   <… строки оператора (для ADD/REP/CHA/CHAC/DEL — ещё 4 строки) …>
   …
```

- **Строка 1** — количество параметров. `0` = параметров нет (большинство
  служебных заданий: `autoin.gbl`, `SumEkzToSZ.gbl`, `Move83.gbl`).
- **Блок параметров** — N пар строк. 1-я строка пары: значение, либо имя файла
  `.MNU`/`.WSS` для выбора значения, либо пусто (тогда пользователю предложат
  ввести). 2-я строка пары: наименование/подсказка (`txt:5922`).
  Примеры реальных шапок:
  - `ClearPolzv.gbl`: `1` / `ClearPolzv.wss` / `Выбрать актуальные значения…` —
    один параметр, опрос через РЛ подполей `.wss`.
  - `Move691.gbl`: `2` / `Перенос отмеченных контингенов…` /
    `if … then 'MoveWPSecCat.wss' else 'Move691.wss' fi` / `ВВЕДИТЕ данные переноса`
    — 2 параметра, причём имя `.wss` для опроса вычисляется ФОРМАТОМ (динамический
    выбор рабочего листа подполей).
  - `del40.gbl`: `1` / (пусто) / `Введите дату (ГГГГММДД)…` — ввод значения вручную.
- **Группы операторов** — каждая начинается строкой-именем оператора; число
  последующих строк определяется оператором. Для `ADD/REP/CHA/CHAC/DEL` — ровно
  4 доп. строки (метка, повторение, ФОРМАТ 1, ФОРМАТ 2); незанятые строки —
  заполнитель `XXXXXXXXXXXXXXXXXXX` или пусто.

**Передача в `IC_gbl`**: задание передаётся либо как `@<имя>` (имя файла `.gbl`
без расширения, файл лежит в каталоге БД), либо непосредственными строками тела
задания в параметре `Agbl` (`PROTOCOL_REFERENCE.md` §3.7; `txt:987-1000`).

## 3. Каталог реальных `.gbl` (по БД)

Назначение приводится по комментариям внутри файлов. Файлы: `_work/cfg_txt/<DB>/<имя>.gbl`.

### 3.1 Автоввод (сценарий, имя по умолчанию AUTOIN.GBL, параметр AUTOINFILE)

«Сценарий автоввода — задание на глобальную (пакетную) корректировку с
фиксированным именем, по умолчанию AUTOIN.GBL» (`txt:6778`).

| Файл | БД | Назначение |
|---|---|---|
| `autoin.gbl` | IBIS | Полный автоввод ЭК: формирование 907 (каталогизатор/дата), обработка авторитетных файлов (621/675/606), размножение экземпляров, шифр/авторский знак, пополнение ATHRA/ATHR* и т.д. (2858 строк) |
| `autoink.gbl` | IBIS | Вариант автоввода |

## 4. Связь с протоколом: вызов `IC_gbl`

Серверная функция (`PROTOCOL_REFERENCE.md` §3.7 «Группа 9 — Пакетная»; `txt:987-1000`):

```
IC_gbl(Adbn, Aifupdate, Agbl, Asexp;  Amin, Amax;  Aseq, Amfnlist...)
       → out: протокол выполнения
```

| Параметр | Назначение |
|---|---|
| `Adbn` | имя БД |
| `Aifupdate` | признак актуализации записей после корректировки (см. `Aifupdate`/`IC_ifupdate`, §3.2/4) |
| `Agbl` | **само задание**: либо `@<имя>` (файл `.gbl` в каталоге БД), либо строки тела задания. Многострочный вход упаковывается `$0D0A→$3130` (`PROTOCOL_REFERENCE.md` §1.3, `txt:996`) |
| `Asexp` | прямое поисковое выражение для отбора фрагмента (§5 PROTOCOL_REFERENCE) |
| `Amin`, `Amax` | границы диапазона MFN (диапазон фрагмента) |
| `Aseq` | формат последовательного поиска (→ `0`/`1`); при заданном `Asexp` — последовательный по результату прямого |
| `Amfnlist` | список MFN: диапазон `0/min/max`; список `N/mfn…`; отрицательный `-N/mfn…` (все КРОМЕ) (`txt:936-952`) |

Сопоставление с интерфейсными параметрами Приложения 16 (оператор `GLOBAL` пакетного
режима Администратора, `txt:6817-6825`):

| `GLOBAL`-оператор / форма ГЛОБАЛЬНОЙ | `IC_gbl` |
|---|---|
| `TaskName` (имя задания) | `Agbl` (`@имя` или строки) |
| `MfnFrom`, `MfnTo` | `Amin`, `Amax` |
| `RecUpdifAble` (актуализация) | `Aifupdate` |
| `Sexp` (запрос ИРБИС; `?`-префикс = последовательный) | `Asexp` / `Aseq` |
| `FlcAble`, `AutoinAble` | — (серверно не входят в сигнатуру `IC_gbl`; задаются режимом БД/контекстом) |
| `BatchLogFileName` | — (протокол возвращается в `answer`) |

**Кто вызывает в вебе**: режим КОРРЕКТИРОВКА-ГЛОБАЛЬНАЯ доступен в АРМе
«Каталогизатор» (параметр `GLOBABLEMAIN`, `txt:3590`) и в серверном «Администратор»
(Приложение 16). Признак разрешения без предварительной отметки записей —
`GLOBTOTALMAIN` (`txt:3591`). Веб-Каталогизатор/Администратор формирует вызов
`IC_gbl` с этими параметрами; задание берётся либо из списка (`GLOB.MNU` в каталоге
БД, `txt:6796`), либо строками из формы.

Примечание: в `PROTOCOL_REFERENCE.md` для `IC_gbl` стоял открытый
`TODO(Проход Б #07: грамматика задания Agbl.)` — настоящий документ закрывает
грамматику тела `Agbl` (разделы 1-2).
