# Управляющие словари (значения `.mnu`) — справочные данные для веб-форм

> Фаза 5. Реальные пары **код → расшифровка** вендор-стандартных справочников ИРБИС (виды документов, языки, страны, статусы, причины и т.д.) — это reference-данные, которые веб-формам/рендеру нужны дословно. Полная таблица (31 словарь, **940 значений**) — [`CONTROL_VOCABULARIES.csv`](CONTROL_VOCABULARIES.csv) (колонки Menu, Code, Label, DB). Ниже встроены операционные (короткие) словари; крупные классификаторы — счётчиком + ссылкой на CSV.
> ⚠ Институционально-зависимые словари демо-установки (факультеты `fak.mnu`, кафедры `kafch/kafv.mnu`, специальности `spec.mnu`, места хранения `mhr.mnu`, категории читателей `50.mnu`, сигла `sigl.mnu`) сюда НЕ включены — это данные конкретной библиотеки; для пилота СПб ГТБ берутся с её сервера.

## 1. Статусы экземпляра — `ste.mnu` (14; поле 910^A) — критично для книговыдачи
| Код | Значение |
|---|---|
| `0` | для ЭК — отдельный экз., поступил по месту хранения |
| `R` | для ЭК — группа экз-ров (размножение с вводом инв.№) |
| `U` | для ЭК ВУЗа — группа экз-ров (безынв. учёт) |
| `C` | группа экз-ров для библиотеки сети |
| `E` | сетевой локальный ресурс |
| `8` | номер журнала/газеты поступил, но не дошёл до места хранения |
| `2` | отдельный экз. ещё не поступал, ожидается |
| `3` | в переплёте |
| `4` | утерян |
| `5` | временно не выдаётся |
| `6` | списан |
| `p` | номер журнала переплетён (входит в подшивку) |
| `1` | **выдан читателю** |
| `9` | **на бронеполке** |
*(Свободны для выдачи: `0`/`U`/`C`. Вариант для списания — `stw.mnu`, 12 значений.)*

## 2. Книговыдача
**Причины отказа — `otkaz.mnu` (10; RQST 44^A):** `01` Занято · `02` Нет на месте · `03` Нет на месте и по топокаталогу · `04` В переплёте · `05` Списано · `06` Лакуна · `07` Дефектный · `08` Ветхий · `09` Перенаправление · `*` Прочие.
**Статусы брони — `reservstatus.mnu` (5; RQST 910^A):** `0` отправлен с МХ на МВ · `1` получен с МХ/на МВ/для выдачи · `2` получен от читателя/на МВ/для выдачи · `3` получен от читателя/на МВ/для возврата на МХ · `4` отправлен с МВ на МХ.
**Сроки возврата — `return.mnu` (4):** 20/30/50/60 дн.

## 3. Комплектование
**Способ поступления — `pst.mnu` (5):** `k` закупка · `d` дар · `p` передача · `m` полевые материалы · `z` неизвестен.
**Канал поступления — `kp.mnu` (30):** ⚠ содержит install-specific поставщиков/магазины демо-базы (ГПНТБ, ВИНИТИ, Межкнига, ЦКБ, Дары, Книгообмен, Покупка…) — в CSV; для пилота заменяется.
**Причины выбытия — `prsp.mnu` (13):** устаревшие / непрофильные / ветхие / узкого спроса / утерянные абонентом / погибшие / излишняя экземплярность / окончание срока / пропажа / длительное отсутствие / отзыв изданий РКП / другие.
**Вид передачи выбывших — `vp.mnu` (2):** передача заданного комплекта отдельных NN журналов / всех NN за год.
**Период подписки — `33.mnu` (8):** `..../0` год · `..../1`,`..../2` полугодия · `..../1-1…1-4` кварталы · `..../P:N-K` нестандартный период (с N-го по К-й месяц).
**Валюта — `val.mnu` (23):** Евро, $ Доллары США, DM, FFr, SFr, Y Йены, R Рэнды… (полный — CSV).
**Этап работы — `etr.mnu` (15):** этапы технологии (ПК/РЗ/ИЗ/КТ/ДК/КР/ВБ/ПФ…) — CSV.

## 5. Да/нет и роли
`danet.mnu`/`dn.mnu`/`di.mnu`/`df.mnu` — переключатели да/нет (для редактора типа 7); `idr.mnu` — константа «и др.»; роли авторов/заглавий — `rol*.mnu` (см. DB_IBIS §6).
