# Полный каталог полей/подполей — машинное извлечение из всех `.wss`

> Фаза 4. Извлечено из **1605 рабочих листов подполей** (`.wss`) по всем БД `Datai`: **3366 пар (поле×подполе) на 274 полях**. Полная машинная таблица — [`FIELD_CATALOG.csv`](FIELD_CATALOG.csv) (колонки: Field, Subfield, Label, EditTypes, InputAid, Occurs, DBs). Дополняет курируемый [FIELD_DICTIONARY.md](FIELD_DICTIONARY.md) (115 полей) — здесь покрытие шире (включая редкие поля), с **режимом ввода/валидации на каждое подполе**.

## 1. Метод и структура `.wss`
Каждый `.wss` = форма подполей одного поля. Формат: строка 0 = число подполей `N`; далее **блоки по 10 строк** на подполе:
| смещение в блоке | содержимое |
|---|---|
| +0 | код подполя `^x` (1–2 символа) |
| +1 | подпись (label) |
| +2 | флаг повторяемости |
| +3 | контекст-код (`8MMMNNN` или 0) |
| +4 | **тип-редактора (режим ввода/валидации)** |
| +5 | средство ввода (input-aid) |
| +7 | подсказка (hint) |

Метка поля выведена из имени файла (`910m.wss`→910, `v40.wss`→40, `701d.wss`→701). 58 пар — «(nonnum)»: вспомогательные `.wss` без числового поля (`fo.wss`, `spec.wss` — меню-помощники).

## 2. Тип-редактора = правило ввода/валидации подполя (с частотами)
| Код | Режим | Встречаемость |
|---|---|---|
| `0` | свободный текст | 1595 |
| `1` | выбор из меню `.mnu` | 738 |
| `2` | ввод с помощью словаря `,префикс=,@формат` | 337 |
| `7` | да/нет (`NoYes.mnu`/`dn.mnu`/`danet.mnu`) | 169 |
| `6` | иерархический справочник `.tre` | 44 |
| `10` | ввод через авторитетный файл (`ATHRA,A=,…`) | 41 |
| `3` | словарь с маской · `4` многострочный · `5` вложенный РЛ · `8` дата · `11` тезаурус · `16` файл | реже |

(Многие подполя имеют несколько режимов в разных вариантах формы — напр. `0; 1; 2` = текст/меню/словарь.)

**Семантика средства ввода (input-aid):**
- `,<префикс>=,@<флк>` — ввод с подсказкой из словаря по префиксу + формат-контроль (напр. `,A=,@!amovu` → словарь авторов `A=` + ФЛК `!amovu`).
- `<DB>,<префикс>=,…` — кросс-БД словарь/авторитет (напр. `IBIS,T=,@!kvmovt`; `ATHRA,A=,1=,@athrai`).
- `<имя>.mnu\\<колонка>` — меню с выбором колонки (напр. `coll.mnu\\2`).
- `<имя>.tre` — дерево; пусто — свободный ввод.

## 4. Примеры (с режимом валидации; ⚠ один числовой тег может иметь разный смысл в разных БД/РЛ)
**Поле 910 (экземпляры)** — выборка: `^A` Статус [меню `ste.mnu`], `^B` Инв.№ [словарь `,INK=,`], `^6` Способ поступления [меню `pst.mnu`], `^9` Коллекция [меню `coll.mnu\\2`], `^4` Не на баланс [да/нет `dn.mnu`], `^0` Число проверенных, `^=` КМИ, `^!` Реальное местонахождение.
**Поле 200 (заглавие)** — `^A` Заглавие [словарь `,TP=,@!tmovzh`], `^C` Заглавие книги/многотомника [`IBIS,T=,@!kvmovt`], `^E` Свед. к заглавию [меню+дерево `200e.mnu`], `^F` 1-е свед. об отв. [словарь `,A=,@!amovf`], `^G` Послед. свед. об отв. [`,M=,@!mmovf`], `^B` Общее обозначение материала [`200b.mnu`]. *(Подполя `^D` «Дата выдачи»/`^K` «Штрих-код» — из циркуляционного варианта `200r.wss`, не каталожного.)*
**Поле 700 (автор)** — `^A` Фамилия [словарь `,A=,(v***^a,|^B|v***^b/)`], `^B` Инициалы, `^C` Доп. данные, `^R` Разночтение фамилии [`,A=,@!amovo`], `^X` «и др.» [меню `idr.mnu`], `^Y` Работает в орг. [да/нет `dn.mnu`].
