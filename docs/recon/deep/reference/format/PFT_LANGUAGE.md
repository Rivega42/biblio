## 2. Базовые элементы — команды вывода полей

| Конструкция | Назначение | Ссылка |
|---|---|---|
| `v<метка>` | Вывод поля (буква V прописная/строчная), напр. `v200`, `v700` | irbis64_2022.doc (txt:4360) |
| `v<метка>^<подполе>` | Вывод подполя; разделитель `^` + латинская буква/цифра (регистр не значим), напр. `v210^c` | irbis64_2022.doc (txt:4328, 4376) |
| `v<метка>^*` | Вывод первого подполя (если оно есть) | irbis64_2022.doc (txt:4377) |
| `v…*<смещение>.<длина>` | Вывод фрагмента поля/подполя; позиции с 0; напр. `v1*3.3`, `v210^c.3` | irbis64_2022.doc (txt:4393–4397) |
| `mfn` / `mfn(d)` | Вывод номера записи MFN; `d` — число цифр (по умолч. 6) | irbis64_2022.doc (txt:4423–4425) |

При ссылке на поле целиком нулевое смещение указывает на первую позицию (включая разделители); при ссылке на подполе — на первый символ данных после разделителя (txt:4401).

## 3. Режимы вывода (`Mmc`)

Команда вида `M` + режим (`m`) + регистр (`c`); действует до следующей команды режима; по умолчанию **`MPL`** (irbis64_2022.doc txt:4448–4458).

| Команда | Режим | Поведение | Ссылка |
|---|---|---|---|
| `MP` | Проверки (proof) | Поля как хранятся, без авторазделителей | irbis64_2022.doc (txt:4442, 4451) |
| `MH` | Заголовка (heading) | Управляющие символы `<` `>` игнорируются; разделители подполей → знаки пунктуации | irbis64_2022.doc (txt:4442, 4452) |
| `MD` | Данных (data) | Как заголовок + автоточка `.` + два пробела после поля (подавляется суффикс-литералом) | irbis64_2022.doc (txt:4442, 4453) |
| `…U` | — | Преобразование букв в ПРОПИСНЫЕ | irbis64_2022.doc (txt:4456) |
| `…L` | — | Без преобразования регистра | irbis64_2022.doc (txt:4457) |

Замещение разделителей подполей в режимах H/D: `^a` → `"; "`; `^b`…`^i` → `", "`; все другие → `". "`; комбинация `><` → `"; "` (txt:4443–4446). Примеры: `mpl`, `mhl`, `mdl`, `mdu` (txt:4458–4475). В реальном `brief.pft` используются `mhl`, `mpl`, `mpu` (cfg IBIS/brief.pft).

## 4. Размещение (вертикальное/горизонтальное)

| Команда | Функция | Ссылка |
|---|---|---|
| `Xn` | Вставить n пробелов | irbis64_2022.doc (txt:4483) |
| `Cn` | Табуляция в позицию n (только моноширинные шрифты) | irbis64_2022.doc (txt:4485, 4504) |
| `/` | Переход на новую строку (условный — если строка не пуста) | irbis64_2022.doc (txt:4487, 4496) |
| `#` | Переход на новую строку (безусловный) | irbis64_2022.doc (txt:4490, 4497) |
| `%` | Удалить ранее созданные пустые строки | irbis64_2022.doc (txt:4492, 4501) |

Идиома `%##V10%##V20…` даёт ровно одну пустую строку между полями независимо от их наличия (txt:4502). **Важно:** `/ # %` неприменимы в RTF- и HTML-форматах; `Cn`, `Xn` не работают в HTML (txt:4504, 5091, 5099).

## 5. Литералы

| Тип | Ограничители | Поведение | Ссылка |
|---|---|---|---|
| Условный | `"…"` | Выводится 1 раз при наличии поля/подполя | irbis64_2022.doc (txt:4508) |
| Повторяющийся | `|…|` | Выводится для каждого экземпляра поля/подполя | irbis64_2022.doc (txt:4508) |
| Безусловный | `'…'` | Выводится всегда | irbis64_2022.doc (txt:4508) |

- Литерал не должен содержать собственного ограничителя (txt:4509).
- `|xxx|+` — повторяющийся префикс перед каждым, кроме первого, экземпляром; `+|xxx|` — суффикс после каждого, кроме последнего (txt:4511–4512).
- Префикс-литералы стоят перед командой вывода, суффикс — после; суффикс-литералы не разделяются запятыми (запятая = конец суффиксов) (txt:4510, 4514–4521).
- Пустые литералы `""`/`||` допустимы (напр. для подавления автопунктуации) (txt:4523).

## 6. Команды фиктивного вывода поля (`d` / `n`)

Условный вывод литералов по наличию/отсутствию поля без вывода его содержимого (irbis64_2022.doc txt:4558–4564):

| Конструкция | Смысл | Ссылка |
|---|---|---|
| `Dt` / `Dt^x` | Связанные условные литералы выводятся, если поле/подполе **присутствует** | irbis64_2022.doc (txt:4562) |
| `Nt` / `Nt^x` | Литералы выводятся, если поле/подполе **отсутствует** | irbis64_2022.doc (txt:4562) |

Команде должен предшествовать хотя бы один условный префикс-литерал; суффикс-литералы недопустимы (txt:4565). Примеры: `"(Отсут.)"n700,v700`; `"[нет даты]"n907^А,V907^A` (txt:4571–4577). В реальном `brief.pft`: `d461`, `d700^g`, `d923`, `d330` (cfg IBIS/brief.pft).

## 7. Выражения

**Числовые** — операнды (константы, числовые функции, `mfn`, числовые выражения в скобках) и операторы `+ - * /`; приоритет как в алгебре (txt:4582–4592). Поля-строки не могут быть операндами — нужна `val()` (txt:4593).

**Строковые** — один операнд: безусловный литерал, команда вывода поля/подполя, строковая функция (txt:4608–4610).

**Логические** — для команды IF. Операторы отношения: `=`, `<>`, `<`, `<=`, `>`, `>=`, `:` (содержит, только для строк, регистронезависим) (txt:4616–4632). Логические операторы: `NOT`, `AND`, `OR`; приоритет NOT > AND > OR (txt:4633–4635).

## 9. Форматные выходы и UNIFOR

### 9.2 Шаблон-форматы (`&uf('6...#...')`)

Шаблон-формат содержит переменные параметры `%N`; расширенная конструкция `&uf('6<шаблон>#<п1>,<п2>,...')` подставляет значения. Позволяет сократить тексты при идентичных конструкциях для разных полей (напр. вывод авторов из 700/701/702 одним шаблоном AUTHOR) (irbis64_2022.doc txt:4960–4978). Список шаблон-форматов — файл `.TRE` (по умолч. `PFT.TRE`) (txt:5240, 5242).

## 10. Команда IF и повторяющиеся группы

**IF:** `IF условие THEN формат-1 ELSE формат-2 FI`; `ELSE` необязателен, `IF`/`THEN`/`FI` обязательны; допускается вложенность, каждый IF закрывается своим `FI` (irbis64_2022.doc txt:5006–5018). Особенно полезна для обобщённых форматов интегрированных БД с разными типами записей (txt:5021).

**Повторяющиеся группы `(...)`:** набор команд в круглых скобках; обработка каждого экземпляра повторяющегося поля отдельно, экземпляр за экземпляром (txt:5022–5029). Не могут быть вложенными, **кроме** случая, когда внутренняя группа — в аргументе `ref()` (txt:5069–5073). Обязательны при: повторяющемся поле как аргументе `l()`; ссылке первого аргумента `ref()` на повторяющееся поле (txt:5076–5078). Безусловные литералы в группе нежелательны (выдаются на раз больше) (txt:5030).

## 11. Глобальные переменные

Хранятся в рабочей области сеанса. Группы/списки — через `&unifor('+1...)`; индивидуальные повторяющиеся — через `&unifor('+7...)` (альтернатива). Эквивалент чтения `+7Rnnn` — конструкция `Gnnn` внутри повторяющейся группы. Очистка всех переменных — `&unifor('+1')` или `&unifor('+7')` (irbis64_2022.doc txt:4836–4851, 4904–4926).

## 12. Оптимизированные форматы (`@имя`, `.OPT`, ветвление по полю 920)

Оптимизированный формат — механизм автопереключения форматов показа по виду документа; имя OPT-файла задаётся параметром **PFTOPT** (irbis64_2022.doc txt:5144). Структура OPT-файла:

```
<метка>|<формат>|@<имя_формата>
<длина>
<значение_1> <имя_формата_1>
...
*****
```

Ключ задаётся меткой поля, прямым форматом или `@<имя_формата>`; `<длина>` — макс. длина значения вида; в значениях допустим символ маскирования `+` (txt:5145–5159). Для IBIS два файла: **PFTW.OPT** (RTF-форматы), **PFTW_H.OPT** (HTML, по умолчанию) (txt:5160–5163).

Подтверждение реальным конфигом (`cfg IBIS/PFTW.OPT`) — ветвление по **полю 920** (код вида описания / имя рабочего листа):

```
920
5
J     JW       NJ  NJW     SPEC  MN     ASP  ASP
MUSP  MUSI      BOUNI VS600      +++++ KN
```

В форматах ветвление по типу записи делается через `IF v920='J' …`, `IF v920:'NJ' …`, `&unifor('Av920#1'):'SPEC'` (cfg IBIS/brief.pft, строки 1–2, 18, 24, 56). Ссылка `@<имя>` на формат — также в `.WSS` для расширенного ввода (`@!tmov0`, `@!amovf`) (cfg IBIS/200SI.WSS).

## 13. Форматы логического контроля (ФЛК)

Применяются при вводе/сохранении (контроль поля и документа). Форматированию подвергается контролируемый документ; **первый символ результата** = результат контроля (irbis64_2022.doc txt:5117–5124):

| Символ | Результат |
|---|---|
| `0` | Положительный |
| `1` | Отрицательный, ошибки **непреодолимые** (обязательны к устранению) |
| `2` | Отрицательный, ошибки **преодолимые** (можно не исправлять) |

Остальной результат (со 2-го символа) при отрицательном контроле выдаётся как сообщение. Форматы ФЛК обычно содержат IF; имена начинаются с `!` (напр. `!10`, `!101`, `!200`, `!900`, `!910`, `!920`, `!964`) (txt:5125, 5136). Общий ФЛК при сохранении — `dbnflc` (txt:5136). Форматы с **переменными метками** `***` (три звёздочки) — для переноса данных через словарь; исходное — термин словаря (txt:5126–5132).

## 14. Вывод RTF / HTML, гиперссылки, штрих-/QR-коды

**RTF-форматы:** команды RTF вводятся как литералы; весь формат заключается в `'{' формат '}'`; команды RTF начинаются с `\` и заканчиваются пробелом (irbis64_2022.doc txt:5081–5088).

| Команда RTF | Назначение | Ссылка |
|---|---|---|
| `\par` | Новый абзац | irbis64_2022.doc (txt:5090) |
| `\liNNN` `\riNNN` `\fiNNN` | Отступы (лев./прав./первой строки), в пунктах | irbis64_2022.doc (txt:5090) |
| `\pard` | Сброс форматирования абзаца | irbis64_2022.doc (txt:5090) |
| `\fsNNN` | Размер шрифта (удвоенный в пунктах) | irbis64_2022.doc (txt:5090) |
| `\ul \ul0` `\i \i0` `\b \b0` | Подчёркивание / курсив / жирность (вкл/выкл) | irbis64_2022.doc (txt:5090) |
| `\tab` `\txNNN` | Табуляция / позиция табуляции | irbis64_2022.doc (txt:5090) |
| `\qr \ql \qc \qj` | Выравнивание (право/лево/центр/по ширине) | irbis64_2022.doc (txt:5090) |
| `\page` | Новая страница | irbis64_2022.doc (txt:5090) |

**HTML-форматы:** HTML-теги как литералы (альтернатива RTF, совмещать нельзя); явное определение — фиктивный тег `'</>'` в начале (txt:5093–5099).

**Гиперссылки и ресурсы ИРБИС** (в HTML, теги `<A HREF>`/`<IMG SRC>`):

| Конструкция | Назначение | Ссылка |

| `<A HREF="IRBIS:3,path,dbname,filename">` | Ссылка на внешний объект (path: 0 осн. дир., 1 общая `\DATAI`, 10 дир. БД) | irbis64_2022.doc (txt:5106–5110) |

| `<IMG SRC="IRBIS:!!ШТРИХ_КОД!!">` | Отображение **штрих-кода** в HTML-формате | irbis64_2022.doc (txt:5115–5116) |

## 15. Ошибки обработки формата

При ошибке форматирование прекращается, в результат выдаётся `*** Format error n` (irbis64_2022.doc txt:5165–5168). Коды (выборка): `1` конец формата в повтор. группе; `2` вложенность групп; `8` IF без THEN; `19/20` непарные скобки `(`/`)`; `26` операнды разных типов; `28` 1-й аргумент REF не числовой; `51` слишком много литералов; `53` IF без FI; `54` `+` без повтор. литерала; `56` переполнение рабочей области; `57` зацикливание группы; `58` нечисловой аргумент F; `60` нестроковая функция как команда; `61` аргумент A/P не команда вывода; `99` неизвестная команда; `101/102` переполнение/ошибка стека (txt:5174–5191).
