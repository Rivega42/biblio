# Каталог PFT-модулей и выходных / табличных форм ИРБИС64 (БД IBIS)

> Фаза 5 ресерча веб-ИРБИС64. Цель — каталог PFT-подпрограмм (шаблон-модулей) и выходных/печатных форм для пересборки отображения и печати.
> Источники: реальные конфиги БД IBIS (CP1251 → UTF-8 зеркало `_work/cfg_txt/IBIS/`, `_work/cfg_txt/Deposit/`). Опора: `reference/format/PFT_LANGUAGE.md`, `reference/databases/DB_IBIS.md` (§5), `reference/protocol/PROTOCOL_REFERENCE.md`.
> Только из файлов. Факты — со ссылкой на файл. Неразобранное — `TODO(recon #MOD-NN)`.

## 0. Где лежат тела форматов (важно для пересборки)

Папка БД `Datai\IBIS\` содержит **291** `*.pft` (преимущественно экранно-индексные `!*`, табличные тела `TAB*`, ФЛК). Главные форматы показа (`KN/MN/ASP/JW/…`) и почти все библиографические модули-подпрограммы (`osvex`, `ozboz`, `otitkn`, `o200*`, …) физически лежат в общем депозитарии **`Datai\Deposit\`** (1396 файлов `*.pft`) — это разделяемый набор форматов, наследуемый через путь поиска PFT (`cfg IBIS/KN.PFT` отсутствует в `Datai\IBIS`, найден в `Datai\Deposit\KN.PFT`; то же для `OSVEX/OZBOZ/OTITKN.PFT`).

| Где | Что | Кол-во |
|---|---|---|
| `Datai\Deposit\*.pft` | главные форматы + модули-подпрограммы `6<имя>` | 1396 |
| `Datai\IBIS\*.pft` | экранно-индексные `!*`, тела таблиц `TAB*.PFT`, ФЛК | 291 |

## 1. Дерево PFT-подпрограмм (`Pft.tre`)

`Pft.tre` (772 строки) — справочник шаблон-форматов (по `PFT_LANGUAGE.md` §9.2: список шаблон-форматов хранится в `.TRE`, по умолчанию `PFT.TRE`). Структура: строки уровня `- <НАЗВАНИЕ ФОРМЫ>` (27 групп верхнего уровня) и вложенные узлы `<имя модуля> - <назначение>` с отступом-табуляцией, отражающим **порядок и иерархию вызова**. Всего узлов-строк с описанием — 739, уникальных имён модулей-подпрограмм — **68** (`cfg IBIS/Pft.tre`).

Модули вызываются из главных форматов как форматный выход `&uf('6<имя>')` (UNIFOR-код `6` — вложенный/шаблон-формат, `PFT_LANGUAGE.md` §9.1), либо напрямую по имени-формату (`Otitkn`, `oizd`, …) в `KN.PFT` (см. §2.2).

## 2. Цепочки главных форматов (диспетчер по полю 920)

### 2.1 Механизм выбора главного формата (`PFTW.OPT` / `Pftw_H.opt`)

Оптимизированный формат — автопереключение по виду документа (поле **920**), формат OPT-файла: `<метка>` / `<длина>` / пары `<значение> <имя_формата>` / `+++++` (умолчание) / `*****` (`PFT_LANGUAGE.md` §12). Для IBIS два файла: `PFTW.OPT` (RTF-показ) и `Pftw_H.opt` (HTML, по умолчанию) (`cfg IBIS/PFTW.OPT`, `cfg IBIS/Pftw_H.opt`):

| Значение 920 | RTF (`PFTW.OPT`) | HTML (`Pftw_H.opt`) | Вид документа |
|---|---|---|---|
| `J` | `JW` | `Jw_H` | журнал |
| `NJ` / `NJP` / `NJK` | `NJW` | `NJw_H` | номер/газета/подшивка |
| `SPEC` | `MN` | `Mn_H` | том многотомника |
| `ASP` / `AUNTD` | `ASP` | `Asp_H` | аналит. описание |
| `MUSP` | `MUSI` | `Musi_H` | музейный предмет |
| `BOUNI` | `VS600` | `VS600_H` | универсальная запись |
| `+++++` (по умолч.) | **`KN`** | **`Kn_H`** | книга (моноиздание) |

Доп. OPT по контексту: `PFTKOW.OPT` (АРМ «Книгообеспеченность», умолч. `BOKO_H`), `PFTPW.OPT` (Комплектатор, умолч. `IBISKW`) (`cfg IBIS/PFTKOW.OPT`, `cfg IBIS/PFTPW.OPT`; `DB_IBIS.md` §5.1). Экранный (RTF) и HTML (`_H`) — одно содержание, разная разметка.

### 2.4 `BRIEF.PFT` (свёртка для списков/словаря) — построчный разбор-образец

`IBIS\brief.pft` (74 строки) — одностроч. БО, диспетчер по 920 без вызова модулей `6*` (всё inline). Ключевые ветки:

| Строка | Логика | Назначение |
|---|---|---|
| 1 | `IF v920='SZPRF' then 'Служебная запись - Проверка фонда' else` | спец-запись групповой проверки фонда |
| 2 | `mhl, if v920:'NJ' then else` | режим заголовка (`PFT_LANGUAGE.md` §3); ветка не-номера журнала |
| 3 | `v503^a, v461^x …` | заглавие общей части (503) + многотомник (461) |
| 13–17 | `if a(v461) then v700^a … v200^A` | для немноготомника: автор (700) + основное заглавие (200) |
| 18–21 | `if v920='J' then (… v923^h/i/k/l/m/n …)` | для журнала — кумуляция номеров (923) |
| 22–28 | `else (if p(v923) then … &unifor('Av920#1'):'SPEC' …)` | для тома/спецификации |
| 30 | `if v920='J' then '. - '…' выходит с 'v210^d'г.' else ', 'v210^d` | год издания (210^d) |
| 37–53 | `if v920:'NJ' then ref(l(\|I=\|v933), …)` | для номера журнала: подтягивание заглавия журнала по 933 через `ref(l(...))` (связь Журнал↔Номер, `DB_IBIS.md` §7) |
| 56–68 | `if v920.1:'A' then '.- 'v963^1 … // 'd463 …` | для аналитики: «// источник» из 463/963 |
| 74 | `&unifor('!'),&unifor('+F'),&uf('[')` | постредактура: очистка двойных разделителей и RTF-конструкций (`PFT_LANGUAGE.md` §9.1 коды `!`, `+F`) |

Источник: `IBIS/brief.pft` (строки 1–74). `BRIEF` ветвится через `IF v920=…`, `IF v920:'NJ'`, `&unifor('Av920#1'):'SPEC'` — канонический пример ветвления по типу записи.

### 2.5 Образец тела модуля — `osvex` (свободные экземпляры)

`Deposit\OSVEX.PFT`: вычисляет наличие свободных экземпляров перебором поля 910 (повторяющаяся группа `(…)` + `rsum`), отбрасывает статусы из `Kmhrvn.mnu`, печатает плашки «(Экземпляры есть только в отделах)», «(Есть только полный текст)», «(Свободных экземпляров нет)» в зависимости от 910^a/993 (`Deposit/OSVEX.PFT` строки 1–20+). Демонстрирует типичную структуру модуля: повторяющиеся группы по 910, `&unifor`-функции, RTF-литералы `\tab \b … \b0 \par`.

## 3. Табличные / печатные формы (для `IC_print`)

### 3.1 Протокол `IC_print` (как сервер запускает табличную форму)

`IC_print(Adbn, Atab, Ahead, Amod, Asexp; Amin,Amax; Aseq, Amfnlist; answer; abufsize)` — `Atab` = `@имя` табличной формы, `Ahead` ≤ 3 строк заголовка, выход — **RTF**; результат = пересечение прямого ∩ последовательного ∩ `Amfnlist` списков (`PROTOCOL_REFERENCE.md:114`, txt:927-962). Имя в `Atab` указывает на комплект файлов формы в папке БД.

### 3.2 Комплект файлов одной табличной формы

Одна форма `<NAME>` физически = набор одноимённых файлов в `Datai\IBIS\` (на примере `TABINW`, `TABIUW`):

| Расширение | Роль | Пример содержимого |
|---|---|---|
| `.tab` | Каркас таблицы (RTF): шапка документа, плейсхолдер `[Header]`, секции | `TABINW.tab` |
| `.pft` | **Тело строки** — формат БО, выполняемый для каждой записи; одна RTF-ячейка (`\cellxNNNN … \cell`) на колонку | `TABINW.PFT` |
| `.hdr` | Привязка рабочего листа+pft (динамическая вёрстка) | `TABINW.HDR` = `@tab4ow.wss!tab4ohw.pft#` |
| `.srw` | Параметры сортировки (ключ сортировки) | `TABINW.SRW` |
| `.tbg` | Декларативное определение таблицы (нового типа ИРБИС64) | `TABSTATJ.tbg` |
| `.tbu` | Альтернативное тело (для статистических с итогами) | `TABSTATJ.tbu` |

### 3.3 Структура `.tab` (секции)

`.tab` — текстовый файл из именованных секций, разделённых `*****`. По всем 17 формам IBIS встречаются (счётчик `grep`):

| Секция | Кол-во форм | Назначение |
|---|---|---|
| `[Tab]` | 15 | RTF-каркас: преамбула `{\rtf1…}`, шапка документа, строка заголовка таблицы |
| `[Header]` | 15 | плейсхолдер-маркер внутри `[Tab]`, куда подставляется строка-шапка с колонками (`\trowd…\trhdr…\cellxNNNN \cell …\row`) |
| `[FormatCode]` | 12 | кодировка вывода (`WIN`) |
| `[Sum]` | 4 | строка-итог (суммирование через `&uf('+7W…')`/`rsum`) |
| `[Numeration]` | 1 | стиль автонумерации строк (`##`) |

**Пример `TABINW.tab`** («Лист инвентарной книги», `IBIS/TABINW.tab`): `[FormatCode]=WIN`; `[Tab]` содержит RTF-преамбулу + заголовок «Лист инвентарной книги»; `[Header]` разворачивается в строку шапки `\trowd … \trhdr` с 10 колонками, заданными парами «граница `\cellxNNNN` + текст»: «Дата, № КСУ» (cellx1809), «№ п/п» (2801), «Инв.№.Штрих-код» (4076), «Подразделение» (4926), «Отм. о пров.» (5634), «Автор и заглавие» (8469), «Год изд.» (9177), «Раздел знаний» (9885), «Цена» (11019), «№ акта выбытия» (12153) (`IBIS/TABINW.tab` строки 12–82). `\cellxNNNN` — правая граница колонки в твипах; ширина колонки = разница соседних значений.

**Тело строки** `TABINW.PFT` — те же 10 ячеек `\cellxNNNN … \cell`, но вместо текста — PFT-выражения по полям записи: дата/№ КСУ из 910^c/910^u, инв.№ из 910^b (с нумерацией `&unifor('S1A')`), штрих-код 910^h, подразделение 910^d, отметка 910^s, автор+заглавие (461^x,700,200), год 210^d, раздел знаний v60, цена 910^e (`IBIS/TABINW.PFT`). Перебор экземпляров — повторяющейся группой по 910 с фильтром по 991 (КСУ/подразделение).

### 3.4 Декларативное определение таблицы (`.tbg`) — формат нового типа

`.tbg` (22 файла) задаёт таблицу **декларативно**, без ручного RTF — ключевой формат для веб-пересборки. Пример `TABSTATJ.tbg` («Статистика поступившей периодики», `IBIS/TABSTATJ.tbg`):

```
[TAB]
TAB_PAGE_FORMAT=0          ; формат страницы
TAB_PAGE_ORIENT=0          ; ориентация (0 портрет, 1 ландшафт)
TAB_PAGE_LEFT/RIGHT/...    ; поля (см)
AUTONUM=3                  ; автонумерация
ALIGN=2 / BORDER_WIDTH=1 / FONT_SIZE=10
TAB_IRBIS64=1              ; признак таблицы ИРБИС64
CELL_TAB_COL_COUNT=7       ; число колонок
CELL_TAB1_1=2              ; ширина колонки 1 (см)
CELL_TAB1_2=N              ; заголовок колонки 1
CELL_TAB1_3=##             ; формат-выход (тело ячейки) — здесь автономер
CELL_TAB1_4= / CELL_TAB1_5=; сноска / прим.
CELL_TAB2_1=1  CELL_TAB2_2=№ п/п  CELL_TAB2_3=&uf('S1A')'.'  CELL_TAB2_4=…
CELL_TAB3_1=10 CELL_TAB3_2=Краткое БО CELL_TAB3_3=&uf('+7W991#'v991),&uf('+7W11#'),…,&uf('7,!I933='v903…
```

Колонка = группа `CELL_TABn_*`: `_1` ширина, `_2` заголовок, `_3` формат-выход (PFT, в т.ч. с `&uf`/`ref`), `_4`/`_5` сноски. `TAB_IRBIS64=1` отличает новый декларативный тип от старых ручных RTF-`.tab`. `KoPlan.tbg` (учебный план КО): 9 колонок, ландшафт (`TAB_PAGE_ORIENT=1`), колонки «N / Дисциплина / Кол-во студентов / Состояние фонда / …» (`IBIS/KoPlan.tbg`).

### 3.5 Меню табличных форм (`Tabw.mnu` / `Tabw.tre`)

`Tabw.tre` (49 строк) — дерево форм по группам; `Tabw.mnu` (79 строк) — плоское меню «имя ↔ описание» (`IBIS/Tabw.tre`, `IBIS/Tabw.mnu`):

| Группа (`Tabw.tre`) | Формы |
|---|---|
| КАТАЛОЖНЫЕ КАРТОЧКИ/ФОРМУЛЯРЫ | `!!KK_K` (комплект), `!!KK_O` (основная), `!!KK_D` (добавочные), `!!KK_S` (ссылочные), `!!KK_Sk` (для СК), `!!KK_Tk` (для ТК), `!!KK_ob` (оборот), `!!KK_obmh` (оборот по МХ), `!!KKF` (формуляры), `!!KK` (комплект с настройкой) |
| УКАЗАТЕЛИ | `TABBUW`, `TABBAW`, `TABBNW` (библиогр.: предметный/авт.-загл./новых поступл.); `TABSUW`/`_1`, `TABSU0`/`_1`, `TABSU0P`/`_1` (систематические, в т.ч. для персоналий) |
| СОРТИРОВАННЫЕ СПИСКИ | `BRIEF_TAB` (по заголовку), `BRDAT_TAB` (по дате ввода), `BRYEAR_TAB` (по году выхода) |
| ИНВЕНТАРНЫЕ КНИГИ/АКТЫ | `TABIUW` (инд.учёт), `TABINW` (общий), `TAB4BW`/`TAB4OW`/`TAB4OINV` (сквозная сортировка), `TABF3W`/`TABF3WI` (лист актового учёта) |
| СВЕДЕНИЯ О ПОСТУПЛЕНИЯХ | `TNNJW` (непоступившие номера), `TABF1W` (справка о поступлении), `TABSTATJ` (статистика периодики), `NumbEkz`, `PeriodJ` |
| ПО КНИГОВЫДАЧЕ | `TSPROST` (спрашиваемость), `StatLand` (выдачи по месяцам), `ProvRDR_U` (контроль статуса U), `TReadHistory`, `TextHistory` |
| ДОПОЛНИТЕЛЬНЫЕ | `AVTS` (труды авторов), `COLL` (опись музейных коллекций) |

Списки (`BRIEF_TAB`), инвентарные (`TABINW`/`TABIUW`), КСУ-связанные формы — подтверждены наличием файлов `.tab`/`.pft`/`.srw`. Связка с пакетной печатью КК: формы `!!KK_*` используют модули `!!kk*` из `Pft.tre` (§1.5) через одноимённые `!!kk*.tab`.

### 3.6 Связь `.srw` (сортировка) с формой

`.srw` — секции `[HeaderNumber]` / `[KeyOptions]` (макс. длина ключа, флаг, формат построения ключа сортировки). `BRIEF_TAB.SRW`: ключ длиной 200 строится из автора/заглавия `&unifor('+S0', v503^a, v461^x, v700^a, …, v200^a)` (`IBIS/BRIEF_TAB.SRW`). `TABINW.SRW`: ключ длиной 15 из инв.№ 910^b с числовым выравниванием `f(val(…),8,0)` (`IBIS/TABINW.SRW`).

## 4. Статформы (`.stf` / STAT, связь с `IC_stat`)

### 4.1 Протокол `IC_stat`

`IC_stat(Adbn, Astat, Asexp; Amin,Amax; Aseq, Amfnlist…)` — `Astat` = задания статистики `FMT,N1,N2,N3`, где `FMT`=`поле^подполе`, `N1`=длина значения, `N2`=макс. число значений, `N3`=сортировка (0/1/2); выход — **RTF** (`PROTOCOL_REFERENCE.md:115`, txt:966-983).

### 4.2 Структура `.stf` (задание статистики)

`.stf` — текстовый файл (`IBIS/Form1.stf` … `Form5.stf`, 5 форм). Структура на примере `Form1.stf` («Ввод каталогизаторами данных по месяцам»):

| Строка | Содержимое | Назначение |
|---|---|---|
| 1 | `Form1` | имя задания |
| 2 | PFT-выражение по 907 (`&uf('+7W1#…')`, `&uf('Av907^A.4#',g1)…`) | формат 1-го параметра группировки (каталогизатор) |
| 3 | `FIO_SF.mnu` | справочник раскодировки для параметра 1 |
| 4 | PFT-выражение по 907^A (месяц) | формат 2-го параметра группировки |
| 5 | `MONTH_SF.mnu` | справочник для параметра 2 |
| 6 | `4,7` | длины значений параметров |
| 7 | `991.wss,1000` | рабочий лист параметров запроса + метка поля |
| 8 | `'Ввод каталогизаторами … за ', v1000^A,' г.'` | заголовок отчёта |
| 9 | `1,1` | флаги/опции выходных таблиц |
| 10–11 | заголовки распределений (по каталогизаторам / по месяцам) | заголовки осей сводной таблицы |

`Form3.stf` (виды документов по разделам знаний): группировка по 900^B (вид) и 60 (раздел) с фильтром по интервалу дат 907^A между v1000^A и v1000^B (`IBIS/Form3.stf`). Меню статформ — `stf.mnu` (`IBIS/stf.mnu`): Form1–Form5 с описаниями (ввод по месяцам / по видам / виды по разделам / разделы по месяцам / корректировка).

`Astat` протокола `IC_stat` (поле^подполе + длина + сорт.) соответствует параметрам группировки внутри `.stf` (строки 2–6): задание из `.stf` транслируется в `Astat`-строки запроса. Для табличных статформ с итогами (`TABSTATJ`) сводка реализована секцией `[Sum]` в `.tab`/`.tbu` через `&uf('+7W…')` (накопление в глоб. переменных) + `rsum` (`IBIS/TABSTATJ.tab` строки 4–33).
