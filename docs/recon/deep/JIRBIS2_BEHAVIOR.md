# Поведенческая спецификация веб-Читателя по мануалу J-ИРБИS 2.0

**Источник:** `docs/reference/ReadMe_jirbis2_text.txt` (5512 строк, UTF-8). Факты — со ссылкой `ReadMe_jirbis2 (txt:строка)`. Извлечено только относящееся к публичному веб-каталогу/Читателю. ПДн не приводятся.

> Референс ожидаемого поведения публичного OPAC для пилота СПб театральной библиотеки. J-ИРБИС 2.0 работает поверх ИРБИС TCP/IP-сервера **так же, как АРМ «Читатель»** `(txt:767)` — воспроизводим читательское поведение, не Joomla/PHP-стек.

## 1. Архитектура J-ИРБИС 2.0
Дистрибутив = XAMPP 1.6.7 (Apache 2.4.38 / MariaDB 10.1.38 / PHP 5.6.40) + Joomla 2.5 + ИРБИС-подсистема `com_irbis` (единственная несвободная часть) `(txt:695-703,515-521,573)`. Порты по умолчанию Apache 8087, MySQL 3307 `(txt:595,707,743)`.
**Ключевое:** ИРБИС-подсистема полностью независима от консольного WEB ИРБИС, работает с базами через TCP/IP-сервер как АРМ «Читатель» `(txt:767)` → наш бэкенд говорит по протоколу ИРБИС TCP/IP.
**Конфиг вывода:** профиль поисковых форм `(txt:579)`; профиль отображения записи (`jirbis_configuration.php`: массивы `formats_profile` элемент→PFT и `presentation_profile` где/при условии показать) `(txt:581,2492)`; форматы = PFT-файлы, переопределяемы по базам, хранятся в папке `Deposit` (без неё БЗ не отображаются) `(txt:587,623)`. Элемент `presentation_profile`: `Title`, `Indicators` (поля-условия, напр. 951→только если есть файлы), `Type` (0 в теле БО / 1 кнопка / 2 закладка), `Request` (`task=show_quick_format`), `Format` (`@имя.pft`) `(txt:2496-2500)`. Модули используют CSS-классы Joomla, но не API → есть режим «без Joomla» (§8).

## 2. Поиск
**Поиск по мере ввода** — на каждый символ, с индикатором прогресса `(txt:777)`.
| Режим | Поведение | Ссылка |
|---|---|---|
| Простой | только по ключевым словам (`K=`) | (txt:799) |
| Свободный | по всем полям (словарь «для чайников» `DS=`; в ИРБИС64+ не поддерживается → `K=`) | (txt:803-805,757) |
| Свободный с ранжированием | устойчив к «мусору»; **лимит 100 записей** | (txt:781-785) |
| Расширенный | типовая форма с полями | (txt:795) |
| Комплексный | произвольная комбинация 19 элементов (допускает 2 одинаковых) | (txt:889) |
| Экспертный/профессиональный | язык запросов ИРБИС: И/ИЛИ/НЕТ + «И В ПОВТОРЕНИИ»/«И РЯДОМ», усечение+морфология, подсказки словарей | (txt:893-895) |
| Полнотекстовый | по проиндексированным PDF; префикс `КТ=FT!`; одновременно по ПТ и БЗ (совпадение в БЗ → выше) | (txt:917-923) |

Нормализация: автор «тургенев иван» → «Тургенев, И.»/«Тургенев, Иван» `(txt:789)`; морфология рус/англ (корректные склонения vs усечение) `(txt:793)`; усечение настраивается чекбоксами `(txt:5102,5189)`.
**Префиксы:** `K=` ключевые слова, `KT=FT!` полнотекст, `T=` заглавия, `V=` вид документа `(txt:757,923,4694,1539)`.
**Фасеты** (ИРБИС64+): конкретизация по связанным авторам/кл.словам/типам (с количеством); выделяются лениво при раскрытии блока `(txt:927-945)`.
**Навигаторы:** ГРНТИ-дерево `(txt:807)`; новые поступления за период (виртуальный календарь) + разделы знаний (раздел = одновременно ББК+УДК+ГРНТИ) `(txt:857-861)`; MeSH-тезаурус (для театральной — нерелевантно) `(txt:905-913)`; авторитетные файлы ATHRA/ATHRC (разночтения автора 410/510/710) `(txt:899-903)`.

## 3. Результаты и запись
- Обложки динамически из интернета (по ISBN, затем по БО); в закрытой сети отключают `(txt:985-989)` → нужен кэш/прокси/graceful degradation.
- Пользователь сворачивает/раскрывает оглавление/аннотацию/кл.слова; настройки в cookie ~год `(txt:993,1005)`. Объёмные сведения — на закладках «Экземпляры»/«Электронные версии»/«Сиглы» `(txt:995)`.
- Форматы: Краткий/Полный + профили (museum/person/reader) `(txt:1033,1051)`; выбор «Формат показа» `(txt:2468)`.
- Сортировка (`sort_chose`) **сильно замедляет поиск** — предупреждение о перегрузках `(txt:1980)` → у нас сортировка/пагинация на индексе. `TODO(irbis-web #02: дефолтный размер страницы обычного поиска в тексте не задан.)`
- Переходы: обложка→оригинал, БО→полная запись; гиперссылки в БО сохраняют выбранные БД; статьи содержания — поле 922 `(txt:879,5139,5118)`.
- Отбор: чекбоксы (`checkbox_enable`), номера (`number_enable`), модуль «Корзина» `(txt:1986,2426)`; печать/экспорт в MS Word, формат краткий/полный, сохранение отмеченных из нескольких запросов, email-рассылка списков `(txt:1317-1325,1975-1979)`.
- WEB 2.0: рейтинг 5-балльный, комментарии с модерацией, Top-10 `(txt:1009-1025,979)`; отключаемо `(txt:3725-3727)`.

## 4. Полные тексты / ресурсы
- Связь файлов: поле **951** (`^A` относит. путь / `^I` URL — взаимоисключающи) и **955** (для полнотекста); префикс `ed_path`, папка = 11-й параметр PAR `(txt:1185-1209,763,1089,1193)`.
- Онлайн-просмотр: PDF постранично (конвертация «на лету», стиль Google Books), MP3/MP4; прокрутка/масштаб/копирование/гиперссылки; не гарантируется для всех PDF (`deny.jpg`) `(txt:1103-1113,1165)`. Закладки (cookie, доступны и неавторизованным) `(txt:1119-1125)`; оглавление PDF (2018.1+) `(txt:1129-1133)`; подсветка терминов (`ji_highlight`), миниатюры релевантных страниц `(txt:1139-1179)`.
- **Права доступа (центрально для OPAC)** `(txt:1213-1261)`: глобальные правила (`ed_pdf_view_only_enable`/`ed_pdf_view_enable`/`ed_media_view_only_enable`/`ed_pdf_view_text_enable`), разные для LAN/интернета (`ed_external_same`); на файл — подполя 951 (`^D` категории, `^L` только LAN, `^9` флаги: выгрузка/просмотр/проигрывание/копирование/печать); псевдокатегория «авторизованный» в `951^D` `(txt:4714)`; **БД RIGHT** (подполе 955: IP/категория/специальность/периоды; приоритет 951 над 955) `(txt:1243)`; ⚠ не все права RIGHT поддерживаются J-ИРБИС и наоборот, RIGHT не контролирует копирование в буфер `(txt:1245-1247)`; LAN по маске `ed_local_users_mask`, причина недоступности `ed_unavalability_explaining` `(txt:1255-1259)`; лимит числа/процента страниц — в RIGHT `(txt:1157-1159)`.
- Два метода выгрузки: `ed_path_type=prf` (по локальному префиксу, быстро) / `std` (через TCP-сервер, лимит ~10 МБ, бинарь не пишется) `(txt:1263-1271)`. HTTP/FTP-ссылки отдаются напрямую — защита низкая `(txt:1211)` → проксировать через бэкенд.

## 5. Личный кабинет читателя
- **Вход:** сквозная авторизация на базе **RDR** (по фамилии + идентификатор поле 30; поле 130 пароль по умолчанию не используется; настраивается `user_password_tag`, можно поле 24); MD5 без соли (`password_md5`); связь категорий RDR↔уровней Joomla; запрет авторизации должников `(txt:1293-1311,761,4704,5488)`.
- **Содержимое ЛК:** читательский формуляр; закладка «Заказы» (отслеживание, удаление до начала выполнения); «Профиль» (электронные версии, выполненные заказы ЭДД); продление онлайн (`debts_prolongation_enable`, times×days); правка ПДн в RDR (`reader_rec_edit_enable`, по умолч. выкл); постоянные запросы (повтор формулировки; ИРБИС64+) `(txt:1351-1393,1343-1347)`.
- **Регистрация/пароль:** онлайн-анкета, 3 сценария (выдача пароля / предрегистрация / полная регистрация); combobox+автодополнение из RDR `(txt:1483-1503)`.
- **Сквозные токен-ссылки ЭИОС:** `?token=...&p=...&bl=...&code=...`; `token`=md5(идентификатор+YYMMDD+секрет), срок 1 сутки `(txt:1721-1751)`.
> «ИРИ» как термин в мануале нет; аналог — постоянные запросы. `TODO(irbis-web #03: нужна ли пилоту авторассылка по постоянным запросам.)`

## 6. Заказ книги
- Предзаказ на закладке БЗ; для авторизованного — кнопка «Заказать» одним щелчком `(txt:999)`; постановка в очередь на занятую (`order_queue_enable`, показ размера очереди) `(txt:1337)`; лимит `order_max_count` `(txt:5305)`.
- Места хранения/правила — таблица «Места хранения и выдачи» (аббревиатура/название МХ, кафедра выдачи, комментарий, флаг «предзаказ доступен») `(txt:3673-3693)`.
- Жизненный цикл: оформил → ЛК «Заказы» → удалить до выполнения → сотрудник меняет статус → в «Профиль»; авто-письма при смене статуса `(txt:1353,1707,1713)`.
- ЭДД (опц. для пилота): заказ скана, диапазон страниц, отслеживание, привязка файла к читателю и БЗ, правила доступа (ст.1275 ГК) `(txt:1701-1717)`.

## 7. Настройка / локализация / доступность
- Параметры `com_irbis` (почти всё отключаемо): `search_form_enable`, `req_description_enable`, `print_*`, `sort_chose`, `formats_chose`, `header`, `cover_enable/width(152)/height(196)`, `checkbox_enable`, `number_enable`; можно передавать в URL (приоритет над меню) `(txt:1968-1989)`.
- Локализация: UTF-8, поиск на языках СНГ; но **переключение языков библиотечной подсистемы НЕ реализовано**; ручной перевод (строки в `jirbis.js`/`*.pft`/`*.mnu`) ломает автообновление `(txt:4357-4375)` → у нас i18n как базовое.
- **Доступность: версии для слабовидящих / ГОСТ Р 52872 / WCAG в J-ИРБИС НЕТ** `(txt:1573)`. Есть лишь адаптив/киоск (Gavick) `(txt:949-973)`. `TODO(irbis-web #05: accessibility реализовать с нуля — обязательное требование гос. OPAC, референса в J-ИРБИС нет.)`

## 8. Замечания для веб-замены
**Устаревший стек:** PHP 5.6/Joomla 2.5/MariaDB 10.1/Apache 2.4.38 — всё EOL `(txt:695-709,4718)`; нестандартные порты, конфликты при установке `(txt:595,749)`; конфиг = ручная правка PHP `(txt:2482-2486)` → замена платформы обязательна, конфиг декларативный.
**Дефекты/ограничения:** нет i18n библиотечной части и accessibility; сортировка тормозит поиск `(txt:1980)`; ранжированный поиск лимит 100 `(txt:785)`; PDF-просмотр не для всех PDF, может висеть >5 мин `(txt:1113-1153)`; HTTP-ссылки отдаются напрямую `(txt:1211)`; часть прав RIGHT не поддержана `(txt:1245)`; фасеты/постоянные запросы/закладки PDF — только ИРБИС64+ `(txt:929-1153)`; обложки из интернета `(txt:985-989)`; история версий фиксирует баги именно в читательских сценариях (предзаказ сверх лимита `(txt:5305)`, «Корзина» `(txt:5130)`, продление `(txt:4940)`) → покрыть тестами.
**Сохранить как поведение:** поиск-по-вводу с прогрессом; автонормализация+морфология; ленивые фасеты; сворачиваемые элементы записи (профили отображения); заказ «в один клик»; очередь на занятую; продление в ЛК; постраничный PDF с подсветкой; токен-ссылки ЭИОС.
**Отвязка от Joomla:** вендор предусмотрел режим «с полным отказом от Joomla» `(txt:4710,171-173)` → читательскую логику реализуем независимо от CMS.
