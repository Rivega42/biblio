# Схема экранов веб-Читателя (каталог СПб ГТБ, пилот P0)

> Веб-интерфейс организован **по функциям и грантам** (не по «АРМам»). Настольный клиент «Читатель» —
> лишь **источник инвентаря функций** (находка #3, мастер-карта §6.1); здесь его режимы привязаны к
> веб-экранам: маршруты, компоненты, вызовы API (`P0_BUILDKIT` §7), требуемые гранты (`ACCESS_MODEL`).
> Принцип — **структурный 1:1**: тот же состав функций, но веб-нативно и с доступностью по
> ГОСТ Р 52872-2019. Учитывать замечания J-ИРБИС из `FINAL_TZ` (гость без персонального и др.).
> Версия 1.0, 20.06.2026.

---

## 1. Принцип переноса

Настольный клиент «Читатель» (источник функций) имел 4 рабочие области + меню-режимы. В вебе они становятся функциями, доступными по грантам:
- 4 области → не 4 окна, а **связанные зоны одного интерфейса** (поиск/словарь → результаты → запись).
- Меню-режимы → маршруты/действия, доступные по контексту и гранту.
- Десктоп-специфику (MDI, модальные окна) не копируем буквально — заменяем веб-паттернами (страницы,
  панели, оверлеи), сохраняя состав и логику.

## 2. Оболочка и состояния доступа

**Layout:** шапка (лого СПб ГТБ · выбор базы · строка поиска · вход/аккаунт) → контент → подвал.
**Состояния:** гость · читатель (вход по билету) · сотрудник (логин). Навигация и доступные действия
зависят от состояния и грантов. ⚠️ **Гость — без персонального** (нет ЛК/корзины/закладок) — замечание J-ИРБИС.

## 3. Карта экранов

| Маршрут | Экран | Исходный режим (наст. клиент) | Ключевые компоненты | API | Грант |
| --- | --- | --- | --- | --- | --- |
| `/` | Стартовый поиск | Вид основного поиска | выбор базы, строка поиска, переключатель режима | — | — |
| `/search/simple` | Простой поиск | «Для чайников» | одно поле, автоподсказки из словаря | `GET /terms`, `GET /search` | search:db:read |
| `/search/advanced` | Расширенный | «Для умников» | поле + префикс + усечение + логика | `GET /search` | search:db:read |
| `/search/fields` | По нескольким полям | Комплексный | строки полей (префикс+значение+И/ИЛИ/НЕ) | `GET /search` | search:db:read |
| `/search/sequential` | Уточнение в результате | Последовательный | доп. условие поверх текущего результата | `GET /search` (searchscan) | search:db:read |
| `/results` | Список результатов | Короткие описания + Просмотр | краткое БО (серверный PFT), сортировка, отметки, пагинация, смена формата | `GET /search` | search:db:read |
| `/record/:db/:mfn` | Полное описание | Полное описание + Связанные | полное БО, **связанные документы** (GUAR Фонд↔Опись), кнопка «Заказать», просмотр файла | `GET /record`, `GET /file` | record.read:db:read |
| `/dictionary` | Словарь/рубрикатор | Область СЛОВАРЬ | просмотр терминов, рубрикатор (ГРНТИ), отбор термина в запрос | `GET /terms`, `GET /posting` | search:db:read |
| `/cabinet` | Личный кабинет | (Web-модель ЛК) | формуляр, история (бумажная/электронная), закладки, корзина, постоянные запросы | `GET /me/cabinet` | читатель (своя RDR) |
| `/login` | Вход | БД→Регистрация | вкладки: читатель(билет) · сотрудник · гость | `POST /auth/*` | публично |

**Действия-оверлеи (не отдельные страницы):**
- **Заказ** — подтверждение заказа (модель Web-ИРБИС `order_form`): `POST /order` (грант order:db:write).
- **Просмотр полного текста** — постраничный вьюер (оверлей): `GET /file` (+ проверка RIGHT).
- **Печать / Экспорт** — над текущим результатом/отмеченными: серверные формы (`IC_print`) / экспорт.

## 4. Рабочие области → веб-композиция

| Область наст. клиента | В вебе |
| --- | --- |
| СЛОВАРЬ | панель/экран `/dictionary` + автоподсказки в полях поиска |
| ТЕКУЩИЙ ЗАПРОС | строка/панель активного запроса над результатами (редактируемая, история) |
| КОРОТКИЕ ОПИСАНИЯ | список `/results` (краткий серверный PFT) |
| ПОЛНОЕ ОПИСАНИЕ | страница `/record/:db/:mfn` (полный серверный PFT) |

## 5. Режимы поиска (десктоп → веб)

| Десктоп | Веб-название (предложение) | Суть |
| --- | --- | --- |
| Для чайников | Простой | одно поле, авто-префикс/морфология |
| Для умников | Расширенный | префикс + усечение + логика вручную |
| Комплексный | По нескольким полям | набор полей с И/ИЛИ/НЕ |
| Последовательный | Уточнение в результате | фильтр поверх текущего набора |
| По связи | По связанным | переход по ссылкам (GUAR/связанные) |
| Выполненные / Постоянные | История / Сохранённые запросы | в ЛК (читатель) |

> Финальные веб-названия — согласовать с `FINAL_TZ` §6/§16 (там зафиксированы наименования и замечания).

## 6. Ключевые потоки

- **Поиск → результат → запись → заказ:** `/search/*` → `/results` → `/record` → оверлей «Заказать»
  (`POST /order`). Гостю кнопка заказа недоступна.
- **Полный текст:** на `/record` ссылка «Постраничный просмотр» → оверлей-вьюер (`GET /file`), доступ
  по правам RIGHT; «электронная книговыдача» фиксируется в истории ЛК.
- **Путешествие по фонду (GUAR):** на `/record` блок «Связанные документы» — переходы вверх/вниз по
  иерархии Фонд↔Опись↔Дело↔Документ↔Лист (поле 488).

## 7. Доступность и замечания J-ИРБИС

- ГОСТ Р 52872-2019: семантическая разметка, навигация с клавиатуры, контраст, альт-тексты, ARIA для
  словаря/результатов/оверлеев; вьюер полного текста — доступный.
- Замечания J-ИРБИС (`FINAL_TZ`): гость без ЛК/корзины; корректные связанные переходы; явные виды
  поиска и префиксы; устойчивая пагинация и сортировка. Сверить полный список десяти замечаний по `FINAL_TZ`.
