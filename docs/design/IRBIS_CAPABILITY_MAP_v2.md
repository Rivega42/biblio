## 0. Цель и принцип
Полная ВЕБ-версия ИРБИС64 (все функции, организация по учёткам/грантам); частный случай — каталог СПб ГТБ (`irbis-web`) = веб-Читатель +
часть Книговыдачи (пилот). Вендора нет → **демо-поставка = действующая спецификация**, ресёрч = её
восстановление. Снятый риск: клиент протокола из BookCabinet + официальная дока API (`irbis64_client_dll.doc`).

## 1. Среда / установка
ИРБИС64+ **2023** (наследует 2018.1: единый функционал, авторизация по учётке Windows), «для дилеров»,
3 клиента. Сервер `192.168.8.19:6666`; `DataPath=C:\IRBIS64\DATAI\`; реестры баз `dbnam1.mnu`
(Каталогизатор) / `dbnam3.mnu` (Читатель); список клиентов/прав `client_m.mnu`. АРМ-exe:
`cirbisa/c/b/k/p_plus.exe`. Конфиг подключения клиента (ключи INI; значения — env):
`ServerIP/ServerPort/UserName/UserPassWord/DBN/WebServer`.

## 2. Реестр баз
51 база в DATAI (полный список — #1/#4). Профильные: `IBIS` (пилот), `GUAR`+`GURF`+`GUSK` (архив,
Фонд↔Опись), `IMAGE` (имидж). Обслуживание: `RDR/RDRV/RDR_ARH`, `RQST` (заказы), `PODB`, `RIGHT`
(права к ПТ), `LICH` (ПДн), `IRI/SPRAV/ZAPR`. Словари: `ATHRA/B/C/G/S/U`, `RSBBK/RSUDC`, `HELP`,
`MESH`, `TEZ`, `MORPH`, `URUB`. Комплект/КО: `CMPL/POST/PAY/VUZ/WORK/COUNT/MBA(_ARH)`; логи
`LOGB/LOGC/LOGP`. ⚠️ Боевых PLAY/TUAR/EK/ESKIZ/HPO в демо нет — с сервера СПб ГТБ.

## 3. Карта протокола (ядро веб-версии)
Из `irbis64_client_dll.doc`. Тип АРМа при `IC_reg` = гейт прав (ошибка `-3338`): `IRBIS_READER/
CATALOG/COMPLECT/BOOKLAND/ADMINISTRATOR/BOOKPROVD`.
- **Сессия:** reg, unreg, set_client_time_live, set_webserver/webcgi, isbusy, nooperation.
- **Ресурсы/файлы:** getresourse (текст), getbinaryresourse (двоичный — 951/955), putresourse
  (только текст), getresoursegroup, update_ini, clearresourse.
- **Записи:** read, readformat (+@PFT), update/updategroup/updategroup_sinhronize, runlock, ifupdate, maxmfn.
- **Поле записи:** field(n), fldadd/fldrep, nfields/nocc/fldtag, fldempty, changemfn, recdel/recundel,
  recunlock, getmfn, recdummy, isActualized/isLocked/isDeleted.
- **Словарь:** nexttrm/prevtrm(+group), posting/postinggroup/postingformat, getposting.
- **Поиск:** search (по словарю, выражение+формат+пагинация), searchscan (последовательный).
- **Форматирование:** sformat (по MFN), record_sformat, sformatgroup.
- **Пакетно:** print (табличные формы), stat (распределения), gbl (глобальная корректировка).
- **Админ:** adm_restartserver, adm_getdeletedlist(s), adm_dbempty/dbdelete/newdb, adm_dbunlock(mfn),
  adm_dbstartcreatedictionry/reorgdictionry/reorgmaster, adm_getclientlist/clientslist/processlist/setclientslist.

## 5. Связи GUAR (Фонд↔Опись) — поле 488
Иерархия `Фонд→Опись→Дело→Документ→Лист` (РЛ Fond/Opis/Delo/Guar/List). **Связь = поле 488**:
`^t` цель, `^1` уровень цели, `^c` название, `^j/^g` шифр. Собственный уровень записи — `v920`.
Поля 461/462/463 — ссылки вверх; индекс `II=` — переходы; шифр 852 (`NF=`/`Nehr=`); ССО — 327/545.
Рубрикаторы GUSK/GURF (как HELP).

## 6. Инвентарь функций (источник — настольные клиенты; для веб-копии «структурно 1:1»)
- **Читатель:** области СЛОВАРЬ/ТЕКУЩИЙ ЗАПРОС/КОРОТКИЕ/ПОЛНОЕ; меню БАЗА·ПОИСК(Для чайников/умников/
  Комплексный/Последовательный/По связи/Выполненные/Постоянные)·ПРОСМОТР(заказ, сортировка, переход,
  отметки, связанные)·СЕРВИС(печать, экспорт, ретрофонд, рекомендованная, настройка)·ПОМОЩЬ.
- **Каталогизатор:** области СЛОВАРЬ/РАБОЧИЙ ЛИСТ; способы ввода (меню/словарь/вложенный РЛ/Рубрикатор/
  Авторитетные/Навигатор); страницы РЛ: Дублетность·Коды·Основное БО·Расширенное БО·Специфические·
  Экземпляры·Технология·Систематизация·Содержание·КО·Редкие·Краеведение. Правила полей — irbiscat.doc разд.4.
- **Книговыдача:** области СЛОВАРЬ/ЧИТАТЕЛЬ/ЗАКАЗЫ; ЗАКАЗЫ(отбор/переадресация/отказ/корректировка/
  выполненные); ЧИТАТЕЛИ(возврат/выдача без заказа/групповая/без ЭК/многоэкз./продлить/посещение/
  перерегистрация/право обслуживания); СЕРВИС(должники/штрих-коды/бронеполка/RFID/статистика). БД RQST/RDR.
- **Комплектатор:** Поступление→КСУ→поиск в каталоге→перенос в БД комплектования→доукомплектование→ЭК;
  многотомники/журналы/утерянные/размножение экз.; выходные формы (инвентарный список, КСУ, движение фонда). CMPL/POST.
- **Книгообеспеченность:** окна основные/связанные/перенос; БД VUZ (дисциплины/контингенты/учебный план/
  импорт), связь с каталогом и студентами; стат./табличные формы (WORK).
- **Администратор:** Серверный (база: новая/опустошить/удалить/импорт/экспорт; актуализация/реорг
  словаря и файла документов; диагностика/восстановление; пакетные задания OpenDB/NewDB/Import/Export/
  SEARCH/GLOBAL/PRINT/STAT; редактор ISO/MST; глобальная корректировка; log) + Клиентский (то же по
  протоколу `IC_adm_*`).

## 7. Веб-модель вендора (Web-ИРБИС64+) — эталон
CGI-шлюз + форм-файлы `*.frm` + `irbis_server_ft.ini`; `Deposit_plus` (pft/mnu). Команды `C21COM=`:
`F`(фреймы)/`S`(поиск)/`T`(словарь)/`Z`(заказ)/`R`(запись). Параметры: `I21DBN`(профиль)/`P21DBN`(БД)/
`Z21ID/Z21FAMILY`(логин/пароль)/`S21FMT/S21REF/S21STN`; поле = `_S21STR`+`_S21P03`(префикс)+`_S21LOG`+
`_S21P01/P02`. Префиксы: `A=/T=/G=/V=/K=/KT=/COLLT=` (R=/DP= уточнить). ЛК `author_3.frm`: история
бумажных/электронных выдач (интервал 24ч, лимит 10), закладки, корзина; заказ `order_form.frm`+`zakaz.pft`.
Гость = RDR-запись с полями 30/50=`GUEST`. Полный текст — всплывающий постраничный просмотр.

## 8. Архитектура веб-клиента (вывод)
Ядро (протокол-клиент BookCabinet + сессия + поиск + словарь + чтение/формат + ресурсы/файлы) — один
раз. Группы функций — модули поверх (приоритет/риск): обслуживание читателя→книговыдача→каталогизация→комплектование→
книгообеспеченность→администрирование. Загрузка бинарных файлов — не через протокол (нужен файловый доступ).
Безопасность: контур, роли, аудит для write/admin; секреты в env; секрет-скан в CI.
