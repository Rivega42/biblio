# Функция → команда протокола (привязка доступа к IC_*)

> Сводит каталог функций (`ACCESS_MODEL_web-irbis.md` §3) с конкретными командами протокола
> (`irbis64_client_dll_TEXT.txt`). Для каждой функции: команды `IC_*` + требуемый АРМ-контекст
> (тип при `IC_reg`; иначе `-3338`). Это слой, который бэкенд вызывает после проверки гранта учётки.
> Версия 1.0, 20.06.2026.

---

## 0. Сессия (под любой функцией)
`IC_reg` (host, **тип АРМа**, служебный user) → работа → `IC_unreg`; keepalive `IC_nooperation`.
Тип АРМа выбирается по требуемому контексту функции (ниже). Сессии — в пуле по типам АРМ.

## 1. Каталог / поиск (контекст READER)
| Функция | Команды IC_* |
| --- | --- |
| Поиск по словарю | `IC_search` (выражение+формат+пагинация) |
| Последовательный/доп. поиск | `IC_searchscan` |
| Просмотр записи | `IC_read` / `IC_readformat` (с `@PFT`) |
| Отображение (серверный PFT) | `IC_sformat` (по MFN) / `IC_sformatgroup` / `IC_record_sformat` |
| Автодополнение/словарь/термины | `IC_nexttrm`/`IC_prevtrm` (+`group`), `IC_posting`/`IC_postinggroup` |
| Просмотр файла/полного текста | `IC_getbinaryresourse` (двоичный) / `IC_getresourse` (текст) + проверка `RIGHT` |
| Экспорт | `IC_sformatgroup` / групповое `IC_read` |
| Печать / табличные формы | `IC_print` |
| Статистика | `IC_stat` |

## 2. Читательские сервисы (контекст READER)
| Функция | Команды / источник |
| --- | --- |
| Оформить заказ | `IC_update` в БД `RQST` (запись заказа); корзина — на стороне приложения (PG) |
| ЛК / формуляр | `IC_read`/`IC_search` по `RDR` (своя запись); история — формат типа `show_history` |
| Закладки / постоянные запросы | приложение (PG) + `IC_search` для выполнения |

## 3. Книговыдача (контекст BOOKLAND)
| Функция | Команды |
| --- | --- |
| Очередь заказов (отбор/отказ/переадресация) | `IC_search` по `RQST` + `IC_update` (статус заказа) |
| Выдача / возврат / продление | `IC_read(lock)` + правка полей экземпляра (910) и читателя → `IC_update` (RDR/каталог) |
| Регистрация/корректировка читателя | `IC_update` по `RDR`/`RDRV` |
| Должники / бронеполка / статистика | `IC_search` / `IC_stat` / `IC_print` |

## 4. Каталогизация (контекст CATALOG)
| Функция | Команды |
| --- | --- |
| Создать запись | `IC_recdummy` (пустая) → правка полей → `IC_update` |
| Редактировать запись | `IC_read(lock)` → `IC_fldadd`/`IC_fldrep`/`IC_field(n)`/`IC_nocc`/`IC_fldtag` → `IC_update` |
| Удалить (логически/физически) | `IC_recdel`/`IC_recundel` → `IC_update`; разблокировка `IC_recunlock`/`IC_runlock` |
| Связанная правка (Фонд↔Опись и пр.) | `IC_updategroup` / `IC_updategroup_sinhronize` |
| Глобальная корректировка | `IC_gbl` |
| Статусы записи | `IC_isActualized`/`IC_isLocked`/`IC_isDeleted`; `IC_maxmfn` |

## 5. Комплектование (контекст COMPLECT)
| Функция | Команды |

## 8. Замечания по реализации
- **Ресурсы базы** (INI/PFT/MNU/FST на сервере) при необходимости — через `IC_getresourse`/
  `IC_getresoursegroup`; обновление профиля — `IC_update_ini`; кэш — `IC_clearresourse`.
- **Запись бинарных файлов** через протокол невозможна (`IC_putresourse` — только текст) → загрузка
  сканов/PDF требует файлового доступа к хранилищу, не протокола.
- **Буферы**: многие `IC_*` принимают размер буфера ответа — на бэкенде задавать с запасом и
  обрабатывать переполнение (повторный вызов/увеличение).
- **Коды результата** — проверять у каждого вызова; `-3338` = нет прав АРМа (грант/контекст не совпал);
  прочие — нормализовать в единый конверт ошибок (детали в лог, не клиенту).
- Точные форматы пакетов/коды — сверять с клиентом BookCabinet и при Проходе Б на живом сервере.
