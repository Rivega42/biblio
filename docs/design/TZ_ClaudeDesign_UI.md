# ТЗ для ClaudeDesign — пользовательско-клиентская часть веб-ИРБИС (`irbis-web`)

> Задание на проектирование **дизайна и UI/UX** клиентской части. Цель: спроектировать современный, доступный, единый веб-интерфейс библиотечной системы (замена ИРБИС64+ и J-ИРБИС). Опирается на: `docs/design/IRBIS_CAPABILITY_MAP_v2.md`, `ACCESS_MODEL_web-irbis.md`, `SCREENMAP_web-reader.md`, `SCREENMAP_web-staff.md`, `WHY_REPLACE_IRBIS.md`, и доказательную базу `docs/recon/deep/` (поведение J-ИРБИС, поля/словари, экраны). Пилот: каталог СПб театральной библиотеки.

## 1. Контекст и цель
Заменить 6 десктоп-АРМов + устаревший web (J-ИРБИС на EOL-стеке) **единым веб-приложением**. Организация — **по учёткам и грантам** (функция × база × уровень), НЕ «по АРМам». «АРМ» — лишь технический контекст протокола, в UI его нет. Принципы — `WHY_REPLACE_IRBIS.md` (что делаем лучше: доступность, безопасность, простота, скорость).

## 2. Пользователи и роли (экраны собираются по грантам)
- **Гость** — публичный каталог: поиск, просмотр записи, доступ к открытым ресурсам. Без ЛК/корзины.
- **Читатель** (вход по билету, БД RDR) — каталог + личный кабинет (формуляр, история, закладки, заказ, продление, постоянные запросы, уведомления, оплата).
- **Сотрудник** — набор функций по грантам (может совмещать): каталогизация, книговыдача, комплектование, книгообеспеченность, инвентаризация, аналитика, администрирование. Интерфейс показывает **только разрешённое**.
> Источник инвентаря функций — `ACCESS_MODEL §3` (домены: Каталог/Читательские/Книговыдача/Каталогизация/Комплектование/Книгообеспеченность/Администрирование). Уровни: чтение/запись/удаление/админ.

## 3. Информационная архитектура и навигация
- Навигация **динамическая по грантам**: показываем доступные функции как «меню задач», а не 5 отдельных программ.
- Три контекста входа: гость / читатель / сотрудник. Единый поиск — всегда доступен.
- Глубокие ссылки на запись/поиск (SEO, sharable URL); сохранение состояния поиска при переходах по связям.

## 4. Экраны — Читатель/публичный каталог (приоритет P0)
Поведенческий эталон — `docs/recon/deep/JIRBIS2_BEHAVIOR.md` (что хорошо — сохранить; что плохо — не повторять).
- **Поиск:** строка с **поиском по мере ввода** + индикатор прогресса; режимы: простой / расширенный (форма по полям) / профессиональный (для опытных); морфология; усечение; **фасеты** (ленивые, с количеством); навигаторы (ГРНТИ/УДК/ББК-дерево). Подсказки из словарей (combobox с автодополнением).
- **Результаты:** список с обложками (кэш/прокси, graceful degradation), краткое БО, пагинация, сортировка (на индексе, быстрая), чекбоксы отбора, корзина.
- **Карточка записи:** полное БО; сворачиваемые блоки (оглавление/аннотация/кл.слова); закладки «Экземпляры»/«Электронные версии»/«Сиглы»; переходы по связям (журнал→номер→статья); кнопки экспорт/печать/поделиться.
- **Полные тексты/медиа:** постраничный просмотр PDF с подсветкой терминов и навигацией по релевантным страницам; права доступа (показать причину недоступности); **никаких прямых ссылок** — через прокси.
- **Личный кабинет:** формуляр (на руках/сроки), история (с приватностью), закладки, **заказ в один клик** + постановка в очередь, продление, постоянные запросы, **уведомления**, **онлайн-оплата** штрафов.
- **Заказ книги:** выбор места выдачи/ячейки, статус заказа, очередь на занятую.

## 5. Экраны — Сотрудник (P1+)
Эталон структуры — `SCREENMAP_web-staff.md` + `docs/recon/deep/reference/arms/*`.
- **Каталогизация:** поиск/список записей; **рабочий лист ввода** (страницы/поля) по профилю базы; редактор поля с типами ввода (см. §6); валидация (ФЛК) с понятными сообщениями; история правок/откат; глобальная корректировка; импорт (copy-cataloging в один клик).
- **Книговыдача:** запись читателя; выдача/возврат/продление (сканер/RFID); очередь заказов; бронеполка; **ячеистое хранение** (адрес ячейки); должники; штрих-/QR-печать.
- **Комплектование:** заказ→поступление→КСУ→списание; мастера; выходные формы.
- **Книгообеспеченность:** дисциплины/контингенты; расчёт ККО; отчёты.
- **Инвентаризация:** **онлайн с ТСД/сканера** — мгновенная сверка, прогресс, отчёт расхождений.
- **Аналитика:** BI-дашборды (книговыдача/фонд/комплектование/читатели).
- **Администрирование:** учётки/гранты/роли, базы, аудит (строгий контур).

## 6. Дизайн-система и ключевые компоненты
- **Динамические формы ввода** — главное. Тип поля определяет контрол (из `docs/recon/deep/reference/format/FIELD_CATALOG.csv`, режимы ввода): свободный текст / выбор из меню (`.mnu`) / ввод по словарю с автодополнением (префикс) / иерархический справочник (дерево `.tre`) / да-нет / ввод через авторитетный файл / дата. Подполя — вложенные группы. Повторяемые поля — добавление/удаление повторений. Подсказки и валидация инлайн.
- **Списки/таблицы результатов** — плотные, сортируемые, с массовыми операциями (отметить/корзина/печать), пагинация.
- **Карточки** записи/читателя/экземпляра — со сворачиваемыми блоками и закладками.
- **Навигация по грантам** — адаптивное меню задач.
- **Состояния:** загрузка, ошибка (единый конверт, детали — в лог, не пользователю), пусто, нет прав (403 без раскрытия деталей).
- **Справочники** как seed — значения из `CONTROL_VOCABULARIES.csv` (статусы экземпляра, виды документов, языки, страны и т.д.) для выпадающих списков и расшифровки кодов.

## 7. Доступность (ОБЯЗАТЕЛЬНО)
**ГОСТ Р 52872-2019 + WCAG 2.1 AA.** Семантическая разметка, полная навигация с клавиатуры, поддержка скринридеров (ARIA), контраст, масштабирование, версия для слабовидящих (в J-ИРБИС этого НЕТ — реализуем как базовое). Доступность проверять автоматически в CI.

## 8. Адаптивность и устройства
- Десктоп (рабочие места сотрудников), планшет (зал), мобайл (читатель).
- **Режим ТСД** (Android-терминал) — крупные элементы, быстрый ввод скана, офлайн-индикатор, синхронизация. См. эпик периферии (#47).
- Сенсорный/киоск-режим (самообслуживание).

## 9. Бренд, тема, локализация
- **Нейтральная настраиваемая тема** (цвет/лого/типографика) под конкретную библиотеку — без правки кода (в отличие от J-ИРБИС, где конфиг = правка PHP). Светлая/тёмная.
- **i18n** — мультиязычный интерфейс (рус как базовый), готовность к языкам СНГ; контент (БО) может быть многоязычным. RTL-готовность.

## 10. Поведенческие требования (взять у J-ИРБИС хорошее)
Поиск-по-вводу с прогрессом; автонормализация автора и морфология; ленивые фасеты; сворачиваемые элементы записи (профили отображения); заказ «в один клик»; очередь на занятую; продление в ЛК; постраничный PDF с подсветкой; токен-ссылки ЭИОС. (Источник — `JIRBIS2_BEHAVIOR §2-6`.)

## 11. Нефункциональные
- **Скорость:** быстрый отклик поиска и пагинации (без деградации на сортировке — болезнь ИРБИС); скелетоны при загрузке.
- **Безопасность UI:** секреты не в клиенте; файлы через прокси с проверкой прав; защита от утечки ПД; аудит действий.
- **Отказоустойчивость UI:** работа при недоступности части сервисов (degraded mode), понятные ошибки.

## 12. Антипаттерны — НЕ копировать из ИРБИС/J-ИРБИС
- Деление UI «по АРМам». → по задачам/грантам.
- Эзотерика языка запросов/PFT в лицо пользователю. → простой UX + опц. экспертный режим.
- Правка конфигурации = правка кода/PHP. → декларативные настройки в UI.
- Прямые ссылки на файлы; пароли/креды в интерфейсе/конфиге. → прокси + env.
- Отсутствие доступности и локализации. → обязательны.
- Сортировка, роняющая поиск; лимит 100 на ранжированный поиск. → индекс справляется.

## 13. Что нужно от ClaudeDesign (ожидаемые артефакты)
1. **Дизайн-система:** палитра (настраиваемая тема), типографика, сетка, состояния, иконографика; библиотека компонентов (формы/поля по §6, таблицы, карточки, навигация, модальные, тосты).
2. **Макеты ключевых экранов** (десктоп + мобайл + ТСД):
   - Читатель: главная/поиск, результаты, карточка записи, просмотр PDF, личный кабинет, заказ/оплата, вход/регистрация.
   - Сотрудник: рабочий стол по грантам, рабочий лист каталогизации (динамическая форма), книговыдача, инвентаризация с ТСД, дашборд.
3. **Потоки (flows):** поиск→заказ→получение; выдача/возврат; каталогизация записи; инвентаризация; вход читателя/сотрудника.
4. **Спецификации компонентов** (props/состояния/доступность) — пригодные для React+TS+Tailwind+shadcn/ui.
5. **Прототип** ключевых сценариев (кликабельный), с проверкой доступности.

## 15. Объём пилота (P0) vs дальше
**P0 (читатель, СПб ГТБ):** поиск/результаты/карточка/PDF/вход/ЛК/заказ — десктоп+мобайл, доступность. **P1+:** экраны сотрудника (каталогизация, книговыдача, инвентаризация, дашборды). Дизайн-систему делаем сразу единой (расширяемой под P1+).
