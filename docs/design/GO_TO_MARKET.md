# Политика продаж (go-to-market) — библиотечная платформа

> Коммерческая модель продукта (см. [ARCHITECTURE](ARCHITECTURE.md): мультиарендный SaaS + edge/офлайн + модули). Черновик для проработки; цифры — ориентиры, уточняются.

## 1. Сегменты (рынок РФ, старт — СПб)
| Сегмент | Объём (ориентир) | Особенности | Что важно |
|---|---|---|---|
| **Публичные** (городские/районные, сети) | СПб ~200; по РФ — тысячи | сети с десятками филиалов; разный уровень ИТ | мультиплощадка, простота, цена, импортозамещение |
| **Школьные** | очень много | тонкий функционал, малый бюджет, часто без ИТ-службы | дёшево, облако, минимум поддержки, учебники/книгообеспеченность |
| **Вузовские / научные** | сотни | большие фонды, ЭБС, наука, репозитории | discovery/OAI, полный текст, книгообеспеченность, интеграции |
| **Специальные/ведомственные** (как театральная) | нишевые | свои базы/поля, защищённый контур | гибкие модели данных, on-prem/edge, кастом-модули |

## 2. Ценностное предложение (vs ИРБИС/J-ИРБИС)
Дешевле и быстрее внедрить · современный веб+мобайл вместо desktop-АРМов и EOL-веба · **офлайн на месте + облако** · доступность (ГОСТ/WCAG) · **ячеистое хранение/постамат** · постоянное развитие (микросервисы/модули) · импортозамещение (реестр отечественного ПО, Astra Linux) · миграция из ИРБИС «под ключ».

## 3. Модель поставки
- **Cloud SaaS** — для большинства (особенно школьные/малые): ничего ставить не нужно.
- **On-prem / edge-узел** — защищённый контур, нестабильный интернет, крупные/ведомственные: локальная работа + синхронизация.
- **Гибрид** — облако-ядро + edge на площадках сети.

## 4. Оси ценообразования
Подписка (год), цена зависит от: **типа библиотеки**, **размера** (объём фонда / число читателей), **числа площадок (edge-узлов)**, **числа рабочих мест сотрудников**, **набора модулей**, **объёма хранилища/полных текстов**. Плюс разовое: онбординг/миграция, обучение, кастом.

## 5. Тарифы (пресеты модулей)
| Тариф | Включено | Кому |
|---|---|---|
| **Базовый** | каталог + поиск + читательский веб (ЛК, заказ) | школьные, малые |
| **Стандарт** | + книговыдача + ячейки + уведомления | публичные |
| **Профи** | + комплектование + книгообеспеченность + аналитика/BI + discovery/OAI | вузы, крупные |
| **Сеть / Энтерпрайз** | мультиплощадка + edge/офлайн + RFID/ТСД + SLA + кастом-модули | сети, ведомственные |
Модули докупаются **à la carte** поверх тарифа (RFID/ТСД, оплата, межбибл. обмен, ИИ-сервисы).

## 6. Онбординг и миграция
Миграция из ИРБИС (через адаптер Прохода Б) — **услуга в пакете внедрения**: зеркало → сверка → переключение. Обучение персонала, перенос настроек/баз. Для сетей — поэтапно по площадкам.

## 7. Поддержка / SLA
Тиры поддержки (базовый/приоритетный/энтерпрайз), для edge — окна обновлений и авто-апдейт. Документация, обучающие материалы, сообщество.

## 8. Каналы и go-to-market
- **Пилоты-витрины** (СПб театральная библиотека) → кейсы.
- Гос/муниципальные закупки (44-ФЗ/223-ФЗ), вхождение в **реестр отечественного ПО**.
- Региональные/вузовские сети, методические объединения, партнёры-интеграторы.
- Импортозамещение ИРБИС как явный триггер.

## 9. Метрики/биллинг
Подписка + использование (хранилище/тексты/запросы/число операций), учёт per-tenant; самообслуживание тарифов в облаке; отчётность для закупок. → реализуется сервисами Tenancy/Licensing/Billing ([ARCHITECTURE §3](ARCHITECTURE.md)).

## 10. Открытые вопросы
Прайс-точки по сегментам · бесплатный тариф для школ (привлечение) · модель партнёрства/реселлеров · политика кастом-модулей (платно/в продукт) · сертификация (ФСТЭК/защищённый контур).
