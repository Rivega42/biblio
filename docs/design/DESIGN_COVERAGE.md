Классы: **✅ спроектировано до реализуемости · 🟡 поверхностно (названо, нет глубины/AC/контракта) · ❌ отсутствует**.

## Честный вердикт

**Широта ≈ 100%** (всё названо, разложено на инициативы I1–I7 + 23 эпика, поля АРМов ws1–ws5 и читателя расписаны). **Глубина реализации неравномерна:**

- **Глубоко (≈90%):** I1 (фундамент), I5 (АРМ-паритет ws1–ws5), I2 (каталог/поиск/читатель).
- **Скелет (30–50%):** I3 (книговыдача/ячейки), I4 (edge/синк), I6 (безопасность/комплаенс), I7 (коммерциализация), GTM — спеки инициатив намеренно были обзорные, без декомпозиции на задачи.
- **Сводно по функциям ИРБИС-паритета:** ≈ **45% ✅ / 39% 🟡 / 16% ❌**.

Итого: «100% спроектировано» относилось к **инициативам и полевому паритету**, а не к каждой функции с глубиной реализации. ~40% функций — поверхностно, ~15% — реальные дыры, и они **концентрируются в сквозных движках, гос-рынке и наших же differentiators**.

## Реестр сквозных «глубоких» пробелов

> Это главный результат: дыры **архитектурные, не точечные**. Закрываются спеками/ADR + декомпозицией на задачи. Эпик-зонтик: **«Доглубление дизайна до реализуемости»**.

### A. Сквозные движки (наибольший рычаг)
- **A1. Движок PFT/печати** — показ, каталожные карточки, формуляры, КСУ-бланки, RTF→PDF; **+ UNIFOR-коды (40+)**. Без него АРМы Каталогизатор/Комплектатор/Книговыдача не печатают. ❌ архитектурный. → I5/I2
- **A2. Декларативный движок ФЛК** — ~70 правил перечислены, нет схемы правил/валидатора/per-tenant-тюнинга. 🟡 → I5
- **A3. Исполнитель глобальной корректировки `.gbl`** — синтаксис описан, нет парсера/превью/отката/переменных. 🟡 → I5
- **A4. Сервис авторитетных файлов** — поиск + автоподстановка `^3` + autoin; алгоритм не определён. 🟡 → I5/I2
- **A5. Сидинг словарей/локаций/авторитетов на тенант** — `kv.mnu`/`jz.mnu`/`spec.tre` пусты → ФЛК и выбор МХ не работают; **онбординг-блокер**. 🟡 → I1/I5
- **A6. Движок уведомлений** — мультиканал (SMS/email/push), предпочтения, ретраи. ❌ → I3/I2
- **A7. OpenAPI-контракты новых движков** — книговыдача, синк, заказы, события (формализма нет). 🟡 → I1

### B. Гос-рынок (блокеры РФ)
- **B1. Гос-статистика** — 6-НК, КСУ-выгрузки, дневники, COUNTER/SUSHI. 🟡/❌ → I5/I2
- **B2. Внешние протоколы** — Z39.50-**сервер**, SRU, OAI-PMH, SIP2, NCIP/ILS-DI: детальные контракты. 🟡 → I2
- **B3. Гос-выгрузки** — НЭБ / СМЭВ / СКБР / ЛИБНЕТ: форматы + маппинг полей + маршрут. 🟡 → I2
- **B4. 152-ФЗ + ФСТЭК** — слой согласий/ПДн-UI, классификация ИСПДн, **схема журнала доступа**, право на забвение; чек-лист аттестации. ❌ → I6

### C. Differentiators (наш моат — доспроектировать)
- **C1. Ячейки/постаматы** — жизненный цикл кода выдачи, слот-аллокация, отказоустойчивость (сбой питания/замок), device-gateway/RFID-станции, автосдача+сортировка, **staff-воркфлоу**. 🟡 → I3
- **C2. Edge/синк детально (#125–127)** — схема событий, разрешение конфликтов **+ UI**, курсор/порядок, grace-лицензия, **хаос-тесты**. 🟡 → I4
- **C3. AI/RAG** — выбор эмбеддингов, KB/вектор-схема + изоляция тенантов, **guardrails/анти-галлюцинация тест-сьют**, автокаталогизация (порог/ревью). 🟡 → I2/I7
- **C4. Accessibility-first (сейчас ❌ hollow)** — реальная a11y-задача: рубрика WCAG 2.2 AA, axe/Lighthouse CI-гейт, VPAT, ГОСТ Р 52872, DAISY/аудио. → I2/I6
- **C5. Безопасность маркетплейса** — схема дескриптора, sandbox/capability, подпись/ротация/отзыв, аудит, тест изоляции плагинов. 🟡 → I7

### D. Платформа / коммерция / ops
- **D1. Low-code конфигуратор** — редактор полей/РЛ/ФЛК/темы + версионирование/откат. 🟡 → I5
- **D2. Ключи/PKI (ADR-006)** — per-tenant envelope, ротация, отзыв. ❌ → I6
- **D3. IaC/Terraform** + edge-флот/OTA (#120) + мульти-регион. ❌ → I1/I6
- **D4. SRE** — конкретные SLO/RPO/RTO, runbooks, график DR-учений. 🟡 → I6
- **D5. Биллинг-движок** — определения метрик метеринга, счета/НДС/платёжный провайдер, lifecycle подписки, **прайс-точки**, enforcement entitlements. ❌ → I7
- **D6. Финансы (Парус) + ERM (#182) + конвейер оцифровки (DG #159/#180) + CDL** — оркестрация. ❌/🟡 → I7/I5
- **D7. GTM-исполнение** — прайс-точки, free-tier политика, реестр ПО + 44/223-ФЗ чек-лист, региональная последовательность, партнёрская модель, миграция-runbook/P&L. ❌ → I7

### E. Бизнес-правила (доменная логика)
- **E1. Комплектование** — demand-driven дозаказ + агрегация отказов; **алгоритмы КСУ-распределения** (поля 44–49/145–158); **формулы ККО** (деление на 0, нормировка, порог дозаказа). 🟡/❌ → I5
- **E2. Книговыдача** — holds-block-renewal, override должника, начисление штрафов/утеря. 🟡 → I3/I5

### Сироты (не закреплены за инициативой)
AI-навигатор/Cmd+K (#162) · Help-портал/живые доки (#142) · i18n (рус/англ+национальные) · библиотека компонентов дизайн-системы (#150/#153) · предиктивная аналитика спроса. → закрепить за I1/I2.

## Приоритет закрытия
1. **A1 PFT/печать + A5 сидинг + A2 ФЛК** — без них ИРБИС-паритет невозможен (АРМы не печатают/не валидируют/не онбордятся).
2. **B1 гос-статистика + B2/B3 протоколы/выгрузки + B4 152-ФЗ** — без них не перейдут вузы/госбиблиотеки.
3. **C1 постаматы + C2 синк + C4 accessibility** — наш моат; сейчас позиционируем как уникальное, а спеки wireframe-уровня.
4. **D5 биллинг + D7 GTM** — без них продукт застрянет в пилоте.
5. Остальное (C3/C5/D1–D4/D6/E) — по мере подхода к фазам.
