# SPEC · I4 — Edge и офлайн-синхронизация

> Инициатива I4 — самый рисковый столп. Эпики: **#113** (edge-узлы + офлайн-синхронизация), **#131** (консистентность, ADR-005), **CN #181** (консорциумы/union-каталог). Опора: [SYNC_DESIGN (ADR-005)](../SYNC_DESIGN.md) (детальная модель), [ARCHITECTURE](../ARCHITECTURE.md) §2,5, [BEST_PRACTICES](../BEST_PRACTICES.md) §2,7. DoD-гейт как в I1 + **хаос-тесты конвергенции**.

## A. Edge-узел (упаковка)
**Подход:** узел библиотеки = Docker-стек с **локальной PostgreSQL-репликой тенанта** + включёнными модулями; работает **офлайн**; подписанные **авто-обновления** (Astra Linux / защищённый контур). Тонкие библиотеки — «только облако».
**AC:** AC1 узел поднимается на месте, обслуживает читателя/сотрудника локально; AC2 переживает обрыв сети неограниченно.

## B. Sync-движок (ядро, ADR-005)
**Подход:** **transactional outbox** + **Hybrid Logical Clock** (не wall-clock); **владение по доменам** (каталог=облако-master → очередь ревью правок edge; книговыдача/экземпляры=edge-master, event-sourced); дельта-синхронизация since-cursor; **идемпотентность по event_id**; tombstones; докачка/backpressure; **журнал конфликтов** + ручной разбор в UI. Оценить ElectricSQL/PowerSync как ускорители.
**AC:** AC1 двусторонняя синхронизация edge↔облако с восстановлением после обрыва; AC2 каталог-правки edge уходят в очередь ревью (облако-master); AC3 операции выдачи складываются без конфликта (event-sourced); AC4 конфликты полей → журнал + разбор.

## C. Офлайн-книговыдача
**Подход:** выдача/возврат/бронь полностью на edge (edge-master), синк в облако при сети; см. I3/D.
**AC:** AC1 полный цикл выдачи офлайн; AC2 дозагрузка при восстановлении сети без потерь/дублей.

## D. Консорциумы и сводный каталог (CN #181)
**Подход:** **трёхзонная модель** (Community/Network/Institution) над тенантами; **union-каталог** + кооперативная каталогизация/дедуп; **межтенантный/межфилиальный resource sharing** (holds/floating между постаматами/ячейками); интеграция **СКБР/ЛИБНЕТ/НЭБ**; протоколы **NCIP/ILS-DI/SRU**.
**AC:** AC1 сводный каталог сети с переиспользованием записей; AC2 межбибл. заказ/доставка между узлами; AC3 заимствование из СКБР/ЛИБНЕТ.

## E. Fleet-management / OTA
**Подход:** подписанные обновления, **canary** по узлам, **автооткат**, skew-политика edge↔облако, **офлайн-валидация лицензии** (grace), телеметрия с узлов. Связь с I6 (#120).
**AC:** AC1 поэтапная раскатка обновления с откатом; AC2 узел работает на просроченной связи в grace-периоде.

## Критический путь I4
1. Edge-упаковка + локальная реплика → 2. **sync-движок** (outbox+HLC+владение+конфликты, #125–127) → 3. офлайн-книговыдача → 4. консорциумы/union + СКБР/NCIP (CN) → 5. fleet/OTA. Параллельно — **хаос-тесты** (обрывы/переупорядочивание/дубли/перекос часов/долгий офлайн).
**Выходной критерий:** библиотека работает без интернета и синхронизируется при сети; сеть/ЦБС шарит фонды между узлами — отраслевой пробел, закрытый нами.
