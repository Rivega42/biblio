# SPEC · I3 — Книговыдача и физическое хранение

> Инициатива I3 (наш differentiator). Эпики: **#95** (свой сервер — книговыдача через ячейки/постамат), **#96** (редактор плана помещения), **PMC #145** (логистика) + подзадачи #155/#179 и др. Опора: [SPEC_ws3](SPEC_ws3_circulation.md) (полевая модель), [SYNC_DESIGN](../SYNC_DESIGN.md) (edge-master для выдачи), [PRODUCT_PRACTICES](../PRODUCT_PRACTICES.md), [ARCHITECTURE](../ARCHITECTURE.md) §3. DoD-гейт как в I1.

## A. Доменная модель книговыдачи (event-sourced)
**Подход:** оперативные операции (выдача/возврат/бронь/продление/перемещение) — **события** (edge-master, ADR-005), состояние = свёртка; сущности reader/loan/request/hold; поля по [SPEC_ws3] (RDR/RQST + поле 40). PostgreSQL JSONB + материализованные статусы.
**AC:** AC1 выдача/возврат меняют статус экземпляра и формуляр; AC2 операции идемпотентны и журналируются (аудит/откат); AC3 лимиты (MaxBooks/срок) и задолженность.

## B. Иерархическое ячеистое хранение
**Подход:** дерево `location` **адрес→здание→этаж→помещение→стеллаж→полка→ячейка** (число ячеек на полке разное) + размещение экземпляров (leaf или «на руках»); постамат и станция книгоприёма со слотами. Уже есть в own-server PoC (sqlite+PG).
**AC:** AC1 экземпляр размещается в листовой локации; AC2 карточка показывает полный адрес хранения; AC3 drill-down карта занятости (агрегаты по уровням).

## C. План помещения (редактор, #96)
**Подход:** геометрия `gx/gy/gw/gh` у локаций; **редактор drag-drop** расстановки стеллажей на плане; загрузка подложки/масштаб; SVG top-down план с заполненностью; навигация «найти на полке/в зале» от карточки/ячейки.
**AC:** AC1 стеллажи расставляются/двигаются на плане, геометрия сохраняется; AC2 «найти на плане» подсвечивает локацию; AC3 цвет = заполненность.

## D. Постамат / self-service / RFID (#145, ★ differentiator)
**Подход:** бронь → размещение в **ячейку/постамат** → **код выдачи** → читатель забирает 24/7 → **авто-checkout** при изъятии; **self-checkout/RFID**; станция **автосдачи** + сортировка по ячейкам; **holds-aware локеры** (outdoor IP54); рекомендации на киоске в момент выдачи (NoveList-стиль). Всё **работает офлайн** на edge (I4).
**AC:** AC1 выдача брони через ячейку/постамат по коду с авто-checkout; AC2 self-checkout (RFID/штрих-код) проводит loan; AC3 автосдача меняет статус; AC4 офлайн-режим (event-sourced, синк при сети).

## Критический путь I3
1. Движок книговыдачи (события/статусы/лимиты) на PostgreSQL → 2. ячеистое хранение (дерево+размещение) → 3. план помещения + редактор (#96) → 4. постамат/self-checkout/автосдача (#145) → 5. офлайн (на I4).
**Выходной критерий:** полный цикл книговыдачи в вебе + **выдача 24/7 через ячейки/постамат** + работа офлайн — превосходит ИРБИС (ни у одного РФ-АБИС нет ячеек/постаматов).
