# SPEC · Веб-Читатель — паритет J-ИРБИС (поведенческий уровень)

> Блок, **закрывающий J-ИРБИС 2.0** (публичный веб-OPAC ИРБИС — аналог АРМ «Читатель», но в вебе). Это читательский counterpart к АРМ-спекам ws1–ws5. Относится к инициативе **I2**, эпик **#97** (паритет читателя) + #130 (поиск) + **PMA #143**/**PMB #144** (discovery/вовлечение) + #124 (ЕСИА). Грунтовано на recon: [JIRBIS2_BEHAVIOR](../../recon/deep/JIRBIS2_BEHAVIOR.md) (42 функции, ссылки на мануал), [ARM_READER](../../recon/deep/reference/arms/ARM_READER.md), [DB_CIRCULATION](../../recon/deep/reference/databases/DB_CIRCULATION.md). Высокоуровнево — [SPEC_I2](SPEC_I2_catalog_search.md). ⚠ ПДн (RDR): минимизация, журнал доступа.

**Принцип:** воспроизводим **поведение** J-ИРБИС (он работает как АРМ Читатель поверх TCP-сервера ИРБИС), но на нашем стеке (J-ИРБИС = PHP5.6/Joomla2.5/MariaDB — всё EOL; конфиг = ручная правка PHP). У нас — декларативный конфиг, наш контракт, **i18n и accessibility как база** (в J-ИРБИС их нет).

## 1. Конфигурация вывода (профили)
**J-ИРБИС:** `formats_profile` (элемент→PFT) + `presentation_profile` (что/где/при условии показывать: `Title`/`Indicators`(поля-условия, напр. 951→только если есть файлы)/`Type`(0 в теле БО·1 кнопка·2 закладка)/`Format`(@pft)); форматы PFT из папки `Deposit`, переопределяемы по базам.
**У нас:** декларативные профили показа per-tenant (карточка/закладки/условия по полям), без PFT-интерпретатора — реимплементация ключевых форматов + рендер-движок.
**AC:** AC1 профиль отображения настраивается per-tenant/per-БД; AC2 условный показ блоков по наличию полей (951/955/922…); AC3 краткий/полный формат + профили (museum/person/reader-аналоги).

## 2. Поиск (8 режимов J-ИРБИС)
| Режим J-ИРБИС | Поведение | У нас |
|---|---|---|
| Простой (`K=`) | по ключевым словам | ✅ (есть) |
| Свободный (`DS=`→`K=`) | по всем полям | ◑ |
| Свободный с ранжированием | устойчив к «мусору», **лимит 100** | ✅ **без лимита 100** (наш плюс) |
| Расширенный | форма с полями | ✅ (есть) |
| Комплексный | произвольная комбинация 19 элементов | ◑ |
| Экспертный | язык запросов ИРБИС (И/ИЛИ/НЕТ/В ПОВТОРЕНИИ/РЯДОМ, усечение+морфология) | ◑ |
| Полнотекстовый (`KT=FT!`) | по PDF + БЗ (совпадение в БЗ → выше) | ◑ (связь DG/OCR) |
| Просмотр базы | 12 записей/стр с обложками | ◑ |
**Поведение:** **поиск-по-вводу с индикатором прогресса**; автонормализация автора («тургенев иван»→«Тургенев, И.»); морфология рус/англ; усечение чекбоксами; **ленивые фасеты** (по связанным авторам/КС/типам с количеством, при раскрытии). **Навигаторы:** ГРНТИ-дерево (множественный выбор/исключение), новые поступления (виртуальный календарь), разделы знаний (ББК+УДК+ГРНТИ), авторитеты (410/510/710), MeSH.
**AC:** AC1 поиск-по-вводу с прогрессом; AC2 нормализация+морфология (см. #122); AC3 ленивые фасеты с counts; AC4 навигатор ГРНТИ с мультивыбором; AC5 полнотекст по PDF+БЗ; AC6 экспертный язык запросов. **Наш +:** семантика+RAG-ассистент с цитированием (#171), appeal-таксономия (#172), FRBR (#97).

## 3. Результаты и карточка записи
**J-ИРБИС:** обложки (динамически из интернета → у нас **кэш/прокси + graceful degradation**); сворачиваемые оглавление/аннотация/КС (cookie ~год); закладки «Экземпляры»/«Электронные версии»/«Сиглы»; статьи содержания (поле 922); переходы обложка→оригинал, БО→полная; гиперссылки сохраняют выбранные БД; отбор (чекбоксы/номера) + **«Корзина»**; печать/экспорт в **Word**, краткий/полный, сохранение отмеченных из нескольких запросов, **email-рассылка списков**; **WEB 2.0**: рейтинг 5-балльный, комментарии с модерацией, Top-10 (отключаемо).
**AC:** AC1 карточка со сворачиваемыми блоками + закладки (экземпляры/э-версии/сиглы); AC2 обложки через прокси/кэш (работает в закрытой сети); AC3 корзина + экспорт (Word/RIS/BibTeX/ГОСТ) + email; AC4 статьи содержания (922) кликабельны. **Наш +:** обогащение/рекомендации/read-alikes; **кросс-тенантный соц-граф** отзывов/рейтингов (#174), Showcases/browse (#175).

## 4. Полные тексты / ресурсы + права доступа (ядро OPAC)
**Связь файлов:** поле **951** (`^A` относит. путь / `^I` URL, взаимоисключающи) и **955** (полнотекст); папка = 11-й параметр PAR.
**Просмотр:** PDF постранично (конвертация на лету, стиль Google Books), MP3/MP4; **закладки** (cookie, и для неавторизованных); оглавление PDF; **подсветка терминов** (`ji_highlight`) + миниатюры релевантных страниц.
**Права доступа (центрально, переносим точно):** глобальные правила (`ed_pdf_view_only/view/text`, разные для LAN/интернета `ed_external_same`); **на файл — подполя 951** (`^D` категории, `^L` только LAN, `^9` флаги выгрузка/просмотр/проигрывание/копирование/печать); **БД RIGHT (955: IP/категория/специальность/периоды), приоритет 951>955**; LAN по маске; лимит числа/процента страниц; причина недоступности. Два метода выгрузки (`prf` локальный / `std` через TCP, лимит ~10МБ). HTTP/FTP-ссылки J-ИРБИС отдаёт напрямую (слабая защита) → **у нас всё через бэкенд-прокси**.
**У нас:** модель прав ложится на наш Access/гранты (функция×база×уровень) + per-file ACL (951^D/^L/^9) + RIGHT-аналог; view-only прокси; связь с **DG/IIIF Content Search** (подсветка по координатам, #180/#162).
**AC:** AC1 постраничный PDF view-only через прокси (бинарь не утекает); AC2 права на файл по 951(^D/^L/^9)+RIGHT(IP/категория/период), приоритет 951>955; AC3 подсветка терминов + миниатюры; AC4 закрытый контур (LAN-маска, лимит страниц, причина недоступности); AC5 аудио/видео-просмотр.

## 5. Личный кабинет читателя
**J-ИРБИС:** вход поверх **RDR** (фамилия + идентификатор поле 30; `user_password_tag` — поле 130/24; MD5 без соли → у нас **безопасный хэш + ЕСИА**); запрет входа должникам; формуляр; закладка **«Заказы»** (отслеживание, удаление до выполнения); **«Профиль»** (электронные версии, выполненные ЭДД); **продление онлайн** (`debts_prolongation_enable`, times×days); правка ПДн в RDR (по умолч. выкл); **постоянные запросы** (авто-повтор формулировки — аналог алертов/ИРИ); регистрация/пароль (3 сценария: выдача/предрегистрация/полная, автодополнение из RDR); **токен-ссылки ЭИОС** (`token`=md5(идентиф+дата+секрет), срок 1 сутки).
**AC:** AC1 вход (билет/ЕСИА), запрет должникам по правилу; AC2 формуляр + Заказы (отмена до выдачи) + Профиль (э-версии); AC3 онлайн-продление с лимитом; AC4 постоянные запросы → алерты на новинки; AC5 регистрация (3 сценария); AC6 токен-ссылки ЭИОС для интеграции с вузовской ЭИОС. **Наш +:** opt-in история (152-ФЗ/ALA), полки/списки (#174), мобильное приложение (#179).

## 6. Заказ книги (читательский)
**J-ИРБИС:** предзаказ на закладке БЗ; авторизованному — **«Заказать» в один клик**; **очередь на занятую** (`order_queue_enable`, показ размера); лимит `order_max_count`; **«Места хранения и выдачи»** (аббревиатура/название/кафедра/флаг «предзаказ доступен»); жизненный цикл оформил→ЛК «Заказы»→сотрудник меняет статус→«Профиль»; **авто-письма** при смене статуса; ЭДД (заказ скана, диапазон страниц, ст.1275 ГК).
**Модель заказа** — в RQST (см. [ws3](SPEC_ws3_circulation.md)/DB_CIRCULATION): 30 читатель, 903 шифр, 40 дата, тип по 41.
**AC:** AC1 заказ «в один клик» по доступному МХ; AC2 очередь на занятую с показом позиции; AC3 лимит заказов; AC4 авто-уведомления о смене статуса; AC5 ЭДД (опц.) с правилами доступа. **Наш +:** заказ → **выдача через ячейку/постамат 24/7** (#95, ws3), прозрачная очередь.

## 7. Настройка, локализация, доступность (наша зона превосходства)
**J-ИРБИS:** почти всё отключаемо (`search_form_enable`/`print_*`/`sort_chose`/`cover_*`/`checkbox_enable`…), параметры можно в URL; **но i18n библиотечной части НЕТ; accessibility (ГОСТ Р 52872/WCAG) НЕТ** (только адаптив/киоск).
**У нас:** декларативная настройка per-tenant; **i18n как база** (рус/англ+), **accessibility WCAG 2.2 AA / ГОСТ как first-class** (наш differentiator на гос-рынке); мультиканальность (веб/киоск/мобайл) + **Cmd+K/AI-навигатор** (#162).
**AC:** AC1 декларативная настройка форм/блоков per-tenant (+ URL-параметры); AC2 i18n; AC3 WCAG 2.2 AA/ГОСТ-аудит проходит (у J-ИРБИС нет — наш плюс).

## Что превзойти (дефекты J-ИРБИС → наши требования)
EOL-стек (PHP5.6/Joomla) → современный · нет i18n/accessibility → база · сортировка тормозит поиск → **на индексе** · ранжирование лимит 100 → без лимита · PDF не для всех/виснет >5мин → надёжный просмотр · HTTP-ссылки напрямую → **прокси** · часть прав RIGHT не поддержана → полная модель прав · обложки из интернета → **кэш/прокси** · фасеты/постоянные запросы/закладки только ИРБИС64+ → у нас всем.

## Сохранить как поведение (ценное из J-ИРБИС)
Поиск-по-вводу с прогрессом · автонормализация+морфология · ленивые фасеты · сворачиваемые элементы записи (профили показа) · заказ «в один клик» · очередь на занятую · продление в ЛК · постраничный PDF с подсветкой · токен-ссылки ЭИОС · корзина/экспорт/email-рассылка.

## Критический путь (читательский блок I2)
1. **Поиск** (режимы + по-вводу + фасеты + навигаторы, #122/#80) → 2. **карточка** (профили показа, закладки, обложки-прокси) → 3. **полные тексты + права** (951/955/RIGHT, view-only прокси, IIIF) → 4. **ЛК** (вход ЕСИА/билет, формуляр, заказы, продление, постоянные запросы) → 5. **заказ** (1 клик, очередь → ячейки/постамат) → 6. **настройка+i18n+accessibility** (наш плюс) → 7. **наши +** (семантика/RAG, соц-граф, мобайл).
**Выходной критерий:** публичный веб-OPAC воспроизводит **все 42 поведения J-ИРБИС** (поиск/карточка/полные тексты с правами/ЛК/заказ) и **превосходит** по accessibility, i18n, релевантности (без лимита 100), безопасности (прокси), AI и соц-слою — на современном стеке.
