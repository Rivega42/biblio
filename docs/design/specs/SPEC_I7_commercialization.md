# SPEC · I7 — Коммерциализация (модули, биллинг, GTM)

> Инициатива I7. Эпики: **#114** (модульная система, событийная шина, биллинг), **#115** (политика продаж), **PMF #148** (платформа/ИИ/аналитика/приватность) + #106–108, #177/#178, #109–111. Опора: [GO_TO_MARKET](../GO_TO_MARKET.md), [ARCHITECTURE](../ARCHITECTURE.md) §4, [COMPETITIVE_LANDSCAPE](../COMPETITIVE_LANDSCAPE.md) (FOLIO-маркетплейс). DoD-гейт как в I1.

## A. Модульная система и маркетплейс (#106, #178)
**Подход:** **реестр модулей** в облаке + **контракт плагина** (события + точки расширения UI, не прямой доступ к БД) + **включение per-tenant/тариф** (entitlements, отдельно от биллинга); формальные **module-descriptors** (FOLIO-стиль) + **подписанные пакеты** (Koha KPZ-стиль, связь supply-chain I6); **webhooks + встроенные мини-приложения** (Alma Cloud Apps); capability-права, sandbox, semver.
**AC:** AC1 модуль включается/выключается per-tenant; AC2 сторонний подписанный плагин ставится через реестр; AC3 webhooks + мини-приложение расширяют без правки ядра.

## B. Событийная шина (#107)
**Подход:** NATS/Kafka/RabbitMQ — межсервисные события + транспорт синхронизации (outbox/CDC, связь I4). Основа слабой связности модулей/микросервисов.
**AC:** AC1 события доставляются между сервисами идемпотентно; AC2 шина — транспорт sync edge↔облако.

## C. Биллинг, метрики, тарифы (#108)
**Подход:** **точный metering** (хранилище/тексты/запросы/операции) per-tenant → подписка + использование; **тарифы** (Базовый/Стандарт/Профи/Сеть), модули à la carte; жизненный цикл тенанта (trial→paid→suspend→churn); self-serve админка; экспорт данных тенанта (анти-lock-in); отчётность для закупок.
**AC:** AC1 метрики использования собираются per-tenant; AC2 тариф/модули управляют доступом (entitlements); AC3 биллинг-отчёт формируется.

## D. Аналитика и ИИ-сервисы (PMF)
**Подход:** дашборды + **кросс-тенант бенчмарк** (анонимно); прямой **read-only SQL к реплике** (SierraDNA-стиль, #177); **ИИ-ассистент/RAG с цитированием** (#171), авто-каталогизация (#158), рекомендации, прогноз спроса (как модули, privacy-by-design).
**AC:** AC1 дашборды/бенчмарк; AC2 SQL-доступ аналитика к реплике; AC3 ИИ-модули включаются как платные.

## E. Go-to-market (#109–111)
**Подход:** сегменты/тарифы/прайс-оси ([GO_TO_MARKET](../GO_TO_MARKET.md)); бесплатный тариф для школ; **реестр отечественного ПО** + закупки 44/223-ФЗ (+ФСТЭК); **миграция-онбординг как услуга** (адаптер Прохода Б → зеркало → сверка → переключение) + поддержка/SLA; пилоты-витрины (СПб театральная).
**AC:** AC1 прайс/тарифы зафиксированы; AC2 продукт в реестре отеч.ПО; AC3 пакет внедрения (миграция+обучение+SLA).

## Критический путь I7
1. Entitlements + реестр модулей (на I1) → 2. событийная шина → 3. биллинг/метрики/тарифы → 4. маркетплейс (webhooks/мини-приложения/подпись) + аналитика/ИИ-модули → 5. GTM (реестр ПО/закупки/онбординг).
**Выходной критерий:** коммерческий запуск — библиотека-тенант подключает модули по тарифу, платит по использованию, мигрирует из ИРБИС как услугу; продукт в реестре отеч.ПО, готов к закупкам.
