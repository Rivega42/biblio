> Уточнение инициативы **I2** до реализуемого уровня: эпики **#97** (паритет читателя), **#130** (поиск/стандарты/гос-интеграции), **PMA #143** (discovery-механики), **PMB #144** (вовлечение), **DS #154** (дизайн-система) → таски/сабтаски с **критериями приёмки (AC)**, подходом, зависимостями, **DoD**. Учтены выводы [COMPETITIVE_LANDSCAPE](../COMPETITIVE_LANDSCAPE.md)/[MATRIX](../COMPETITIVE_MATRIX.md). Опора: [ARCHITECTURE](../ARCHITECTURE.md), [PRODUCT_PRACTICES](../PRODUCT_PRACTICES.md), [SPEC_I1](SPEC_I1_foundation.md), [TZ_CLAUDEDESIGN](../TZ_CLAUDEDESIGN_design-system.md).

**DoD-гейт (как в I1):** код + автотест + контракт-тест (openapi) + наблюдаемость (trace/tenant-id) + a11y-проверка (WCAG 2.2 AA / ГОСТ) + обновление доков. Размеры S/M/L.
**Зависимость I2 от I1:** мультиарендность + Identity (ЕСИА), PostgreSQL-ядро, контракт. **Cross-cutting:** дизайн-система **DS #154** — UI всех экранов I2 строится на ней.

## A. Поиск и discovery (ядро I2)

### #122 — Релевантность: русская морфология/синонимы/fuzzy/бустинг · L
**Подход:** PostgreSQL FTS `to_tsvector('russian')` + `pg_trgm` (опечатки/«возможно вы имели в виду»), `ts_rank` с **бустингом полей** (заглавие>автор>рубрики), словарь синонимов/тезаурус (ts dictionary); порог перехода на **OpenSearch** при росте корпуса/требований; **реиндексация без простоя** (alias-swap).
**AC:** AC1 морфология (запрос «книга» находит «книги/книгу»); AC2 опечатка («Андроид»→«Android»-подобное) даёт результат + «возможно, вы имели в виду»; AC3 бустинг (совпадение в заглавии выше, чем в примечании); AC4 реиндексация без даунтайма; AC5 фасеты (#80) сохраняются. **DoD:** бенч релевантности на выборке.

### PMA #143 → семантический поиск + RAG-ассистент (#171) · L · 🔴 differentiator
**Подход:** мультиязычные **эмбеддинги** (pgvector или OpenSearch kNN) + **гибридное ранжирование** (лексика+семантика); **RAG-ассистент** поверх индекса: ответ **обязан ссылаться на объекты каталога** (жёсткий **anti-hallucination**, модель Libby), мультиязычие, **«сначала доступное сейчас»**, структурированные **inspiration-промпты** (модель Primo Research Assistant/Inspire Me). Privacy-by-design: **без логирования свободного текста**.
**AC:** AC1 семантический запрос по смыслу находит релевантное, отсутствующее в лексическом; AC2 **каждый ответ ассистента содержит ссылки на записи каталога**, без «галлюцинаций»; AC3 ранжирование поднимает доступные экземпляры; AC4 нет хранения сырых пользовательских формулировок. **Deps:** #122, ядро. **DoD:** + tenant AI-policy (вкл/выкл, приватность).

### #97 → FRBR-группировка + навигаторы-классификаторы · M
**Подход:** кластеризация **произведение→издания→экземпляры** (по заглавию/автору/идентификаторам); **навигаторы** ГРНТИ/УДК/ББК как дерево (**shelf-browse**, «рядом стоящие»).
**AC:** AC1 несколько изданий одного произведения группируются под одной записью с раскрытием; AC2 клик по узлу классификатора фильтрует/открывает «виртуальную полку». **Deps:** #122.

### PMA #143 → appeal-таксономия (#172) + Showcases/browse (#175) · M
**Подход:** слой метаданных **tone/pace/mood/content-warnings** (StoryGraph/NoveList) как фасеты; **кураторские Showcases** + **browse-домашняя** (сезонное/жанр/возраст/новинки — модель BiblioCore Browse&Discover). Сиды словаря + ИИ-предложения с проверкой куратором (HITL).
**AC:** AC1 appeal-фасеты фильтруют выдачу, content-warnings видны в карточке; AC2 библиотекарь собирает Showcase, читатель видит browse-домашнюю. **Deps:** дизайн-система.

### PMA #143 → обогащение + рекомендации + readers' advisory · M
**Подход:** обложки/аннотации/оглавление/отзывы в карточке; **«похожее/также брали»** и **read-alikes** (appeal-факторы); сохранённые поиски + **алерты на новинки**; экспорт цитирования.
**AC:** AC1 карточка обогащена; AC2 блок рекомендаций релевантен; AC3 подписка на сохранённый поиск шлёт новинки.

## B. Читательский опыт

### #97 → карточка записи (связи/похожее/экспорт/просмотрщик) · M
**Подход:** курированная карточка (готово) + **связи** (серии/продолжения), «похожие», **экспорт RIS/BibTeX/ГОСТ**, постоянная ссылка/**QR**, просмотрщик обложек/**IIIF**/PDF (связь с DG #159/#180).
**AC:** AC1 экспорт в 3 форматах валиден; AC2 IIIF/PDF-просмотр открывается с подсветкой найденного (Content Search, связь с #180/#162); AC3 постоянный URI + QR. **a11y:** карточка проходит WCAG 2.2 AA.

### #97 → личный кабинет читателя · M
**Подход:** заказы/бронь/история/штрафы/**продление в 1 клик**, прозрачная **очередь брони** («вы N-й, ~дней»), уведомления; **fine-free**-опции + автопродление; **opt-in история чтения** (приватность ALA/152-ФЗ).
**AC:** AC1 операции self-service работают; AC2 очередь показывает позицию/срок; AC3 история — только по согласию, удаляема. **Deps:** I1 (ЕСИА/аккаунты), книговыдача (I3) для данных.

### PMB #144 → соц-слой (#174) + геймификация · L
**Подход:** полки/списки/рейтинги/отзывы/теги; **кросс-тенантный федеративный соц-граф** по **opt-in + модерация** (модель BiblioCore); геймификация (челленджи/серии/бейджи/**общий прогресс-бар сообщества** — Beanstack).
**AC:** AC1 UGC создаётся/модерируется; AC2 шеринг между тенантами только по согласию тенанта/читателя; AC3 челлендж с коллективной целью отображается. **Риск:** приватность/модерация — обязательны.

### PMB #144 → мобильное приложение (#179) · L
**Подход:** PWA/нативное: **цифровой читательский билет**, **self-checkout (RFID/штрихкод)**, **push** (модель Vega Mobile/Libby), офлайн-кэш.
**AC:** AC1 цифровой билет открывается офлайн; AC2 self-checkout проводит выдачу; AC3 push о готовности брони/сроках. **Deps:** I3 (книговыдача), I4 (офлайн).

### PMB #144 → платформа вовлечения (#176) · M
**Подход:** события/программы (регистрация), маркетинг-кампании/рассылки, **конструктор сайта библиотеки с SEO** (модель Vega LX). 
**AC:** AC1 событие с регистрацией; AC2 рассылка новинок; AC3 публичный сайт тенанта с schema.org/SEO.

## C. Стандарты и гос-интеграции (#130)

### #123 → стандарты обмена · L
**Подход:** импорт/экспорт **MARC21/RUSMARC/ISO 2709/MARCXML**; **Z39.50/SRU клиент И сервер** (нас тоже могут искать); **OAI-PMH-провайдер**; **SIP2** (RFID/self-service); +NCIP/ILS-DI/SRU (общая часть с **CN #170**).
**AC:** AC1 круговой импорт→экспорт MARC без потерь; AC2 внешний клиент ищет нас по Z39.50/SRU; AC3 OAI-PMH отдаёт записи агрегатору; AC4 SIP2-станция проводит выдачу. **DoD:** контракт-тесты протоколов.

### #124 → гос-интеграции РФ · L
**Подход:** **ЕСИА/Госуслуги-вход** (OIDC) + единый билет (= PMB **#163/pmb3**); **СМЭВ**; выгрузка в **НЭБ**; отчётность **КСУ/6-НК**.
**AC:** AC1 читатель входит через Госуслуги; AC2 выгрузка в НЭБ проходит валидацию; AC3 КСУ/6-НК-отчёт формируется. **Примечание:** это паритетный must-have РФ — без него вузы/гос не перейдут.

### связь с PME #173 (entity/BIBFRAME/LOD)
Сущностный поиск (works/instances) и устойчивые URI/LOD проектируются в PME (I5), но **поверхность discovery — в I2**: карточка/поиск используют связи сущностей, когда появятся.

## D. Дизайн-система (DS #154) — cross-cutting
Все экраны I2 — на стиле **A «Claude-like»** (light/dark) с **мультиарендной темизацией**; компоненты: SearchBar/фасеты/FilterChip, RecordCard, TreeNav (навигаторы), CommandPalette (**Cmd+K + AI-навигатор #162**), просмотрщики. Маркетинговый сайт тенанта (#176) — стиль **B Apple Glass**. DS — **предусловие** полировки UI I2.

## Критический путь I2
1. **DS-фундамент** (#149/#151) + **#122 релевантность** + **ЕСИА** (#124/pmb3) →
2. **discovery-механики** (семантика+RAG #171, FRBR/навигаторы, фасеты+appeal #172, Showcases/browse #175, обогащение/рекомендации) →
3. **карточка+просмотрщик+ЛК** (#97) →
4. **вовлечение** (соц-граф #174, геймификация, мобайл #179, платформа #176) →
5. **стандарты+гос** (#123/#124) — серверная часть, параллельно со 2–4.

**Выходной критерий I2:** читательский опыт ≥ ИРБИС/J-ИРБИС и выше: релевантный русский поиск + семантика/AI с цитированием, FRBR, фасеты+навигаторы+appeal, обогащённые карточки + IIIF/PDF-просмотр, ЛК, соц-слой/геймификация, вход через ЕСИА, стандарты (Z39.50/OAI/SIP2) + гос-интеграции (НЭБ/КСУ) — всё на новой дизайн-системе и доступно (WCAG 2.2/ГОСТ).
