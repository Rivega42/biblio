# SPEC · I6 — Безопасность, надёжность, комплаенс

> Инициатива I6 (сквозная, пик в фазах 3–4). Эпик **#129** (supply-chain / edge-OTA / SRE) + сквозные требования. Опора: [BEST_PRACTICES](../BEST_PRACTICES.md) §6,7,8, [ARCHITECTURE](../ARCHITECTURE.md) §7, [SPEC_I1](SPEC_I1_foundation.md). DoD-гейт как в I1.

## A. Мультиарендная изоляция (под тестом)
**Подход:** схема-на-тенанта + RLS для общих таблиц (связь I1 **#100**); **автоматические негативные тесты изоляции** в CI (ни один запрос не пересекает тенант); per-tenant ключи шифрования (envelope); лимиты ресурсов (анти-noisy-neighbor); pen-test.
**AC:** AC1 тест изоляции — обязательный CI-гейт; AC2 per-tenant шифрование at-rest; AC3 pen-test пройден.

## B. Цепочка поставок (supply-chain, #119)
**Подход:** **SBOM**, **подпись образов (cosign)** + верификация (критично для edge-OTA), скан уязвимостей (**Trivy**), pinned deps, **gitleaks/trufflehog в CI**, секреты в **Vault/SOPS** (не env в проде), ротация.
**AC:** AC1 образы подписаны и верифицируются перед деплоем; AC2 скан/secret-scan в пайплайне блокируют уязвимости/утечки; AC3 SBOM генерируется.

## C. Edge-OTA / fleet (#120) — связь I4
Подписанные обновления, canary, автооткат, офлайн-лицензия, телеметрия узлов. См. [SPEC_I4](SPEC_I4_edge_offline.md) §E.
**AC:** см. I4/E + верификация подписи на узле.

## D. Надёжность / SRE
**Подход:** **SLO/SLI** на поиск/выдачу + error budget; **OpenTelemetry** (трейсы+метрики+логи, сквозные trace-id+tenant-id — связь I1 **#117**); health/readiness, graceful degradation, circuit breakers; **бэкапы + регулярные restore-учения (RPO/RTO)**, run-book’и.
**AC:** AC1 SLO-дашборды + алерты; AC2 DR: восстановление из бэкапа проверено учением; AC3 деградация без полного отказа.

## E. Приватность и комплаенс РФ (152-ФЗ / ФСТЭК)
**Подход:** классификация **ИСПДн**, минимизация ПДн, **журнал доступа к ПДн**, сроки/право на забвение, шифрование at-rest/in-transit; **opt-in история чтения** (этика ALA); **ФСТЭК-аттестация** для гос-сегмента; согласия/политики; zero-trust/mTLS между сервисами; RBAC/ABAC (наши гранты).
**AC:** AC1 ПДн под минимизацией + журнал доступа; AC2 право на забвение реализуемо; AC3 готовность к аттестации (защищённый контур, без внешних CDN — уже соблюдено в дизайн-системе).

## Критический путь I6
1. Изоляция тенантов + тест (на I1 #100) → 2. supply-chain в CI (#119, на #118) → 3. наблюдаемость/SLO (#117) → 4. DR/бэкапы → 5. 152-ФЗ/ФСТЭК + edge-OTA подпись.
**Выходной критерий:** продукт безопасен (zero-trust, подписанная цепочка, изоляция под тестом), наблюдаем (SLO/OTel/DR) и **комплаентен для гос-сегмента** (152-ФЗ, путь к ФСТЭК).
