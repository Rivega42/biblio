# Продуктовые практики и механики мирового уровня

> Что привнести в продукт (не инженерные практики — те в [BEST_PRACTICES](BEST_PRACTICES.md), а **продуктовые фичи/механики**). Источники: ведущие библиотечные системы (**FOLIO, Ex Libris Alma/Primo, Koha, OCLC WorldShare/WorldCat, EBSCO, BiblioCommons**) и смежные индустрии (e-commerce, стриминг, финтех, логистика, edtech). Помечено: ★ сигнатурный differentiator · ⬆ выше ИРБИС · → инициатива. Привязка к [ROADMAP](ROADMAP.md).

## Сигнатурные differentiators (топ-6 «фишек», на чём выигрываем)
1. **★ Постамат-выдача 24/7 + ячеистое хранение** — last-mile-логистика как у Ozon/Wildberries: бронь → ячейка/постамат → код выдачи, без очереди и часов работы. У ИРБИС нет. → I3.
2. **★ Офлайн-first edge + синхронизация** — работает без интернета, синхронизируется при сети. Критично для регионов/школ РФ. → I4 (ADR-005).
3. **★ Discovery-слой**: FRBR-группировка изданий одного произведения, семантический (векторный) поиск, обогащение (обложки/аннотации/оглавление/отзывы), «читавшие это также брали». Как Primo/Summon + Amazon. → I2.
4. **★ Маркетплейс модулей** (per-tenant включение, как FOLIO-apps / GitHub Marketplace) — коммерческая и продуктовая гибкость «под особенности библиотеки». → I7.
5. **★ Социально-читательский слой** (полки, списки, рейтинги, отзывы, вызовы-челленджи) — как Goodreads/StoryGraph; вовлечение для публичных/школьных. → I2.
6. **★ Вход через ЕСИА/Госуслуги** + единый читательский билет региона — снимает барьер регистрации, «галочка» для гос-сегмента. → I2/I6.

## ИИ-механики (как модуль, с явной приватностью)
- **ИИ-ассистент справки/чат-бот** (вопрос→ответ по фонду и правилам).
- Семантический поиск + рекомендации + «саммари книги».
- Автокаталогизация/классификация, OCR, дедуп/авторитеты с ML.
- Прогноз спроса для комплектования; антиплагиат для ВКР (есть поле 215^W).

## Доверие, приватность, интероперабельность
- **Privacy-by-design / 152-ФЗ** как ценность (минимизация ПДн, право на забвение, прозрачный журнал доступа) ★.
- Стандарты обмена: **Z39.50/SRU, OAI-PMH, SIP2/NCIP, MARC/RUSMARC, BIBFRAME**; гос: **ЕСИА/СМЭВ/НЭБ**, КСУ/6-НК. → I2/I6 (#123, #124).
- Открытость/отсутствие vendor lock-in — отдельное конкурентное сообщение против ИРБИС.

## Кросс-доменные заимствования (резюме)
- **E-commerce** (Amazon/Ozon): фасеты, отзывы, wishlist, «также брали», 1-клик, **постаматы**.
- **Стриминг** (Netflix/Spotify): персонализация, рекомендации, «продолжить», подборки-плейлисты (тематические полки).
- **Финтех** (Stripe): usage-биллинг/метрики, аудит, безопасность, self-serve.
- **Логистика** (Ozon/Wildberries): ячейки/постаматы, last-mile, трекинг брони.
- **Edtech** (Duolingo): геймификация, челленджи, серии — для школ/публичных.
- **Платформы** (GitHub/FOLIO): маркетплейс модулей, API/вебхуки, экосистема.

## Библиотечный домен — специфичные механики (углублённо)
> То, чего нет в кросс-доменном списке, но есть в зрелых библиотечных системах (FOLIO, Alma/Primo, Koha, OCLC, DSpace, NoveList, Leganto) и в библиотечной науке. Это «настоящая» доменная глубина.

**Обнаружение и доступ к ресурсам**
- **Web-scale discovery** — единый поиск по каталогу + ЭБС/э-ресурсам + репозиторию + статьям (как Primo/Summon/EDS).
- **OpenURL / link-resolver** «найти полный текст» (доступность ресурса по контексту), интеграция Google Scholar.
- **Просмотр по классификации (shelf-browse)** — виртуальная полка по УДК/ББК/ГРНТИ, «рядом стоящие» книги.
- **Readers' advisory / read-alikes** (NoveList-стиль: «appeal terms», похожие по настроению/стилю), «что почитать дальше».
- **SEO каталога / schema.org-разметка** — книги библиотеки находятся в Google; постоянные URI.
- **★ AI-навигатор + Cmd+K** — командная палитра поиска (Cmd/Ctrl+K) + ассистент, который по запросу **открывает нужный экран/раздел/запись и подсвечивает** релевантное (deep-link + URL text-fragment + OCR-координаты для подсветки в странице). Работает и в **справочном/help-портале** (docs-as-site, как у современных продуктов), и в самом **приложении/каталоге**. Связан с ИИ-ассистентом (#142) и CommandPalette из дизайн-системы.

**Совместное использование и консорциумы**
- **Сводный (union) каталог** региона/консорциума + **копикаталогизация** из СКБР/WorldCat (заимствование записей).
- **МБА/ЭДД-автоматизация** (ILL): межбиблиотечный абонемент и доставка документов, маршрутизация запросов, unmediated ILL (у нас есть знание БД МБА).
- **Floating collections + межфилиальная доставка/транзит** между адресами сети.

**Управление ресурсами**
- **Периодика/сериалы**: подписка, check-in номеров, claiming пропущенных, union list of serials.
- **ERM** (управление э-ресурсами): лицензии, доступы, **COUNTER-статистика использования**, knowledge base.
- **Курсовые резервы / списки литературы** (Leganto-стиль), **книгообеспеченность** (ВУЗ/ККО — у нас разобрано).
- **Demand-driven / evidence-based acquisition** (DDA/PDA/EBA), **weeding/списание по обращаемости**, инвентаризация/shelf-reading.

**Цифровая библиотека и сохранность**
- **Институциональный репозиторий** (DSpace-подобный, самоархивирование), workflow оцифровки.
- **IIIF-просмотр** редких книг/рукописей/карт (глубокий зум, манифесты), **OAI-PMH-провайдер** (отдаём метаданные).
- **Цифровая сохранность** (OAIS, fixity/контрольные суммы, миграция форматов), **controlled digital lending**.
- **★ Конвейер оцифровки (capture → DAM → OCR → тег → каталог → IIIF):** интеграция **планшетных/книжных/планетарных сканеров** (TWAIN на edge-узле библиотеки) → **DAM** (цифровые объекты: мастер + деривативы/тайлы, метаданные, fixity, object storage) → **OCR** (полный текст + ALTO/hOCR-координаты для поиска и подсветки в странице) → **ИИ-тегирование/классификация** изображений и контента → связь с записью (поля **951/953/955**, БД IMAGE/ESKIZ) → **IIIF-просмотр**. **Цель — заменить отдельный сканирующий продукт** интегрированным DAM (актуально для СПб театральной: книги + отдельные изображения/эскизы).

**Метаданные и авторитетный контроль**
- **Национальные авторитеты** (РНБ/РГБ, LCNAF), связанные авторитеты, тезаурусы.
- **BIBFRAME / RDA / IFLA-LRM**, **Linked Open Data / Wikidata**-обогащение (переход от плоского MARC к связанным данным).
- **ИИ-автокаталогизация** и **дедуп/слияние с ML**.

**Обслуживание читателей (библиотечная специфика)**
- **Семейные / детские / классные (школьные)** аккаунты, возрастной доступ, родительский контроль.
- **Fine-free-тренд** + автопродление; прозрачные правила; читательская этика приватности (минимум истории, **opt-in история чтения** — принцип ALA).
- **Доступность для незрячих**: DAISY/говорящие книги, тифлоформаты.

**Быть стандартом для других систем (сервер, не только клиент)**
- **Z39.50 / SRU-сервер**, **OAI-PMH-провайдер**, **SIP2-сервер** (для вендоров самообслуживания/RFID-станций/ворот) — нас могут искать и обслуживать чужие системы.
- API для **НЭБ / СМЭВ / ЕСИА**, отдача статистики.

**Аналитика, отчётность, программы**
- **6-НК, КСУ, COUNTER**, gate-counts/посещаемость; дашборды + **анонимный бенчмарк с другими тенантами**.
- **События/мероприятия** (регистрация), **бронь залов/оборудования/makerspace** (LibCal-стиль), маркетинг/рассылки новинок.

**Открытость** — открытые стандарты (и возможный open-source-слой) как конкурентное сообщение против проприетарного ИРБИС.
