Поля — дословно из HTML (label + `name` + ИРБИС-префикс). Назначение: реплицировать per-base формы в портале Biblio (issue #255 п.4).

## Как jirbis выбирает базу (важно для архитектуры)
- Набор полей зависит от базы, которая хранится в **серверной сессии** (cookie), НЕ в URL. Постоянного URL на форму конкретной базы нет.
- Левое меню «Базы данных» — ссылки `index.php?option=com_irbis&view=irbis&Itemid=<N>` (открытие выставляет базу в сессии). Разделы ЭК — чекбоксы `set_selected_bases` (`bl_id_array_selected[<id>]`).
- Режимы: Простой=114 · Комплексный=422 · **Расширенный=115**.
- **Вывод для нас:** делаем ЧЕСТНЫЙ per-base роутинг (URL → набор полей), а не stateful-сессию.

## Заметки для реализации
- **3 лёгкие базы** (Либретто, Драмцензура-без-списков, Указатель лит. о СПбГТБ) → общий пресет из 3 полей (КС `KT=FT!`, Автор `A=`, Заглавие `T=`).
- **Диапазон год с–по** только там, где два поля `year1/year2`: ЭК, Периодика, Эскизный фонд. В пьесах/премьерах/хронологиях «год» — одиночный или часть fieldset «Дата события».
- **Списки (L)** наполняются из словарей ИРБИС отдельным AJAX (`get_document_form`/`get_document_language`/…) — у нас брать из `/api/terms` или MNU.
- **`ed_filter`** (наличие эл.копии/ПТ) — сквозной почти везде.
- Архитектура портала: декларативная карта «база → поля[name,prefix,type]», роутинг URL→форма (без серверной сессии jirbis).
