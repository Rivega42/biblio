# Матрица покрытия: наш продукт vs САБ ИРБИС64+ (поблочно, %)

> Честная оценка «сколько функционала ИРБИС закрыто» по каждому блоку. База сравнения — реверс-инжиниринг ИРБИС (`docs/recon/`) и фактически собранное (`irbis-web/`). `%` — функциональное покрытие блока относительно эквивалента в ИРБИС (грубая инженерная оценка, не метрика). Обновляется по мере работы.

## Клиент (по доменам АРМов ИРБИС)
| Блок | % | Что есть у нас | Чего нет (ключевое) |
|---|---:|---|---|
| **Поиск (читательский)** | **40%** | простой (K/A/T/V) + расширенный (поле+значение, И/ИЛИ) + автодополнение из словаря + селектор 31 базы + пагинация | фасеты, навигаторы-классификаторы (ГРНТИ/УДК/ББК-деревья), спецформы (роли PLAY, фонд/опись GUAR, даты), сортировка, история и сохранённые запросы, per-DB префиксы из `[SEARCH]`, профессиональный режим |
| **Карточка / просмотр записи** | **45%** | структурная карточка (автор/выходные/ISBN/язык/УДК/рубрики/экземпляры) + обложка (953) + поиск по рубрике + серверный PFT-бриф + сырой MARC по тумблеру | просмотрщик PDF/полного текста (951/955), переходы по связям 390/488 (журнал→номер→статья), экспорт форматов (RUSMARC/RIS/BibTeX), несколько профилей отображения, галерея изображений |
| **Личный кабинет читателя** | **30%** | вход по билету (RDR `RI=`), формуляр (книги на руках, поле 40) | продление, история заказов/выдач, постановка в очередь, оплата штрафов, уведомления, закладки, сохранённые запросы |
| **Заказ** | **15%** | кнопка заказа под грантом + аудит (заглушка) | реальная запись RQST, очередь, резерв экземпляра/ячейки, выдача по коду из постамата |
| **Каталогизация (АРМ Каталогизатор)** | **20%** | один рабочий лист «Книга» на `DynamicField` (тип поля→контрол, подполя) + сохранение (создать/править) + ФЛК «обязательность» | полные РЛ всех баз, живые словари/.mnu/деревья .tre/авторитетные файлы, реальный ФЛК (`dbnflc`+`!NNN`), импорт/экспорт (MARC/ISO2709), copy-cataloging, глобальная корректировка (GBL), дедупликация, версии записи, пакетные операции, печать |
| **Книговыдача (АРМ Книговыдача)** | **10%** | формуляр читателя (чтение) | выдача/возврат/продление, очередь заказов, бронеполка, должники, штрафы, RFID/ШК на рабочем месте — весь стол книговыдачи |
| **Комплектование (АРМ Комплектатор)** | **5%** | только модель в recon | заказ→поступление→КСУ→списание, поставщики, счета, книги КСУ, перенос в ЭК |
| **Книгообеспеченность (АРМ КО)** | **3%** | только recon | дисциплины, контингенты, расчёт ККО, отчёты |
| **Администрирование (АРМ Администратор)** | **25%** | модель доступа (гранты функция×база×уровень, роли, аудит) + вход guest/staff/reader | управление БД (создать/реорг/копия/бэкап), мониторинг сервера (список клиентов/процессов), правка параметров, UI управления учётками |

**Клиент в целом ≈ 35%** «всех клиентов ИРБИС» (читательская часть ~45%, сотрудническая ~15%). Где сильнее: единый веб-клиент вместо 6 desktop-АРМов + EOL J-ИРБИС.

## Данные, движок, протокол, сервер
| Блок | % | Что есть | Чего нет |
|---|---:|---|---|
| **Структура БД / модель данных** | **50%** | поля/подполя как JSONB + материализованные колонки + FTS + экземпляры + **иерархическое ячеистое хранение (превосходит ИРБИС)** | паритет инверсии/индексов, авторитетные файлы, версии записи, всё многообразие 49 баз и спец-моделей |
| **Поисковый движок / индексация** | **40%** | FTS (PostgreSQL `tsvector` + `pg_trgm` / sqlite FTS5), префиксный и многополевой поиск | полная грамматика ИРБИС (усечение `$`, близость `(G)(F)`, 5 приоритетов), просмотр словаря-инверсии, все операторы |
| **Протокол / адаптер к ИРБИС** | **55%** | Проход Б: подключение, `A/B/N/O/K/C/G/D/H/L` (рег/поиск/чтение/рендер/запись/словарь/файл), числовые коды | остальные `IC_*` (postings, admin `+`, GBL, print, stat, бинарные ресурсы), полная таблица кодов |
| **Собственный сервер (vs сервер ИРБИС)** | **20%** | чтение/поиск/карточка/каталогизация-запись на **PostgreSQL** + ячейки; единый контракт; миграция из ИРБИС | книговыдача, комплектование, администрирование, проводной протокол-совместимость, конкурентность/масштаб, бэкап/реорг |
| **Миграция данных ИРБИС→наш сервер** | **25%** | PoC: 1 база (IBIS) read+write, обложки, ячейки | все базы, авторитетные файлы, инкрементальная синхронизация, сверка, файлы/полный текст массово |

## Где мы УЖЕ лучше ИРБИС (нет аналога / качественно выше)
| Блок | Оценка |
|---|---|
| **UX / доступность / стек** | веб-нативно, адаптив, темы, a11y (ГОСТ/WCAG), без CDN — против Win32-десктопа и EOL J-ИРБИС. **Качественно выше** |
| **Ячеистое хранение + план помещения** | в ИРБИС **нет**: иерархия здание→этаж→помещение→стеллаж→полка→ячейка (разное число ячеек), постамат/книгоприём, top-down план. **Новая возможность** |
| **Архитектура** | единый REST-контракт + два взаимозаменяемых бэкенда (адаптер ИРБИС / свой сервер, strangler), Docker-деплой, конфиг из env. **Новое** |

## Главные пробелы (приоритет довести)
1. Клиент-читатель: фасеты+навигаторы, просмотрщик PDF/полного текста, расширенный ЛК (продление/очередь/оплата).
2. Клиент-сотрудник: книговыдача (выдача/возврат/очередь/бронеполка), полная каталогизация (живые словари/ФЛК/импорт-экспорт).
3. Инфраструктура: **Access-набор клиента на PostgreSQL** (сейчас sqlite), прод-упаковка own-server.
4. Сервер: книговыдача через ячейки/постамат, синхронизация и сверка, расширение охвата баз.
