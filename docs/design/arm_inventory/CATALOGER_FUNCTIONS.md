# Инвентаризация функций — АРМ «Каталогизатор» vs. наше покрытие

> **Цель.** Исчерпывающий пофункциональный реестр настольного АРМ «Каталогизатор» (САБ ИРБИS64+) с явной отметкой покрытия в нашей веб-реализации — чтобы при пересборке **не потерять ни одной функции/экрана**. Каждая дискретная функция = отдельная строка. Без сэмплирования: перечислены ВСЕ пункты меню, ВСЕ ~63 вида словарного поиска, ВСЕ типы РЛ, ВСЕ группы ФЛК, ВСЕ операторы `.gbl`, ВСЕ форматы/формы вывода, ВСЕ утилиты Сервиса.
>
> **Источники (ground truth).** Извлечённый INI: [ARM_CATALOGER](../../recon/deep/reference/arms/ARM_CATALOGER.md) + сырой `C:\IRBIS64\irbisc.ini` (1475 строк, секции `[Main]`/`[Entry]`/`[Display]`/`[Rubricator]`/`[Private]`/`[NAVIGATOR]`/`[USERMODE]`/`[TEXTS]`/`[FULLTEXT]`/`[SEARCH]`; меню — по ключам `SHORTCUT_*`). Форматы: [FIELD_DICTIONARY](../../recon/deep/reference/format/FIELD_DICTIONARY.md), [FLC_VALIDATION](../../recon/deep/reference/format/FLC_VALIDATION.md), [GLOBAL_CORRECTION](../../recon/deep/reference/format/GLOBAL_CORRECTION.md), [PFT_MODULES](../../recon/deep/reference/format/PFT_MODULES.md), [PFT_LANGUAGE](../../recon/deep/reference/format/PFT_LANGUAGE.md). [CAPABILITY_MAP](../../recon/deep/CAPABILITY_MAP.md).
>
> **Наше покрытие.** Спеки: [SPEC_ws1_cataloger](../specs/SPEC_ws1_cataloger.md), [SPEC_I5_staff_arm_parity](../specs/SPEC_I5_staff_arm_parity.md). Движки: [SPEC_engine_pft](../specs/engines/SPEC_engine_pft.md), [SPEC_engine_flk](../specs/engines/SPEC_engine_flk.md), [SPEC_engine_gbl](../specs/engines/SPEC_engine_gbl.md), [SPEC_service_authority](../specs/engines/SPEC_service_authority.md), [SPEC_seeding](../specs/engines/SPEC_seeding.md), [SPEC_api_contracts](../specs/engines/SPEC_api_contracts.md).
> **Код (есть реализация + тесты):** `irbis-web/backend/access/{flk,pft,gbl,authority,catalog,seed,seed_vocab,mnu}.py` (+ `backend/tests/test_*`). Где функция реализована в коде — помечено `[code]`.
>
> **Легенда статуса.** ✅ покрыто (спека + код/контракт, паритетно) · 🟡 тонко (спека есть, но реализация частичная / правила недонесены / только контракт) · ❌ отсутствует (нет ни спеки полевого уровня, ни кода).

### Топ-5 🟡 THIN (приоритет «донаполнения» — наибольший риск недозамены)

1. **Словарный поиск — индекс только 4 префиксов из 63** (3.1.1, 3.2). `catalog.py` строит инвертированный индекс лишь для T=/A=/K=/IN=. Без остальных префиксов (особенно `S=`,`U=`,`R=`,`B=`,`I=`,`V=`,`G=`,`MHR=`) каталогизатор не воспроизведёт привычный отбор. → расширить indexing-spec + сидирование словарей.
2. **ФЛК — 10 правил-канон из ~70** (раздел 10). Движок и семантика (0/1/2, ветвление по 920, ISBN/ISSN-суммы) готовы, но недонесены ключевые проверки: дублетное описание (свёртка БО), предметизация 600/601, роли-число, выбытие 910W/940W, журнальные 934/936. Это и есть «наполненность» ввода. → транскрибировать оставшиеся ~60 правил в ruleset.
3. **Импорт/экспорт MARC + заимствование (Z39.50/СКБР/ЛИБНЕТ)** (5.3/5.5/5.6/5.8, 8.3, ws1 §4/§5). Критично для ежедневной работы. → реализовать FST-мэппинг + ФЛК импорта + Z39.50/SRU-клиент.
4. **Глобальная корректировка — нет превью/отката и части операторов** (7.6–7.13, 7.23). Парсер 22 операторов есть, поле-операторы применяются, но NEWMFN/CORREC/UNDOR/межбазовые UNIFOR и обязательное по ws1 §7 **превью diff + откат** отсутствуют. → достроить исполнение + diff/rollback с аудитом.
