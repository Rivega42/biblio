# Инвентаризация функций — АРМ «Администратор» vs наше покрытие

> **Назначение.** Исчерпывающий реестр (строка на каждую дискретную функцию) десктоп-АРМ «Администратор» САБ ИРБИС64+ — меню **База данных / Сервер / Актуализация / Сервис / Опции / Инструменты** — сверённый с нашим веб-покрытием. Перечислены ВСЕ функции из ground truth, ничего не «просемплировано».
>
> **Ground truth (прочитано дословно):**
> - INI/spec: [ARM_ADMIN](../../recon/deep/reference/arms/ARM_ADMIN.md) (разбор `IRBISA.INI`: `[Main]` ~55 прав-`Item`, `[FullText]`, `[CONTEXT]`) + **сырой `C:\IRBIS64\irbisa.ini`** (полный список `*Item` — единственный достоверный перечень пунктов меню).
> - [FINDINGS_04](../../recon/deep/FINDINGS_04_server_admin.md) (архитектура сервера, `irbis_server.ini`, `CLIENT_M.MNU` — модель прав/АРМов, диспетчерские режимы ядра, установка/обновление, обязанности администратора).
> - [CAPABILITY_MAP](../../recon/deep/CAPABILITY_MAP.md) §2/§10/§11 (server admin, par/ini, реорганизация/актуализация, полнотекст, права-гейтинг).
> - [CONTROL_VOCABULARIES](../../recon/deep/reference/format/CONTROL_VOCABULARIES.md) (редактирование MNU/TRE — 31 словарь, 940 пар).
> - Сырой `C:\IRBIS64\irbis_server.ini` (серверные параметры — секции `[MAIN]`/`[REDIRECT]`/`[FullText]`).
>
> **Наше покрытие (сверено):** [SPEC_ws5_admin](../specs/SPEC_ws5_admin.md), [SPEC_I1_foundation](../specs/SPEC_I1_foundation.md) (#99/#100/#101/#102/#116/#117/#118), [SPEC_I6_security_compliance](../specs/SPEC_I6_security_compliance.md), [SPEC_seeding](../specs/engines/SPEC_seeding.md), [gov/SPEC_gov_statistics](../specs/gov/SPEC_gov_statistics.md), [platform/SPEC_sre](../specs/platform/SPEC_sre.md), [platform/SPEC_billing](../specs/platform/SPEC_billing.md), [platform/SPEC_iac_fleet](../specs/platform/SPEC_iac_fleet.md).
>
> **Особенность.** Администратор ИРБИС работает **напрямую с физической структурой БД** (MST/XRF/L01-N01/IFP: актуализация/реорганизация/импорт-экспорт), а не через поисковый интерфейс. У нас физический уровень = PostgreSQL-операции (миграции / реиндексация alias-swap / бэкап-restore / VACUUM-REORG) + конфиг тенанта. Поэтому многие ИРБИС-функции маппятся «по смыслу», а не «один-к-одному»; в примечании указано проектное соответствие.
>
> **Легенда статуса.** ✅ покрыто спекой с AC · 🟡 названо/частично (нет схемы либо контракта, либо лишь упомянуто) · ❌ не покрыто. **«сверх»** — наша платформенная функция, у десктоп-ИРБИС аналога нет.
>
> Источник перечня пунктов меню — буквальные ключи `*Item=1` из `irbisa.ini` (приведены в колонке «Функция (ИРБИС)»).
