## Сводка «что осталось заполнить реальными реквизитами / доработать»

**Реквизиты (📝) — нужны реальные данные правообладателя (в шаблонах плейсхолдеры):**
- ФИО/наименование, ИНН, ОГРН/ОГРНИП, адрес, контакты правообладателя; структура владения (доли).
- ФИО и данные авторов.
- Точное наименование и версия продукта для регистрации, год создания.
- Банковские реквизиты для оплаты пошлины; данные УКЭП.
- Адрес публичной страницы с документацией; адрес/доступ к демо-экземпляру (сервер в РФ).

**Юрист/патентный поверенный (⚖️):**
- Цепочка прав на код (служебные произведения / договоры).
- Финальная форма и корректность заявления ПрЭВМ; реферат и листинг (объём, состав).
- Формулировки деклараций соответствия п.5 для реестра.
- Выбор правообладателя и налоговая модель.

**Доработка продукта (🔧):**
- Полная русская локализация UI (критерий «л»).
- Хранение исходного/объектного кода, средств компиляции и лицензионных ключей — на территории РФ (критерии «и», «к»).
- Контейнеризированная сборка/демо-стенд для проверки экспертами (сервер в РФ).
- Публичная страница документации.
- (Перспективно) совместимость с ≥2 доверенными российскими ОС (критерий «м», поэтапно).

**Уточнить у первоисточника (❓):**
- Точные коды класса ПО (приказ № 486, действующая редакция).
- Точные лимиты реферата (700 vs 900 знаков) и листинга (50/70 стр.) — приказ № 211.
- Наличие льгот по электронной подаче ПрЭВМ (ст. 333.35 НК РФ).
- Официальный регламент/FAQ включения SaaS-продукта в реестр.
- Поэтапные даты требования совместимости с 2 доверенными ОС (ПП № 1937).
- Действующая механика национального режима в госзакупках (ПП № 1875).
