# РУКОВОДСТВО по установке и администрированию «Biblio»

> Обязательный документ для реестра росПО (эксплуатационная документация / инструкция по установке + администрирование), на русском языке. Наполнен на основе `deploy/README.md` и `deploy/docker-compose.yml`. Реквизиты/версии — заполнить (`<ЗАПОЛНИТЬ: …>`).

## 1. Назначение и состав ПО
- Biblio — система автоматизации библиотек (АБИС): серверная часть (Python: JSON API + отдача собранного SPA) + веб-клиент (React).
- Поставка: контейнеризированный стек, разворачивается через **docker-compose**.
- **Топология стека:** `proxy` (nginx, терминация TLS) → `backend` (Python, API + SPA) → `db` (PostgreSQL 16).

## 3. Установка (одной командой, docker-compose)
> Источник: `deploy/README.md`. SPA собирается внутри образа (`Dockerfile.backend`, многостадийная сборка `vite build` → Python-рантайм отдаёт `frontend/dist`).

```sh
cd deploy
cp .env.example .env          # 1. заполнить секреты (метки CHANGE IN PROD)
sh proxy/gen-cert.sh          # 2. dev самоподписанный TLS-сертификат (для прода — реальный сертификат CA)
docker compose up --build     # 3. поднять db + backend + proxy
```
Открыть **https://localhost** (в dev принять предупреждение о самоподписанном сертификате).

Остановка / сброс:
```sh
docker compose down           # остановить, данные сохранить
docker compose down -v        # остановить + удалить тома (стереть данные)
```

## 4. Конфигурация (переменные окружения)
> Всё через env; ничего чувствительного не зашито в образ. Скопировать `.env.example` → `.env` (gitignore) и задать минимум:

| Переменная | Назначение | Прод |
|---|---|---|
| `POSTGRES_PASSWORD` | пароль суперпользователя PostgreSQL | **СМЕНИТЬ** — длинный случайный |
| `APP_SECRET` | подпись сессий/JWT | **СМЕНИТЬ** — `openssl rand -hex 32` |
| `POSTGRES_USER` / `POSTGRES_DB` | роль / база PG | по необходимости |
| `HTTP_PORT` / `HTTPS_PORT` | порты хоста для proxy | по необходимости |
| `IRBIS_HOST` / `IRBIS_PORT` / `IRBIS_USER` / `IRBIS_PASS` | мост к живому серверу ИРБИС64 (опционально, на время миграции) | ваш сервер |
| `IRBIS_DB_DEFAULT` / `IRBIS_PUBLIC_DBS` | дефолтная и видимые читателю БД | по необходимости |

> Полная модель env — `irbis-web/backend/config.py` и `access/pgstore.py` (`ACCESS_BACKEND`, `ACCESS_PG_DSN`, sqlite-пути `*_DB`). Compose прокидывает их автоматически.

## 5. Провижининг первого арендатора
Access-стор мультиарендный (control-схема + схема-на-арендатора `t_<slug>`). После подъёма стека:
```sh
docker compose exec backend python -m access.provision <slug> \
  --name "Имя библиотеки" --admin admin --password '<надёжный-пароль>' --plan standard
```
> Выполняет `ensure_control_schema()` + создание схемы/миграции/сидинг словарей арендатора против `ACCESS_PG_DSN`.

## 6. Администрирование
- **Арендаторы (тенанты):** создание/настройка (тип библиотеки, бренд/тема, пресет модулей, словари).
- **Пользователи и доступ:** роли, гранты (функция × база × уровень); публично видимые базы — allow-list.
- **Резервное копирование (PostgreSQL, pg_dump):**
  ```sh
  sh deploy/backup/pg_backup.sh                 # дамп с таймстампом → том backups
  KEEP=14 sh deploy/backup/pg_backup.sh         # оставить 14 новейших
  sh deploy/backup/pg_restore.sh                # список дампов
  sh deploy/backup/pg_restore.sh <файл-дампа>   # восстановление (FORCE=1 без подтверждения)
  ```
- **Обновление:** `docker compose pull && docker compose up -d --build` в окно обслуживания; тома сохранять. Обновления узлов подписаны; принудительного обновления из-за рубежа нет (критерий «ж»).
- **Мониторинг/логи:** `docker compose ps` (статусы healthy); структурные логи backend; healthcheck `/api/health` (без авторизации, 200 даже при недоступном ИРБИС).
- **TLS в проде:** заменить самоподписанный сертификат на сертификат CA; включить HSTS (`proxy/nginx.conf`).

## 7. Эксплуатация (роли пользователей)
- **Каталогизатор:** рабочий лист, редактор полей/подполей, live-ФЛК, экземпляры, .gbl.
- **Сотрудник книговыдачи:** выдача/возврат/продление по билету, формуляр, штрафы, бронь.
- **Комплектатор:** заказ → поступление → КСУ → передача в каталог.
- **Читатель (веб-портал):** поиск, бронирование, личный кабинет, уведомления.
- **Администратор:** пользователи/гранты, провижининг, словари, настройки баз.
