## 1. Общие сведения
- **Наименование:** Biblio — система автоматизации библиотек (АБИС).
- **Версия:** `<ЗАПОЛНИТЬ: версия и дата сборки>`.
- **Правообладатель:** `<ЗАПОЛНИТЬ>`.
- **Назначение:** комплексная автоматизация процессов библиотеки — от каталогизации и учёта фонда до обслуживания читателей; мультиарендная облачная платформа для множества библиотек.
- **Область применения:** публичные, школьные, вузовские библиотеки и их сети.

## 2. Архитектура (кратко)
- Мультиарендный SaaS: облачное ядро + локальные узлы (edge) на местах с офлайн-работой и синхронизацией.
- Изоляция данных по тенантам (схема-на-тенанта; в облачных сервисах — `tenant_id` + RLS).
- Модульность: функции подключаются per-tenant/тариф.
- Веб-клиент (SPA/PWA), серверные сервисы, REST-контракт как единый шов.

## 3. Технические характеристики
| Параметр | Значение |
|---|---|
| Языки программирования | Python (бэкенд), TypeScript/JavaScript (фронтенд) |
| Серверная платформа | FastAPI (async), микросервисы |
| Клиент | Веб-браузер; SPA на React; PWA-офлайн |
| СУБД | PostgreSQL (JSONB, полнотекстовый поиск tsvector/pg_trgm) |
| Поиск | PostgreSQL FTS (edge) / OpenSearch (облако, опц.) |
| Хранилище объектов | S3-совместимое (MinIO) — обложки, полные тексты |
| Развёртывание | Docker + docker-compose (узлы); Kubernetes (облако) |
| Совместимость с ОС | Linux-серверы (в т.ч. российские доверенные ОС: Astra Linux и др. ❓ подтвердить ≥2) |
| Интерфейс | На русском языке |

## 4. Функциональные возможности (по модулям)
> Отметить, что входит в регистрируемую версию (✅), что — в развитии. Привести к фактическому состоянию.

**Каталогизация (ядро):**
- ведение библиографических записей (поля/подполя), рабочий лист каталогизатора;
- контроль ввода (ФЛК — формально-логический контроль);
- управление экземплярами фонда;
- авторитетные файлы/словари; глобальная корректировка (.gbl).

**Учёт фонда и размещение:**
- экземпляры, иерархическое хранение, план помещения/ячейки.

**Книговыдача:**
- выдача, возврат, продление по читательскому билету;
- формуляр читателя, очередь бронирования, бронеполка, штрафы;
- работа со сканером/RFID (модуль); критична офлайн-работа.

**Комплектование и книгообеспеченность:**
- заказ → поступление → КСУ (книга суммарного учёта) → списание;
- учёт книгообеспеченности дисциплин (для вузов).

**Читательский веб-портал (OPAC):**
- поиск (несколько режимов), фасеты, карточка записи;
- личный кабинет/формуляр, онлайн-бронирование, уведомления (инбокс), пользовательские полки, постоянные ссылки.

**Администрирование и доступ:**
- управление пользователями, ролями и грантами (функция × база × уровень);
- провижининг арендатора (тенант → схема → словари → администратор);
- аудит-журнал, разграничение доступа.

**Платформа/коммерция:**
- мультиарендность, тарифы/лицензии (entitlements), лимиты использования;
- синхронизация edge↔облако; уведомления; (опц.) аналитика/BI.

**Опциональные модули:** оцифровка/DAM, AI-навигатор по справке, интеграции (OAI/discovery), оплата.
