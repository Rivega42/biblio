## Детальная проверка по подпунктам п. 5

### «а» — Правообладатель из установленного перечня
**Требование:** исключительное право на весь мир и весь срок принадлежит одному/нескольким из: РФ / субъект РФ / МО / российская НКО без преобладающего иностранного участия / российская коммерческая организация без преобладающего иностранного участия (контроль >50% голосов у РФ/граждан РФ) / гражданин РФ.
**Biblio:** правообладатель — **гражданин РФ** либо **российское ООО** под контролем граждан РФ.
**Статус: ⚖️ оформить (единственный содержательный хвост).** Выбрать правообладателя (см. §5 гайда); документально закрепить цепочку прав на код (служебные произведения / договоры отчуждения); для ООО — подтвердить отсутствие преобладающего иностранного участия.

### «б» — Правомерный ввод в оборот и свободная реализация в РФ
**Требование:** ПО правомерно введено в оборот в РФ; экземпляры / права / **услуги по предоставлению доступа** свободно реализуются на всей территории РФ.
**Biblio:** SaaS-продукт — формулировка «услуги по предоставлению доступа» прямо покрывает облачную модель; ограничений на распространение в РФ нет.
**Статус: ✅.** При подаче приложить оферту/прайс/договор как подтверждение предложения на рынке РФ.

### «в» — Выплаты иностранным лицам < 30% выручки
**Требование:** общая сумма выплат в пользу иностранных лиц по лицензионным/иным договорам — менее 30% выручки от реализации ПО за истекший год.
**Biblio:** оригинальная отечественная разработка на открытом стеке (Python, React, PostgreSQL, Docker — свободные лицензии, выплат нет). Иностранных лицензионных платежей нет.
**Статус: ✅.** Зафиксировать в декларации; при наличии платных иностранных зависимостей — подтвердить порог < 30%.

### «г» — Сведения не составляют гостайну
**Требование:** ПО и сведения о нём не содержат гостайны.
**Biblio:** библиотечная АБИС — гостайны нет.
**Статус: ✅.**

### «д», «е» — Требования для средств защиты конфиденциальной информации (СЗИ)
**Biblio:** защита конфиденциальной информации не является основной функцией (это АБИС, не СЗИ).
**Статус: ➖ неприменимо.** *(Обработка ПДн читателей по 152-ФЗ — отдельный комплаенс, не критерий «д»/«е»; см. `docs/design/AUDIT_REPORT_IRBIS64_compliance.md`.)*

### «ж» — Нет принудительного обновления и управления из-за рубежа
**Требование:** ПО не имеет принудительного обновления и управления из-за рубежа.
**Biblio:** архитектура облако-ядро + edge-узлы, **работает офлайн**, обновления edge подписываются и доставляются из контура (см. `docs/design/ARCHITECTURE.md` §7), без внешних CDN/трекеров.
**Статус: ✅.** Зафиксировать в описании процессов ЖЦ (отсутствие зарубежной телеметрии/авто-апдейта/удалённого управления).

### «з» — Гарантийное обслуживание/техподдержка/модернизация российским лицом
**Требование:** гарантийное обслуживание, техподдержка и модернизация (включая модификацию исходного текста) осуществляются российской организацией без преобладающего иностранного участия либо гражданином РФ.
**Biblio:** поддержку и развитие осуществляет сам правообладатель (гражданин РФ / российское ООО) — **российское лицо**.
**Статус: ✅ (оформить документально).** Подготовить гарантийное письмо ([templates/08](templates/08_garantijnoe_pismo_podderzhka.md)), указать каналы и персонал поддержки (увязать с описанием процессов ЖЦ). Роадмап: [ROADMAP_MVP §7.5](../design/ROADMAP_MVP.md).

### «и» — Хранение исходного/объектного кода и компиляция на территории РФ
**Требование:** технические средства хранения исходного и объектного кода и средства компиляции находятся на территории РФ.
**Biblio:** хранение исходного/объектного кода и сборка (компиляция/CI) **осуществляются на территории РФ**.
**Статус: ✅ (задокументировать).** Зафиксировать локацию инфраструктуры (репозиторий, сборка) в описании процессов ЖЦ ([templates/06](templates/06_opisanie_processov_zhiznennogo_cikla.md), раздел «Инфраструктура»). Роадмап: подтвердить/задокументировать.

### «к» — Средства активации/управления лицензионными ключами в РФ
**Требование:** технические средства активации, выпуска, распространения и управления лицензионными ключами находятся в РФ / контролируются российскими лицами.
**Biblio:** биллинг/лицензирование per-tenant; средства управления ключами **в РФ под контролем правообладателя**.
**Статус: ✅ (задокументировать).** Зафиксировать локацию серверов биллинга/активации в описании процессов ЖЦ.

### «л» — Русскоязычный графический интерфейс
**Требование:** GUI реализован на русском языке.
**Biblio:** интерфейс (читательский портал + АРМ-интерфейсы сотрудника) **реализован на русском языке**.
**Статус: ✅ (аудит полноты).** Роадмап: аудит локализации UI на отсутствие непереведённых строк (читательский портал + админ-части). Наличие других языков как опции допустимо.

### «м» — Совместимость не менее чем с 2 доверенными ОС
**Требование:** ПО совместимо не менее чем с 2 ОС, соответствующими требованиям к доверенному ПО (исключения: одна ОС, если правообладатель ПО и ОС в одной группе лиц либо ПО в составе ПАК). Вводится **поэтапно** изменениями ПП № 1937 от 28.11.2025; даты по классам — ❓ уточнить.
**Biblio:** серверный стек (Python + PostgreSQL + docker-compose); архитектура поддерживает Astra Linux на узлах (см. `docs/design/ARCHITECTURE.md` §1, §8).
**Статус: 🟡 задокументировать.** Роадмап: подтвердить и задокументировать работу на ≥2 российских доверенных Linux-ОС из реестра (например, Astra Linux + РЕД ОС / Альт); отслеживать даты вступления требования.

### «н» — Внутригрупповая выручка < 30%
**Требование:** только для госкорпораций и организаций с долей участия РФ/субъекта/МО > 50% и лиц их группы.
**Biblio:** не относится (частный правообладатель).
**Статус: ➖ неприменимо.**

### Признак «ПО в сфере искусственного интеллекта»
**Требование:** при наличии — в реестровой записи проставляется маркер ИИ (отдельных доп. функциональных требований нет).
**Biblio:** ИИ-функции (AI-навигатор/help-портал, ИИ-теги при оцифровке) — **опциональные модули**, не ядро.
**Статус: ➖ / 🟡.** Решить, заявлять ли признак ИИ (зависит от того, входят ли ИИ-модули в регистрируемую версию).

## План действий до подачи в реестр (приоритизированный)

1. **⚖️ Корпоративно-правовое (единственный содержательный блокер):** выбрать правообладателя; закрепить цепочку прав на код; для ООО — подтвердить отсутствие преобладающего иностранного участия. *(критерий «а»)*
2. **🟡 Документация на русском:** функц. характеристики, руководство/установка, описание процессов ЖЦ (шаблоны [04](templates/04_opisanie_funkcionalnyh_harakteristik.md)–[06](templates/06_opisanie_processov_zhiznennogo_cikla.md)) — в т.ч. фиксация локации инфраструктуры в РФ (для «и», «к») и поддержки рос. лицом (для «з»). *(состав заявки)*
3. **🟡 Демо-стенд + публичная страница:** поднять демо-контейнер (сервер в РФ) и публичную страницу с документацией + форма демозаявки — issue [#226](https://github.com/Rivega42/biblio/issues/226); роадмап [§7.5](../design/ROADMAP_MVP.md). *(проверка экземпляра экспертами)*
4. **🟡 Аудит локализации UI** — подтвердить 100% русский. *(критерий «л»)*
5. **🟡 Совместимость с ОС** — задокументировать ≥2 доверенные российские ОС; следить за датами ПП-1937. *(критерий «м»)*
6. **🟡 Класс по классификатору** — выбрать код по приказу № 486 ❓.

> Пп. 2–6 — подтверждение/документирование уже выполненного по существу. Содержательная работа сосредоточена в п. 1 (юрист).
