# Соглашения по документации и репозиторию

> Чтобы документация и визуализации жили в репозитории и не отставали.

## Диаграммы и визуализации — ПРАВИЛО
- **Любая диаграмма/схема живёт в репозитории**, а не только в чате/ином внешнем рендере. Эфемерный рендер (виджет, картинка из обсуждения) обязательно переносится в репо в той же сессии.
- **Формат по умолчанию — Mermaid** в Markdown (```mermaid```): текст версионируется, диффится, рендерится прямо на GitHub, не требует бинарей/CDN. Типы: `flowchart`/`graph`, `sequenceDiagram`, `erDiagram` (модель данных), `stateDiagram`, `gantt` (план), `classDiagram`.
- Диаграмму **встраиваем рядом с контекстом** (в соответствующий design-док), а не в отдельную свалку. Сложные ASCII-схемы можно оставлять как `_текстовый fallback_` под Mermaid.
- Если нужен растровый/SVG-артефакт (например, экспорт для презентации) — класть в `docs/design/diagrams/` и ссылаться из дока; но **первичный источник — Mermaid**.
- Подписи диаграмм — на русском (язык проекта); метки узлов с спецсимволами (`()`, `:`, `/`) — в кавычках `["…"]`.

Текущие диаграммы: топология и C4-контейнеры — [`design/ARCHITECTURE.md`](design/ARCHITECTURE.md) §2–3; владение данными синхронизации — [`design/SYNC_DESIGN.md`](design/SYNC_DESIGN.md) §1.

## Живые индексы — держать синхронными
- [`PROJECT_INDEX.md`](PROJECT_INDEX.md) — точка входа; при добавлении значимого дока добавляем строку (раздел 0 — слой архитектуры/продукта).
- [`CHANGELOG.md`](CHANGELOG.md) — фазовый журнал; на каждый блок работы — запись фазы.
- Обновлять **в той же сессии**, что и сам артефакт, не «потом».

## Git и безопасность
- Инкрементальные коммиты узким `git add <путь>` (не `git add -A`), осмысленные сообщения; пуш в `github.com/Rivega42/biblio` (`main`).
- **Секреты только в env**; `*.ini`/`.env`/`*.db`/вывод зондов — в `.gitignore`; ПДн (реальные данные RDR) не коммитим — только структура/имена ключей.
- Кириллица — только в файлах через редактор (UTF-8); скрипты `.ps1` держим ASCII (консоль PowerShell искажает кириллицу).
