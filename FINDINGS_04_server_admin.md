## Архитектура сервера

Клиент-серверная СУБД над строковым протоколом ИРБИС64 поверх TCP/IP; БД — дерево глубины 2, идеология близка CDS/ISIS, внутренняя кодировка UTF-8 `Сервер_64.doc (txt:90-98)`. Максимум записей 4 294 967 296, файлы MST/инверсные до 4 Тб `Сервер_64.doc (txt:97)`. Число клиентов ограничено лицензией, кроме АРМ Читатель (ограничен только ресурсами ОС) `Сервер_64.doc (txt:92)`.

### Состав модулей `Сервер_64.doc (txt:106-109)`
- `irbis_server.exe` — ядро сервера, Windows-приложение (пиктограмма в трее, диспетчерское меню) `Сервер_64.doc (txt:106,112,129)`.
- `service_64.exe` — запуск как сервиса Windows (`Irbis64_Service`) `Сервер_64.doc (txt:107,117)`.
- `server_64.exe` — процесс обработки; запускается ядром, не для ручного запуска `Сервер_64.doc (txt:108)` (реальный ключ `Server_Exe`, `irbis_server.ini:79`).
- `irbis64.dll` — доступ к БД и форматирование `Сервер_64.doc (txt:109,357)`.
- Консольная альтернатива ядру — `server_64_console.exe` (через `SERVICE_NAME`) `Сервер_64.doc (txt:118,242)`.

### Ядро + процессы обработки
Ядро ведёт сетевой обмен, регистрацию клиентов, администрирование БД, управление процессами, журнал `Сервер_64.doc (txt:366)`. Обработка вынесена в независимые процессы `server_64.exe`, число растёт до регулируемого предела `Сервер_64.doc (txt:358)`.
- Связь ядро-процесс — уникальный WINDOWS MESSAGE + флаг активный/пассивный `Сервер_64.doc (txt:359)`.
- Запрос процессу: файл `!IDClient_RequestNumber`; ответ: файл `IDClient_RequestNumber` (без `!`) `Сервер_64.doc (txt:360-361)`. Обмен через файлы (надёжно) или системную память/mapping (быстро) `Сервер_64.doc (txt:166)`.
- Процессы: долгоживущие (N запросов подряд) и однократные `Сервер_64.doc (txt:367)`. Пакетные (печать, статистика, глобальная корректировка) -> всегда однократный `Сервер_64.doc (txt:371,373)`.
- Виды запросов: специальные, разовые, пакетные, останов `Сервер_64.doc (txt:371)`. Специальные (регистрация, контекст, раз-регистрация, NO-OP) — само ядро `Сервер_64.doc (txt:372)`.
- Клиент строго последователен (нет N+1 до ответа на N) `Сервер_64.doc (txt:365)`; после ответа соединение закрывается `Сервер_64.doc (txt:361)`.

### Три режима `Сервер_64.doc (txt:158-202)`
1. Обычный (`DUPLICATE_SOCKETS=0`,`THREADS_AVAILABLE=0`).
2. Многопотоковый чтение-запись (`DUPLICATE_SOCKETS=0`,`THREADS_AVAILABLE=1`).
3. Многопроцессорный — чтение-запись в самом процессе (`DUPLICATE_SOCKETS=1`); надёжнее `Сервер_64.doc (txt:283-292)`.
