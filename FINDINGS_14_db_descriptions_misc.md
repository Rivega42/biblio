## БД GUAR (гуманитарный архив): назначение, поля, поиск, рабочие листы, связи

**Назначение.** Дистрибутивная тестовая БД GUAR «Гуманитарный архив» — описание архивных материалов; примеры взяты из РГАЛИ, ГАРФ, ЦДАВОВ Украины, РГИА `Описание_БД_гуманитарного_архива_GUAR.doc (txt:5-9)`. За основу приняты рекомендации РУСМАРК (набор и содержание полей), но для части полей с повторным вводом однотипной информации предложены иные решения; экспорт в РУСМАРК «находится в работе» `(txt:21)`.

**Объекты описания (рабочие листы).** Пять уровней иерархии, каждому свой РЛ `(txt:11-16)`:
- Фонд (РЛ **Fond**) — описывает фонд в целом; виртуальная запись, не выдаётся читателю `(txt:12, 49)`
- Опись (РЛ **Opis**) — «инвентарная книга» части фонда; может выдаваться (поле экземпляров 910) `(txt:13, 70)`
- Дело (РЛ **Delo**) — папка с документами; выдаётся (поле 910) `(txt:14, 98)`
- Документ / ед. хр. (РЛ **Guar**) — отдельный документ, рукопись, книга; выдаётся (поле 910) `(txt:15, 119)`
- Лист (РЛ **List**) — аналитическое описание `(txt:16, 122)`

**Связи (ключевая особенность).** Технология ввода автоматически устанавливает иерархические связи. В форматах просмотра показываются прямые отсылки вверх (ко всем более высоким уровням) и вниз (ко всем более низким) с прямым переходом — «путешествие по фонду» `(txt:18)`. Рекомендуемая последовательность ввода «сверху-вниз»: Фонд – (Подфонд) – Опись – (Подопись) – Дело – Документ – Лист `(txt:19)`. Дочерняя запись уже содержит ссылку(и) на вышестоящие, нумерационные данные и сформированный шифр `(txt:72, 100, 121)`. Документ может создаваться напрямую из Фонда/Описи минуя промежуточные уровни — тогда в ОР только реальные связи `(txt:123)`.

**Поля (упомянутые).**
- 910 — экземпляры (Опись/Дело/Документ) `(txt:70, 98, 119)`
- 327 — Схема систематизации описи (ССО), многоуровневая; в каждую Опись `(txt:29, 73)`
- 852 — место хранения/шифр; при отсутствии нумерации в 852 и шифре ставится `???` `(txt:143)`
- 392 — примечание (об особенностях листов книги: дарственные надписи, автографы); из строк формируются записи Листов `(txt:122, 129-131)`

**Поисковый аппарат (специфические средства архивов).** `(txt:23-30)`
- Схема классификации архива — отдельная БД **GUSK** (фрагменты классификации РГАЛИ) `(txt:24, 28)`
- Рубрикатор фондов — отдельная БД **GURF** (фрагменты рубрикаторов архивов) `(txt:25, 27)`
- Обе по структуре идентичны БД **HELP** (Рубрикатор ГРНТИ), могут быть иерархическими, готовятся пользователем `(txt:26)`
- Схема систематизации описи — иерархическая, в каждую Опись `(txt:29)`
- Словари: вид/тип документа, персоны, названия, крайние даты, тематическая классификация, географические рубрики, схема классификации, ключевые слова и др. `(txt:30)`

**Технология ввода — «Оперативные режимы» (ОР).** При сохранении родителя возникает накладка с кнопкой опросного листа `(txt:45, 51)`. В опросном листе — вид нового документа (своё умолчание на каждом уровне), затем строки из оглавления ИЛИ номера новых записей (перечислением/интервалом) `(txt:55-60, 80-87, 107-112, 127-133)`. По «Ввод» — глобальная корректировка; сообщение о новых записях и об отвергнутых дублетных номерах `(txt:65, 91, 115, 135)`. Кнопки ОР: переход к новым записям сеанса или ко ВСЕМ записям уровня; «Х» удаляет «Перейти к новым» `(txt:66, 92, 116, 136)`. Опросный лист Описи задаёт раздел ССО для всех создаваемых Дел/Документов `(txt:83, 87)`.

**Просмотр.** Проход по иерархии Фонд→листы с завершением экспортом в формате ИРБИС или РУСМАРК `(txt:158)`. Различается число по «Количественным характеристикам» (каталогизатор) и реально введённое в ЭК (красные ссылки) `(txt:167-168)`. Ссылки: «Просмотреть описи фонда», «Схема систематизации описи», «введено в ЭК» `(txt:171, 180, 230)`.

**Сопутствующее.** Для старта создать БД GUAR, GUSK, GURF и скопировать директории из дистрибутива `(txt:35-39)`. Таблица специфических форматов — файл `guar_PFT.doc` `(txt:32)`.

## Фасетный поиск (механика)

**Определение.** Фасет — поисковый инверсный термин из верхних элементов списка, отсортированного по числу вхождений терминов данного типа в результат поиска. Тип фасета = тип инверсного термина, задаётся префиксом (напр. `A=`). Каждому результату — свой набор фасетов `Построение_и_использование_фасетов_при_поиске.doc (txt:4)`. Инструмент сжатия выдачи по нескольким типам терминов; полезен при большой выдаче/нечёткой задаче; по структурированной библиографии — «извлечение знаний» `(txt:5)`.

**Построение.** Инверсные файлы строятся при загрузке словаря БД ЭК `(txt:6)`.

**Параметры (`irbisa.ini`, АРМ Администратор).** `(txt:7-9)`
- `CREATE_CELLS_ON_IF_REORGANIZATION=1` (умолч.0) — строить фасеты при перезагрузке словаря
- `MAX_POSTINGS_NUMBER_IN_CELL` (умолч.5) — мин. число ссылок термина для вхождения
- `MAX_LENGTH_CELL_TERM` (умолч.2) — мин. длина фасета (термина без префикса)

**Файлы.** `(txt:10-11)` `dbname.cells` — список типов (префиксы); `dbname.cellN` — бинарный инверсный файл типа N, структура `MFN INDEX` (одному MFN — несколько INDEX); файл доп.терминов словаря для расшифровки: `=Префикс=INDEX=Термин` (`=A=3203=SHORT`).

**Пример.** ЭК ГПНТБ, `U=5$` → 164910 записей; поиск 100 ms, расчёт фасетов 150 ms; фасеты по авторам/годам/издательствам `(txt:12-26)`.

## Статистические формы (как создаются)

**Определение.** Двухмерная таблица — распределение документов БД по результатам расформатирования по двум форматам; формируются в АРМах «Каталогизатор» и «Книговыдача» `Инструкция_по_созданию_стат.форм.doc (txt:3)`.

**Файл `.STF` (текстовый, одноимённый форме), строки `(txt:4-20)`:**
1. Имя Excel-шаблона (`.xlt`, без расширения) — размерность/надписи/вёрстка/суммирование
2. Формат вертикали
3. Справочник `.mnu` вертикали (число значений = число строк переменной части)
4. Формат горизонтали
5. Справочник `.mnu` горизонтали (число значений = число столбцов)
6. Координаты верхнего левого угла переменной части (колонка,строка)
7. РЛ подполей `.wss` для ограничений + метка модельного поля (может быть пустой)
8. Формат переменного заголовка (литералы + модельное поле; может быть пустой)
9. Координаты ячейки заголовка (может быть пустой)
10. (с 2017.1) Формат: название верт. распределения + название элемента
11. (с 2017.1) Формат: название гориз. распределения + название элемента

**Версии.** С 2019.1 при `STFHTML=1` (профиль пользователя) строки шаблона/координат не используются — `RELEASE_19_1.doc` `(txt:22-24)`.

**Размещение/регистрация.** `.STF` и связанные файлы — в директории БД `(txt:26)`. Имя формы — в справочнике параметра `STFMNU` (`[MAIN]` INI «Каталогизатора», умолч. `STF.MNU`) `(txt:27)`.

**Пример Form3 (БД IBIS) — виды × разделы знаний за период.** `(txt:29-123)` Шаблон `Form3.xlt`; вертикаль `vdu_SF.mnu` (виды 01-12 + `*` ПРОЧИЕ — значения вне справочника), горизонталь `rzn3_SF.mnu` (15 разделов); формат вертикали — вид документа 900^B, формат горизонтали — раздел 60, фильтр по дате ввода (первое повторение 907^A в интервале модельного поля 1000 `^A`/`^B`); РЛ `999.wss` (A/B = начало/конец ГГГГММДД), метка 1000; переменная часть от `4,7` (D7), сумматоры C7:C17 (гориз.) и C18:R18 (верт.); заголовок в `1,1` (A1) `(txt:66, 101-123)`.

## Дополнительные материалы ИРБИС64+ (перечень возможностей/настроек)

**Назначение.** ИРБИС64+ (с 2018.1) — ведение электронной библиотеки; включает ИРБИС64 + ИРБИС64 ПБД плюс функционал ЭБ `Дополнительные_материалы_ИРБИС64_.doc (txt:24-27)`.

**Функционал ЭБ.** `(txt:28-42)` Полнотекстовый поиск с авто-разбиением на страницы и пословной индексацией; ранжирование по релевантности (документы и страницы); постраничный просмотр с маркировкой слов, навигацией по релевантным страницам/оглавлению/закладкам; права доступа; учёт обращений; полные тексты — распознанные PDF, библиография — RUSMARC; поиск и по библиографии («нулевая страница»), морфология русского. Загрузка читателем своего текста (умолч. БД **VKR**) `(txt:80)`.

**БД ЭК — новое поле 955** (полный текст, неповт.) `(txt:81-92)`: `A` имя PDF (<=64, уникальное, без запятых/кавычек/двойных подчёркиваний/скобок/кириллицы), `B` идентификатор права (БД RIGHT), `N` число страниц (авто). Путь — 11 строка `.par`, относительный со слэша (обратные слэши). Вводить 955 в последнюю очередь. Не путать с 951 `(txt:82, 89-92)`.

**БД RDR — новое поле 130** (Пароль, неповт., необязат.) `(txt:93-94)`.

**БД RIGHT (типовые права).** `(txt:95-137)` Поля: **1** идентификатор (уник., инверт. `I=`); **2** общий период (`D`/`E` ГГГГММДД); **3** право (повт.: `A` элемент по `3A.MNU` — 01 идентификатор/02 категория/03 IP/04 домен/05 факультет/06 семестр/07 специальность; `B` значение с маской `*`; `C` право по `3C.MNU` — 0/1/2; `F` число-ограничение; `G` единицы `3G.MNU` пусто=страницы/1=проценты; `D`/`E` даты периода права); **4** название.
