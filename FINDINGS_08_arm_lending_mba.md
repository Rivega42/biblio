## Книговыдача: назначение и БД
Файл WBKNJ описывает СПИСАНИЕ книг/журналов, не циркуляцию: ввод записи КСУ выбытия -> установка номера КСУ -> отбор и регистрация выбытия -> завершение `WBKNJ.doc (txt:2-6)`.
БД: БД КОМПЛЕКТОВАНИЯ (ведётся КСУ выбытия, запись=документ о партии) `WBKNJ.doc (txt:8-9)`; БД КАТАЛОГА (корректировка статусов экземпляров) `WBKNJ.doc (txt:5,31,37)`.
Идентификация экземпляра: штрих-код или инвентарный номер `WBKNJ.doc (txt:40)`; повторяющееся поле «Сведения об экземплярах» `WBKNJ.doc (txt:57-59)`.
БД RDR/RQST/IBIS/RDR_ARH как модель книговыдачи здесь НЕ описаны -> TODO(recon #0801).

## Модель циркуляции (запись читателя -> заказ -> выдача -> возврат -> продление; статусы; задолженность)
Модель «заказ -> выдача -> возврат -> продление» описана в контексте МБА (IrbisMBA). Прямой модели книговыдачи РДР в файлах нет -> TODO(recon #0802).
Этапы МБА: оформление -> регистрация этапа -> выдача -> возврат -> продление -> расчёты -> итоговое списание -> архивация `IrbisMBA.doc (txt:84,119-138)`.
- Абонент: идентификатор абонента — уникальный ключ связи с заказами; если абонент читатель — указывается идентификатор в БД читателей `IrbisMBA.doc (txt:151,193,196)`.
- Заказ: режим «Оформление заказа», номер+дата, этап авто; при сохранении этап «Оформление заказа» в поле 30, техпуть в поле 942 `IrbisMBA.doc (txt:208,212,215)`.
- Выдача (модуль «Выдача»): проверка ZkVid=ODER; статус против CanStatusLand=M0 P1 B1 B2 B3 M4 K1 K2 K3 K4 K5 K6 `IrbisMBA.doc (txt:243-245)`; код операции DoLandMnu=DoLand.Mnu / DoLandPft=DoLand `IrbisMBA.doc (txt:249-251)`; выполнение RunLandGbl=RunLand.gbl, вход — поле 991 `IrbisMBA.doc (txt:264-267)`. Коды: оригинал MBASTATUSLAND=O1, копия MBASTATUSLANDCP=O3 `IrbisMBA.doc (txt:270-271)`; перенаправление MbaStatusLib=M4 `IrbisMBA.doc (txt:259,272)`. При выдаче (O1,O3,O4) — поле 40 в записи абонента `IrbisMBA.doc (txt:273)`.
- Возврат (ReturnGbl=Return.gbl): дата факт.возврата в поле 40; код S1 (в срок) MBASTATUSRETN / S2 (не в срок) MBASTATUSRETNNOT `IrbisMBA.doc (txt:279-281)`.
- Продление (ProlongGlob=Prolong.Gbl): «должник»=истёк срок; новая дата или дни (prolong.mnu); код MBASTATUSLONG=O2 `IrbisMBA.doc (txt:286-294)`.
- Задолженность: строки «Долг»/задолженные заказы выделены цветом `IrbisMBA.doc (txt:278-279,287)`.
- Связь с каталогом/RDR: CorrCatAble — статус экз./кол-во выданных и «возврат» в ЭК `IrbisMBA.doc (txt:274,282)`; CorrRdrAble — поле 40 записи RDR, место выдачи MBA, «возврат» `IrbisMBA.doc (txt:253,275,283)`.
Статусы экземпляра (из WBKNJ): '0' ИУ в фонде; '6' списан; 'C' филиал ЦБС; 'U' БИУ `WBKNJ.doc (txt:60-64,77,86)`. Смена '0'->'6' при выбытии `WBKNJ.doc (txt:61,86)`. Кодификатор «свободности» для выдачи не описан -> TODO(recon #0803).

## МБА: модель и БД (MBA/MBA_ARH), процессы заказа между библиотеками
Назначение: ведение БД абонентов МБА; регистрация/отслеживание этапов; обслуживание в ЭК и у сторонних фондодержателей; подсчёт стоимости; связь с ЭК и БД читателей `IrbisMBA.doc (txt:142-147)`.
БД: MBA (записи Абонент и Заказ) `IrbisMBA.doc (txt:150-152)`; БД ЭК (источник + регистрация выдачи/возврата) `IrbisMBA.doc (txt:154)`; БД читателей RDR (выдача/возврат, если абонент — читатель) `IrbisMBA.doc (txt:155)`; БД заказов RQST (личный кабинет) `IrbisMBA.doc (txt:152,222-229)`; MBA_arh/MBA_ARH (архив) `IrbisMBA.doc (txt:340,378,387)`.
Заказ из RQST: SearchIni=rqstMBA.ini; вид поиска «Место выдачи»=МБА; отбор -> «Новый» подаёт заготовку; номер заказа; вне фонда — поле 902; этап «Оформление заказа»/«Заказ зашифрован»; удаление RQST при DelZakRqst=1 (по умолч. запрещено) `IrbisMBA.doc (txt:222-229)`.
Между библиотеками: при MbaStatusLib=M4 — перенаправление, список библиотек sigl.mnu, код фондодержателя в поле 942 `IrbisMBA.doc (txt:259,272)`.
Расчёты: по записи CENA и формату SubfldDL; итоговое списание PriceItogGlog=Itog.gbl, код MBASTATUSITOG=H1, проверка CanItogSpis=S1 S2 O3 O4 Z1; остаток поле 35^D `IrbisMBA.doc (txt:321-337)`.
Архивация MBA_ARH: ToArhiv.gbl (автоввод); абонент — перенос выдач/возвратов, полей 42/39, пересчёт 41; заказ переносится только выполненный (MBASTATUSITOG=H1), дубль + поле 300 `IrbisMBA.doc (txt:339-368)`.
Выходные формы: AbOder (список заказов) `IrbisMBA.doc (txt:371-378)`; StatMBA (статистика, 942^C, рабочая БД WORK) `IrbisMBA.doc (txt:383-388)`.
