# FINDINGS 02 — Статические конфиги и профили баз данных

> Выведено напрямую из конфиг-файлов в `C:\IRBIS64\` и `C:\IRBIS64\Datai\` (CP1251 → UTF-8). Ссылки указывают реальные файлы. Значения IP/паролей/ПДн НЕ приводятся (только структура). Дополняет `FINDINGS_04` (сервер по документации) и `FINDINGS_10` (общесистемные механизмы по доку 2022).

## 1. Архитектура каталога данных и типы конфиг-файлов

Сервер: `irbis_server.ini` задаёт `DataPath=C:\IRBIS64\DATAI\`, `workdir`, таблицы сортировки `ACTABPATH`/`UCTABPATH`, реестры БД `DBNNAMECAT=dbnam1.mnu` (каталогизатор), `DBNNAMECATR=dbnam3.mnu` (читатель), список клиентов `CLIENTLIST=client_m.mnu`, порт `IP_PORT=6666`, формат краткого описания `BriefPft=BRIEF`, пустую БД-шаблон `EmptyDBN=BLANK` (`irbis_server.ini`).

Каждая БД — отдельная папка в `Datai\<DB>\` + файл `Datai\<DB>.PAR`. Типы файлов в папке БД:

| Расширение | Назначение |
|---|---|
| `.PAR` | реестр путей к файлам БД (10 ISIS-слотов + полные тексты) |
| `.MST/.XRF/.L01/.N01/.IFP/.CNT` | данные (мастер-файл, ссылки, словарь, инверсия) — **не анализируются** |
| `.FST` | таблица инвертирования: какие поля → словарь, с какими **префиксами** |
| `.PFT` | форматы показа/вывода (язык форматирования ИРБИС) |
| `.WS` | рабочий лист ввода (многостраничная форма полей) |
| `.WSS` | рабочий лист подполей (встроенная форма для одного поля) |
| `.OPT` | таблицы выбора ресурса по типу записи (поле 920) |
| `.MNU` | меню/справочники (плоские списки «значение → подпись») |
| `.TRE` | иерархические справочники (дерево) |
| `.TAB`/`.TBU`/`.TBG` | таблицы переформатирования/соответствия |
| `.STF`/`.STW` | сценарии статистических форм |
| `.GBL` | задания глобальной (пакетной) корректировки |
| `.FMT`/`.HDR`/`.SRW`/`.XLT` | форматы/заголовки/выгрузки/переключатели |

## 2. PAR — пути к файлам базы

`<DB>.PAR` — нумерованные строки `N=путь`. Строки 1–10 = размещение десяти стандартных ISIS-файлов БД, строка 11+ = доп. пути (полные тексты). Пример `IBIS.PAR`: `1..10=.\datai\ibis\`, `11=.\DATAI\ibis\texts\` (каталог полных текстов). `RDR.PAR`/`RQST.PAR` — все слоты в одной папке (`IBIS.PAR`, `RDR.PAR`, `RQST.PAR`). Полные тексты IBIS — в `Datai\IBIS\Texts\` и `Ibis--text\`.

## 3. FST — инвертирование и поисковые префиксы

Формат строки FST: `<метка> <метод> <формат>`. **Префикс поиска — это литерал перед `=`** в формате (`'K='`, `|T=|`, `"A="`). Пример (`Datai\IBIS\ibis.fst`):
- `1200 8 MHL,'/K=/'(v200^a,|%|d200/)` — заглавие/ключевые слова → префикс `K=`;
- `700 0 MHL,...'A='v700^a...` — индивидуальный автор → `A=`;
- `200 0 ...'T=',&unifor("9"v200^a...)` — заглавие → `T=`.

Коды метода инвертирования (по `ibis.fst`, 451 строк индексации): **0** — по строкам формата (335 строк), **8** — поэлементно из формата с разбором подполей (100), **6** — по словам (14), **5**, **9** — спец. (по 1). Метод записывается первым после метки; комментарий `/* …` группирует блоки (`IBIS_BIB_CARD`, `IBIS_KL_TIT`, `IBIS_AUT`, `IBIS_DUBL`…).

`ibis.fst` определяет **109 различных префиксов**, среди них: `T`,`TP`,`TJ`,`TS`,`TH`,`TMN`,`TV`,`TI` (заглавия/варианты), `A`,`AZ` (авторы), `K` (ключевые слова), `M`,`MK`,`MI`,`MR`,`MS`,`MPR` (коллективы/мероприятия/MeSH), `S`,`SU`,`PS` (предметные/издат. индекс), `I`,`IN`,`INS`,`INK`,`IZ` (инв./штрих-коды), `B` (ISBN/ISSN), `V` (вид документа), `G`/`GS`/`GEO` (год/географ.), `U` (УДК/ББК), `R`/`RIN` (рубрикатор/др. классификация), `P` (персоналия), `O` (организация), `N`/`NA` (носитель/название), `DP`/`DR`/`DZ` (даты), `KP`/`KOMP`/`PLANZK` (комплектование), `GUID`,`TXT` (полный текст), `DNJ` (журналы) и др. Полный список с частотами — см. источник `Datai\IBIS\ibis.fst`.

## 4. INI / секция [SEARCH] — виды поиска → префиксы

Соответствие «видимый пользователю вид поиска → префикс словаря» задаётся в INI-файле АРМа (профиль пользователя, отдаётся `IC_reg`). Структура секции `[SEARCH]` (пример `irbisr.ini`, `ItemNumb=23`): на каждый вид поиска N — ключи `ItemNameN` (подпись), `ItemPrefN` (префикс), `ItemDictionTypeN` (0 словарь / 1 меню / 2 иерархия), `ItemLogicN` (логика по умолчанию), `ItemMenuN` (справочник F-меню), `ItemF8ForN` (формат расширенного ввода/F8), `ItemAdvN` (авторитетный файл `<db>,<преф>,@<формат>`), `ItemHintN`, `ItemTrancN` (усечение), `ItemExactlyN` (точное совпадение).

## 5. PFT — форматы показа; выбор формата по типу записи

**Меню форматов** (`Datai\IBIS\PFTW.MNU`, выбирается параметром `PFTMNU` АРМа): пары «имя формата → подпись», включая экранные и HTML-варианты (суффикс `_H`): `KN`=БО книг, `MN`=БО томов МН, `ASP`=Аналитическое описание, `JW`=БО журнала, `NJW`=Описание номера, `BRIEF`=краткое БО, `UMARCW`=UNIMARC, `USMRCW`=USMARC, `RMARCW`=RUSMARC, `INFOW`=информационный, `DUBLINC_`=Dublin Core, комплекты каталожных карточек (`!KKKOW` основная КК, `!KKKDW` добавочные, `!KKKC` ссылочные, `!KKKT` топографическая, `!KKKFW` формуляры, `!KKKFH` штрих-коды) и др.

**Выбор формата по умолчанию** — `.OPT` по полю 920: `PFTW.OPT` (экран): `J→JW`, `NJ→NJW`, `SPEC→MN`, `ASP→ASP`, `MUSP→MUSI`, по умолчанию `+++++→KN`. Реестр читателя использует `PFTOPT=PFTW_H.OPT` (HTML). Формат `.OPT`: строка1=метка поля-ключа (920), строка2=длина ключа (5), далее пары `<значение> <ресурс>`, `+++++`=по умолчанию, `*****`=конец.

**Экранные vs веб(HTML)-форматы**: HTML-варианты помечены суффиксом `_H`/`W_H` и содержат HTML-разметку; для web-шлюза используются веб-PFT со встроенными командами шлюза (см. `FINDINGS_09`). Краткое описание в списке результатов — `BRIEF.PFT`.

## 6. Рабочие листы (.WS) и подполя (.WSS); поле 920 = тип записи

**Выбор рабочего листа** — `WS.OPT`/`WS31.OPT` по полю 920: `PAZK→PAZK`, `PVK→PVK`, `SPEC→SPEC`, `J→!RPJ`, `NJ/NJP/NJK→!NJ`, `AUNTD→AUNTD`, `ASP→ASP`, `MUSP→MUSP`, по умолчанию `+++++→IBIS` (`Datai\IBIS\WS.OPT`). Версии `…31`/`…42`/`…71` — варианты форм (`WS31.OPT`: `PAZK→PAZK42`, `J→!RPJ51` и т.д.).

**Поле 920** — селектор типа/уровня записи (имя рабочего листа). Значения в IBIS: `PAZK`/`PVK` (книги/монографии), `SPEC` (сводное многотомное), `J` (журнал), `NJ`/`NJP`/`NJK` (номер/подшивка журнала), `ASP`/`AUNTD` (аналитика — статьи), `MUSP` (музейный предмет), `BOUNI`/`SZPRF`/`IBIS` (упрощённые) (`WS.OPT`, `WS31.OPT`, `PFTW.MNU`).

**Формат .WS (рабочий лист)** — пример `PAZK.WS`: строка1=число страниц (9), далее имена страниц («Дублетность», «Основное БО», «Расширенное», «Редкие», «Коды», «Экземпляры», «Систематизация», «КО», «Технология»); затем по каждому полю блок: `метка / подпись / повтор / управление / тип-редактора / связанный .WSS / формат-FLC (@…) / подсказка`. Напр.: поле `700` «1-й автор - Заголовок», редактор 5, подформа `701d.wss`, FLC `@!700d`.

**Формат .WSS (подполя)** — пример `481.WSS` (16 подполей): строка1=число подполей; по каждому: `код подполя (X,2,3,9,C…) / подпись / 3 флага / тип-редактора / средство ввода / подсказка`. Средство ввода: `,A=,@!amovu` (словарь по префиксу `A=` + контроль-формат `!amovu`), `gor.mnu\\1` (меню, колонка 1), `df.mnu`, `il.mnu`. Тип-редактора: 0=текст, 1=выбор из меню, 2=ввод с помощью словаря, 7=меню (роли). Это и есть «правила полей».

## 7. Меню/справочники и интеграция авторитетных файлов
`.MNU` — плоский справочник (пары строк «код / расшифровка», `*****` конец): напр. `vd.mnu` (виды), `jz.mnu` (языки), `str.mnu` (страны), `nos.mnu` (носители), `sigl.mnu` (сигла). `.TRE` — иерархические (рубрикатор, навигатор `DBNAM2.TRE`). Авторитетные/нормативные данные вынесены в отдельные БД (см. §8): индивидуальные/коллективные авторы, предметные заголовки, тезаурус, АПУ к УДК/ББК. Связь через `ItemAdv`/`Int`-блоки INI и `&uf`/`&unifor`-вызовы в FST/PFT.

## 9. Связи между записями
- **Уровни/иерархия в IBIS** через поле **920** (тип записи) и поля связи: журнал↔номер↔статья (поиск по связи `Scnt`, форматы `!NJ`/`!RPJ`), многотомники (461/46 — общая часть и тома), приплетённые (481), издание-источник аналитики (463/963).
- **Каталог↔Комплектование**: поле 66 (перенос), 901/938 (заказ/поступление переносится в ЭК).
- **Каталог↔Книговыдача**: поле 910 (экземпляры, статусы), 999 (число выдач); RQST связывает RDR (поле 30/читатель) и IBIS (903 шифр, 910 экземпляр) — теги в `irbisr.ini` [Request]: `RQSTTAGREADER=30`, `RQSTTAGSHIFR=903`, `RQSTTAGEKZ=910`.
- **Авторитетный контроль**: подполе «3» (номер авторитетной записи) связывает IBIS с ATHRA/ATHRC/ATHRS.
- **Книгообеспеченность**: VUZ ↔ IBIS (через индексы/шифры) ↔ RDR (контингент).

## 10. Полные тексты и права
- Хранение: `<DB>.PAR` строка 11 → каталог текстов (IBIS: `Datai\IBIS\Texts\`). Поля-указатели: `TagFullText=951`, `TagImage=950`, ссылка/URL в 951 (`irbisr.ini` [Main]).
- Извлечение/кэш PDF: `irbis_server.ini` [FullText] — `PDFExtractTime`, `PDFPageNumber`, `STORE_TXT_PAGES_IN_CASH`, `ExtractPDFPageMode`, `ftcachebuilder`. Актуализация ПТ при актуализации словаря — `FULLTEXTACTUAL=1`.
- Права: БД **RIGHT** (типовые права доступа к полным текстам, префиксы `I=`,`II=`), БД **LICH** (личные данные при просмотре/скачивании). В web — параметр `Show_ed` (см. `FINDINGS_09`).
