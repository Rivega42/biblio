# Справочник КЛАССИФИКАТОРОВ и ТЕЗАУРУСОВ САБ ИРБИС64 (reconstruction-grade)

> Фаза 2. Источник — реальные конфиги БД (CP1251 → UTF-8) в `C:\IRBIS64\_recon\_work\cfg_txt\<DB>\`.
> Все факты — со ссылкой на файл. ПДн опущены. Неясное помечено `TODO(recon #CLS-NN)`.
>
> Охват: **RSUDC** (УДК — рабочие таблицы), **RSBBK** (ББК — рабочие таблицы), **HELP** (рубрикатор ГРНТИ), **MESH** (медицинские предметные рубрики MeSH), **TEZ** (тезаурус ИРБИС), **URUB** (универсальный тематический навигатор/тезаурус).

## 0. Обзор и общая архитектура

Шесть служебных БД-навигаторов. Делятся на три типа по модели данных:

| Тип | БД | Модель | Ключ навигации |
|-----|-----|--------|----------------|
| **Иерархический классификатор** (код→текст) | RSUDC, RSBBK | Индекс (код) + текст рубрики, вышестоящий индекс | префикс `G=` (нижестоящие), поле «вышестоящий индекс» |
| **Иерархический рубрикатор/предметник** | HELP (ГРНТИ), MESH | Код рубрики + наименование, родитель/потомок по коду | `G=` / `R=` / `M=` |
| **Тезаурус дескрипторов** (термин-центричный) | TEZ, URUB | Дескриптор/недескриптор, BT/NT/RT-связи через текст термина | `U=` (нижестоящие), `T=`/`I=`, `C=`/`CK=`/`CA=` |

Все шесть подключаются к каталогу (IBIS) и читателю как **«окна-навигаторы» (Wn)** через единый список `WnLink` в `irbisr.ini` (web/читатель) и `irbisc.ini` (АРМ Каталогизатор) — см. раздел 7. Навигатор — это набор web-форматов (`*_wn.pft`), которые строят HTML-дерево; пользователь, отметив рубрику чекбоксом `CHECKINDEX`, переносит её код в поле ввода каталогизатора (механизм `CHECKNAME=CHECKINDEX`).

Общая модель web-навигатора (одинакова у всех): **TOP-формат** (`*_top_wn`, шапка + форма поиска фрагмента) + **строчный формат узла** (`*0_wn`, одна строка дерева с иконкой `folder.gif`/`text.gif`/`textfolder.gif` и гиперссылкой) + **формат узла-страницы** (`*_wn`, разворот одной рубрики с предками/потомками/ссылками) + **BOTTOM-формат** (`*_bottom_wn`) + **SEARCH-формат** (`*search*_wn`, превращает отобранный код в поисковое выражение для IBIS) + **SELECT-формат** (`*select_wn`, список отмеченных).

## 1. PAR / идентификация

Каждая БД зарегистрирована PAR-файлом в `C:\IRBIS64\Datai\`; все 11 путей (строки 1..11) указывают в один каталог данных БД.

| БД | PAR-файл | Каталог данных (пути 1..11) | Назначение |
|----|----------|------------------------------|------------|
| **RSUDC** | `RSUDC.par` | `.\datai\RSUDC\` | Рабочие таблицы классификации **УДК** (ГПНТБ) |
| **RSBBK** | `rsbbk.par` | `.\datai\rsbbk\` | Рабочие таблицы классификации **ББК** |
| **HELP** | `HELP.par` | `.\datai\help\` | Рубрикатор **ГРНТИ** (+ доп. справочники) |
| **MESH** | `MESH.par` | `.\datai\mesh\` | Медицинский тезаурус **MeSH** (предметные рубрики) |

## 2. FST — поисковые индексы

> Метод (2-я колонка): `0` — по формату (PFT-выборка из полей); `8` — разбор по разделителям (режим словаря из текста, `/.../`, `|%|`). Метка (1-я колонка) — номер поля. Источник — `<DB>.FST`.

### 2.1 RSUDC — `RSUDC.FST`

| Метка | Метод | Префикс | Что индексируется |
|-------|-------|---------|-------------------|

| 3 | 0 | `U=` | Заглавный индекс v3, **если** v1 начинается с `0` или пусто (ведущая запись) |
| 3 | 0 | `N=` | Тот же v3 как «запись продолжения», **если** v1 начинается с `1` |
| 4 | 8 | `K=` | Ключевые слова из заглавной рубрики (поле 4) |
| 5 | 8 | `K=` | Ключевые слова из расширения загл. рубрики (поле 5) |
| 10 | 0 | `S=` | v3, если есть отсылки «смотри» (поле 10 не пусто) |
| 11 | 0 | `SA=` | v3, если есть ссылки «смотри также» (поле 11 не пусто) |

## 4. Модель иерархии и навигации (дерево, родитель↔потомок)

### 4.1 RSUDC / RSBBK — иерархия по полю «вышестоящий индекс»

- **Узел** = запись классификатора; код в поле 3 (`U=`), родитель — в поле 35 (RSUDC) или через поле 12 «Раскрытие» (RSBBK), индексируется как `G=<код-родителя>`.
- **Вниз** (потомки): отбор по `G=<код-текущего>` (FST метка 35→`G=`). В формате узла `&unifor(|D,!U=|v12|!,@ref2_wn|)` разворачивает раскрытие; иконка `folder.gif` если поле 12 (раскрытие) не пусто, иначе `text.gif`.
- **Вверх** (родитель): поле 35 → переменная `v1111`; ссылка `<A href="IRBIS:1,,,udc_wn,U=<v1111>">` + текст родителя через `&unifor(|D,!U=|v1111|!,...)` (`ref0_wn.pft`, `ref1_wn.pft`, `udcsee_wn.pft`).
- **Переход из каталога IBIS**: чекбокс `name="CHECKINDEX" value="<код>"`; форма SEARCH с `S21P03=K=` (поиск по ключевым словам внутри классификатора); кнопка «Отобранные» (`C21COM=5`, `PFTNAME=udcselect_wn`).
- **udc_wn.pft — общий формат для УДК и ББК**: ветвление `if &uf('+D') : 'RSUDC' then 'УДК' else 'ББК' fi` — один набор форматов обслуживает обе БД (различаются только подключённой БД в `WnLink`).

### 4.2 HELP (ГРНТИ) — иерархия по структуре кода

- **Узел** = рубрика ГРНТИ; код в поле 3, наименование в поле 2.
- **Вверх**: `&uf('+R',v3)` — вышестоящий код вычисляется усечением (unifor `+R`); если пусто → «Верхний уровень». Ссылка `<A HREF="IRBIS:1,,,rub_wn,<родитель>">`.
- **Вниз**: `&uf('J,G=',v3)` — проверка наличия потомков (есть ли термины с ключом `G=<код>`); `&uf('7,|G=',v3,'|,@rub0_wn')` — итерация всех потомков. Иконка: `textfolder.gif` если есть потомки (`&uf('J,G=',v3)<>''`), иначе `folder.gif`.
- Корень: `rub_wn,0` (`v3='0'` → «Верхний уровень»).
- **Навигатор-форматы лежат в общем каталоге `Datai\Deposit\`** (`rub_wn.pft`, `rub0_wn.pft`, `rub_top_wn.pft`, `rub_search_wn.pft`, `rub_select_wn.pft`, `rub_see_wn`/`rub_see1_wn`/`rub_see2_wn`/`rub_see3_wn`), а **не** в каталоге HELP — резолвятся как депонированные форматы. TODO(recon #CLS-03): путь к Deposit задаётся отдельной строкой PAR/конфигом ИРБИС — в HELP.par все пути ведут только в `.\datai\help\`; механизм подключения Deposit требует уточнения.

### 4.3 MESH — иерархия по коду рубрики `R=`

- **Узел** = дескриптор/синоним/справочник (поле 100); код рубрики в поле 3 (`R=`).
- **Вверх**: `&uf('+R',v3)` (усечение кода) → текст родителя `ref(l('R=',&uf('+R',v3)),v2)`.
- **Вниз**: `&uf('J,G=',v3)` (есть ли потомки), `&uf('7,!G=',v3,'!,@mesh0_wn')` (итерация). Иконка: `text.gif` (лист) / `textfolder.gif` / `folder.gif`.
- **Связи**: «смотри» — поле 88 (`S=`, переход `ref(l('T=',v88),mfn)`); «смотри также» — поле 4 (`meshseeso_wn`); подрубрики — `M=`.
- Глобальные переменные G1/G2/G3/G33/G4/G88/G30/G31 — переключатели режима показа (головной термин группы, список кодов на странице).
- Корень: `mesh_wn,R=0`.

### 4.4 TEZ — иерархия по `U=` (термин-родитель)

- **Узел** = термин (дескриптор v2=`1` / недескриптор v2=`0`).
- **Вниз** (нижестоящие): `l("U="v1)>0` — есть ли потомки; `&uf('7,!U=',v1,'!,@tez0_wn')` — итерация. Связь хранится текстом самого термина (поле 4 ребёнка = текст термина-родителя, индексируется `U=`).
- **Вверх**: поле 4 «Вышестоящий дескриптор» → ссылка `tez_wn?...&KEY=T=<родитель>`.
- **Синонимика**: недескриптор → дескриптор: «Смотри» поле 6 (`&uf('7,!T=',v6,...)`); синонимы дескриптора — `l("C="v1)` (`&uf('7,!C=',v1,...)`); «Смотри также» (RT) — поле 5.
- Иконка: `text.gif` если v2=`0` (синоним); `textfolder.gif`/`folder.gif` по наличию `U=`.

### 4.5 URUB — иерархия по `U=` + двунаправленная генерация в `AUTOIN.GBL`

- **Узел** = термин; идентификатор `I=` (поле 2 или термин v1), родитель — поле 4 (`U=`).
- **Вниз**: `val(&uf('J,U=',g5))>0` (есть ли потомки), `&uf('7,!U=',g5,'!,@urub0_wn_')`.
- **Вверх**: поле 4 → ссылка `urub_wn?...&KEY=I=<родитель>`.
- **Синонимика недескриптора**: `C=` (синоним, поле 6), `CK=` (комбинация, поле 7), `CA=` (альтернатива, поле 8) — переходы `urub_wn?...&KEY=I=<v6/v7/v8>`.
- **Уровень иерархии** считается рекурсивно форматом `level0.pft`: `&uf('D,!I=<родитель>!,@level0')` поднимается по цепочке `I=` до корня, увеличивая глобальную G1 (записывается в поле 14).
- **`AUTOIN.GBL` (автоввод) строит дерево двунаправленно**:
  - поле 100 «Нижестоящие» (введённые потомки) → для каждого: если запись-потомок (`I=<код>`) существует — `CORREC` добавляет текущий термин в её поле 4 (вышестоящий); если нет — `NEWMFN` создаёт новую запись потомка с полем 1 и полем 4; затем поле 100 **удаляется** из текущей записи.
  - аналогично для поля 5 (ассоциативные/RT): синхронизирует обратную ассоциацию.
  - служебное поле 14 (уровень) пересчитывается через `level0`.
- Корень навигатора: `urub0_wn` с параметрами `S21CNR=100`, `FREEPAR0..5` (переключатели режима).

**Расшифровка `irbis:`-ссылки**: `irbis:<тип>,,,<стартовый-PFT>,<стартовый-ключ>?<параметры>`.
- тип `1` = открыть запись/дерево по ключу; тип `2` = открыть с верхнего уровня (`U=` без значения = корень); `R=0`/`0` = корень рубрикатора.
- `DBNAME_DEFAULT` — какую БД-навигатор открыть.
- `CHECKNAME=CHECKINDEX` + `CHECKPFT=<search-формат>` — механизм возврата отмеченного кода в поле каталогизатора: отмеченный чекбокс `CHECKINDEX` форматируется через search-PFT (`udcsearch_wn`→`U=…`, `rub_search_wn`→`R=…`, `tez_search_wn`→`K=…`, `urub_search_ft`) и вставляется как поисковое выражение/значение поля.

### 7.2 Префиксы поиска в IBIS, ведущие в навигаторы

В каталогизаторе поля документа IBIS связаны с навигаторами через доп.ввод (`Int…`/доп.ввод поля) и через WnLink. Соответствие «поле IBIS → навигатор → префикс поиска»:

| Навигатор | Префикс возврата в IBIS | Поле IBIS (типичное) |
|-----------|--------------------------|----------------------|
| RSUDC (УДК) | `U=` (через `udcsearch_wn`) | 675 (УДК) |
| RSBBK (ББК) | `U=` (через `udcsearch_wn`, БД RSBBK) | 686 (ББК) |
| HELP (ГРНТИ) | `R=` (через `rub_search_wn`) | 964/606-рубрики ГРНТИ |
| MESH | `T=`/код через `mesh_search_wn` | 60x предметные (мед.) |
| TEZ | `K=` (через `tez_search_wn`) | ключевые слова (610/653) |
| URUB | через `urub_search_ft` | тематические рубрики |

> В `irbisc.ini` блок `Int` (внутренние словари каталогизатора) задаёт связь полей с АФ: `IntPref0=K=` Тезаурус; `IntPref1=M=` АФ «Коллективные авторы»; `IntPref2=S=` АФ «Предметные заголовки»; `IntPref3=A=` АФ «Индивидуальные авторы»; **`IntPref4=U=` «Алфавитно-предметный указатель к УДК»** (`IntAddata4=athru,M=,4,@athrugrr`) — связь УДК-индексации с АПУ. Параметр `RSUDCINPUT=1` (`irbisc.ini`) включает ввод индекса УДК из навигатора RSUDC.
> Перекрёстные ссылки внутри классификаторов реализованы прямыми `IRBIS:1,,,<формат>,U=<код>`/`R=<код>` гиперссылками в строчных и узловых PFT (см. разд. 4–5): «смотри» (`S=`), «смотри также» (`SA=`), отсылки (`O=`), вышестоящий/нижестоящий (`G=`/`U=`).
