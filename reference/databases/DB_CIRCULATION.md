# Справочник подсистемы КНИГОВЫДАЧИ ИРБИС64 (reconstruction-grade)

> Фаза 2. Источники — реальные конфиги CP1251 (`C:\IRBIS64\Datai\<DB>\`), конвертированные в UTF-8 в `_work\cfg_txt\<DB>\`. Клиентские карты тегов — `irbisr.ini` (АРМ Читатель/web) и `irbisb.ini` (АРМ Книговыдача).
> ⚠ БД **RDR/RDRV** содержат ПДн читателей — ниже только СТРУКТУРА полей/подполей; реальные значения не извлекались.

Подсистема — 4 БД:

| БД | Назначение | PAR |
|---|---|---|
| **RDR** | Картотека читателей (постоянная), хранит выдачи на руках (поле 40) | `Datai\RDR.PAR` |
| **RQST** | Заказы/брони/выполнение/отказы — транзитная БД (очищается) | `Datai\RQST.PAR` |
| **RDR_ARH** | Архив завершённых (возвращённых) выдач | `Datai\RDR_ARH.par` |
| **RDRV** | Временные/«внешние» читатели — клон RDR + поле 31 (регистратор) | `Datai\RDRV.par` |

## 0. Сводная модель связей
```
                 IBIS (ЭК)  ── поле 910 экземпляры, статус 910^A (ste.mnu)
                    ▲  shifr (903 ↔ I=)   ekz (910^B инв / 910^H ш-к)
                    │
   RQST (заказ) ────┘
   30=читатель(RI=)  903=шифр  910=выданный экз  40=дата заказа
   41=дата выполн.   42=дата возврата(планов.)   44=отказ
        │ выдача (RQSTRDR.pft строит v40)
        ▼
   RDR (читатель)  поле 40 = «книга на руках»
   40^A шифр 40^B инв 40^H ш-к 40^D выдача 40^E план.возврат
   40^F факт.возврат 40^L продление 40^U утеря
        │ возврат → autoin/AUTOARH переносит запись
        ▼
   RDR_ARH (архив выдач)  поле 40 (та же структура)
```
Ключевые карты тегов клиент берёт из `irbisb.ini`/`irbisr.ini`, секции `[Reader]`/`[Request]` — см. §5.

# 5. МОДЕЛЬ ЦИРКУЛЯЦИИ (из конфигов)

Префиксы: `RDRPREFREADER=RI=`, `DOLGNIK=RB=`, `SHIFR=C=`, `BRIEF=N=`, `BAR=H=`. Лимиты: `MaxBooks=20`, `MaxDolgBooks=20`, `MaxReturnDays=20`. Флаги: `RETURNABLE=1`, `PROLONGABLE=1`, `STRONGPROLONG=1`, `RETURNTORESERVABLE=1`, `PEREREGABLE=1`. (В `irbisr.ini` web: `MaxBooks=5`, `MaxDolgBooks=1`.)

**Экземпляр в ЭК (IBIS, 910)**: `DBNTAGEKZ=910`, `DBNTAGSHIFR=903`, делимитеры `STATUS=A`,`INV=B`,`BAR=H`,`STORE=D`. Свободный экз: `DBNFREEEKZ=0`, безынв.учёт `DBNWINVEKZ=U`.

### Жизненный цикл
1. **ЗАКАЗ.** Запись RQST: 30=читатель, 903=шифр, 1=БД, 40=дата(ISIS), 201/31=brief. `autoin.gbl` подтягивает свободные экз. (`@free_ekz_*`), ставит 999. Печать требования — `order`/`ORDER.PFT` (RTF) или `cntrlW_html` (талон). Отметка печати → 911 (`P0=ДА`).
2. **ВЫДАЧА.** Оператор выбирает свободный экз (статус 910^A в ЭК), пишет RQST.910 (`^B`инв,`^H`ш-к,`^D`МХР), ставит 41=дата. Формат **`RQSTRDR.PFT`** строит RDR.40: `^A`=903, `^B`=910^B, `^H`=910^H, `^K`=910^D, `^D`=41, `^E`=42, `^F`=`******`(маркер «не возвращено»), `^G`=1, `^C`=201. Экз в ЭК → 910^A=`1` (выдан).
3. **БРОНИРОВАНИЕ.** 43=дата брони; статус 910^A по `reservstatus.mnu`: `0`отправлен МХР→МВ, `1`получен на МВ/для выдачи, `2`получен от читателя, `3`для возврата на МХР, `4`отправлен МВ→МХР. `RETURNTORESERVABLE=1`. Уведомления — `reservmail.pft`.
4. **ВОЗВРАТ.** В RDR.40^F пишется дата факт.возврата (вместо `******`). FST `DOLG=ДА/НЕТ` по `40^F:'******'`. F8 — `!f8ret`. Запись о выдаче переносится в **RDR_ARH** (`autoin_light.gbl`, `AutoinFile`). Заказ в RQST с 41 удаляется `rqst_clear.gbl`.
5. **ПРОДЛЕНИЕ.** RDR.40^L=дата, обновляется 40^E. Лимит `MaxReturnDays`/`return.mnu`.
6. **ОТКАЗ.** RQST.44: `^A`причина(`otkaz.mnu`: 01 Занято, 02 Нет на месте, 04 В переплёте, 05 Списано, 06 Лакуна, 07 Дефектный, 08 Ветхий, 09 Перенаправление, * Прочие), `^B`дата. FST `I=3`, поиск `RO=`. Удаляется `rqst_clear.gbl`.
7. **УТЕРЯ.** RDR.40^U=`'1'`; поиск `HU=`. Долговые формы — `RdDolgBK.*` (с ценой по 910^E из ЭК).

### Статусы экземпляра (ЭК IBIS, 910^A — `ste.mnu`)
`0` отдельный экз (свободен) · `R` группа экз (с инв) · `U` группа безынв.учёта (ВУЗ) · `C` группа для б-ки сети · `E` локальный сетевой ресурс · `7` передан из БД каталога (треб. докомплектование) · `2` ожидается · `3` в переплёте · `4` утерян · `5` временно не выдаётся · `6` списан · `1` **выдан читателю**. Свободные для выдачи = `0`/`U`/`C`; занятые = `1`/`3`/`4`/`5`/`6`/`2`.

## 8. Связи между БД
| Связь | Через поля |
|---|---|
| **RQST → RDR** | RQST.30 = RDR.30 (`RI=`) |
| **RQST → IBIS** | RQST.903 = IBIS.`I=` (`DBNPREFSHIFR=I=`); RQST.1=имя БД; RQST.910 ↔ IBIS.910 (`^B/^H/^D/^A`) |
| **RQST → RDR (выдача)** | `RQSTRDR.PFT`: 903→40^A, 910^B→40^B, 41→40^D, 42→40^E |
| **RDR → IBIS** | RDR.40^A = IBIS.`I=`; FST RDR через `&uf('D'…)` (freekz, аналоги) |
| **RDR ↔ RDR_ARH** | по 30 (`RI=`); возврат переносит 40 в архив (`autoin*.gbl`) |
| **RDRV ↔ RDR** | независимая БД-клон |
