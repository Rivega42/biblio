## Фаза 2 — Reconstruction-grade (в работе)
Цель: глубина, достаточная для полной пересборки приложений. Структура:
- `reference/databases/DB_<NAME>.md` — полный справочник каждой БД: PAR, **каждая строка FST** (поле/метод/префикс), **все рабочие листы** `.ws` (страницы+поля) и `.wss` (подполя), словарь полей/подполей, каталог форматов PFT и ветвление по 920, справочники MNU/TRE, связи между БД.
- `reference/arms/ARM_<NAME>.md` — полный разбор INI каждого АРМа (секция за секцией) + экраны/режимы/потоки.
- `reference/format/PFT_LANGUAGE.md` — язык форматирования (команды, UNIFOR, условные, повторяющиеся группы).
- `reference/format/FIELD_DICTIONARY.md` — сводный словарь полей/подполей формата (ЭК + комплектование).

### Журнал Фазы 2
- _(старт)_ Каркас `reference/`, `CHANGELOG.md`, помощник `conv_db.ps1` (конвертация конфигов БД в UTF-8).
- **Волна 1 (готово):** 7 reconstruction-grade справочников из реальных конфигов:
  - `format/PFT_LANGUAGE.md` — язык форматирования (44 конструкции, 41 UNIFOR-код, 16 RTF-команд, 20 кодов ошибок).
  - `format/FIELD_DICTIONARY.md` — сводный словарь полей (74 ЭК + 41 комплектование = 115 полей, по 18 блокам).
  - `databases/DB_IBIS.md` — ядро каталога (~95 полей, 90 префиксов, 13 типов РЛ по полю 920, 60+ форматов, импорт/экспорт MARC).
  - `databases/DB_CIRCULATION.md` — RDR/RQST/RDR_ARH/RDRV: **полная модель циркуляции из конфигов** (заказ→выдача→возврат→продление; `RQSTRDR.pft`; статусы `ste.mnu`/`reservstatus.mnu`) — закрыт пробел Фазы 1.
  - `databases/DB_ACQUISITION.md` — CMPL/PODB/POST: модель заявка→заказ→SZ→поступление→КСУ→списание→перенос в ЭК (~114 префиксов, RTF-бланки КСУ).
  - `databases/DB_VUZ.md` — книгообеспеченность (10 типов записи, «связка», расчёт ККО, поле 693).
  - `databases/DB_AUTHORITY.md` — ATHRA/C/S/G/U/B: 4-блочная модель авторитетного контроля, подполе `^3`, autoin-генерация связанных записей.
- **Волна 2 (в основном готово):**
  - `databases/DB_MBA.md` — МБА: дискриминатор 920 (AB/ODER/CENA/ARHIV), цикл оформление→выдача(O1)→продление(O2)→возврат(S1/S2)→списание(H1)→архив, перенаправление M4 (sigl.mnu), статусы из `irbism.ini`.
