# CAPABILITY_MAP — сводная карта возможностей САБ ИРБИС64(+)

> Мастер-карта: как слои продукта собираются вместе. Каждый блок ссылается на детальные FINDINGS и/или исходные файлы. Построено независимо из файлов в `C:\IRBIS64\` и `C:\IRBIS64_COPY 17-09-2025\`. Реестр файлов и статусы — [DOCUMENT_REGISTER.md](DOCUMENT_REGISTER.md); незакрытое — [OPEN_QUESTIONS.md](OPEN_QUESTIONS.md).

## 1. Что это и состав поставки
**САБ ИРБИС64+** — клиент-серверная система автоматизации библиотек (разработка ГПНТБ России / Ассоциация ЭБНИТ). Состав ([FINDINGS_10](FINDINGS_10_overview_system.md), [FINDINGS_04](FINDINGS_04_server_admin.md)):
- **Сервер**: `irbis_server.exe` (+ служба `service_64.exe`), процессы обработки `server_64.exe`, ядро доступа/форматирования `irbis64.dll`, нативный TCP-протокол (порт 6666).
- **Настольные АРМы** (клиент `irbis64_client.dll`): Читатель, Каталогизатор, Книговыдача, Комплектатор, Книгообеспеченность, Администратор.
- **Web-подсистема**: Web-ИРБИС64+ (CGI-шлюз `irbis64r_plus`), Irbis_Navigator; J-ИРБИС (упоминается).
- **БД** (49 папок в `Datai\`) с параметрическими конфигами (PAR/FST/PFT/WSS/MNU/OPT/…).
- **Утилиты**: глобальная корректировка (`IrbisGblAdmin`), массовая загрузка (`IrbisMultiLoad`), извлечение ПТ (`IrbisExtractFullText`, `pdftotext`), генераторы (PFT/штрих-/QR-коды), Z39.50 (`yaz.dll`), RFID (`IRBIS_RFID.exe`).

## 2. Архитектура и протокол ([FINDINGS_03](FINDINGS_03_protocol.md), [FINDINGS_04](FINDINGS_04_server_admin.md))
- **Модель**: ядро принимает TCP-запросы и порождает независимые процессы `server_64.exe`; обмен ядро↔процессы — через системную память или временные файлы в `workdir`. Режимы: последовательный, многопотоковый (`THREADS_AVAILABLE`), многопроцессорный (`DUPLICATE_SOCKETS`). Долгоживущие/однократные процессы (`KEEP_PROCESS_ALIVE`, `MAX_SERVERS`, `MAX_PROCESS_COUNT=20`, ошибка `SERVER_OVERLOAD`).
- **Сессия**: `IC_reg(host,port,arm,user,password)` → INI-профиль; периодические `IC_nooperation`; авто-разрегистрация по `CLIENT_TIME_LIVE`. Клиент строго последователен (один запрос за раз).
- **Клиентский API** (`irbis64_client.dll`): **80 функций** в 11 группах — сессия, ресурсы/INI, мастер-файл (read/update/групповая/связанная), работа с записью (локально), словарь, поиск (прямой/последовательный), форматирование, пакетная обработка (print/stat/gbl), администрирование, вспомогательные. Полный каталог — FINDINGS_03.
- **Кодировки**: основной обмен — UTF-8; ресурсы/INI/админ-списки — ANSI(CP1251). Псевдоразделитель `$3130` ↔ `$0D0A`.
- **Блокировки/версии**: запись имеет статус и номер версии; конфликт записи → `VERSION_ERROR`; монопольная блокировка БД (`ERR_DBEWLOCK`).

## 3. Сборка слоёв: путь от поиска до показа
```
[Пользователь АРМа]
   ├─ выбирает ВИД ПОИСКА  ── irbis*.ini [SEARCH] ItemNameN→ItemPrefN ──┐
   │                                                                    ▼
   ├─ словарь термов  ◄── инвертирование .FST (метка,метод,формат→'ПРЕФИКС=') ◄── .MST (записи)
   │        ▲ префиксы: K= A= T= M= S= U= R= V= B= I= … (109 в IBIS)
   ├─ язык запроса: "<преф><терм>$"/(метки)  операторы + * ^ (G)(F).
   ▼
[Результат: список MFN] ── BRIEF.PFT (краткое) ──► [Полный показ]
   │                                                     ▲
   │  формат выбирается .OPT по полю 920 (тип записи) ───┤
   │  меню форматов PFTW.MNU (экранные + _H HTML) ───────┘
   ▼
[Ввод/корректура] ── рабочий лист .WS (страницы) → подполя .WSS → ФЛК @-форматы → справочники .MNU/.TRE
```
(Источники: `irbisr.ini`, `Datai\IBIS\ibis.fst`, `WS.OPT`, `PFTW.OPT`, `PFTW.MNU`, `*.WS/.WSS` — детали в [FINDINGS_02](FINDINGS_02_config_and_db_profiles.md).)

## 4. Модель данных ([FINDINGS_10](FINDINGS_10_overview_system.md), [FINDINGS_12](FINDINGS_12_record_structure.md), [FINDINGS_13](FINDINGS_13_irbisprl.md))
- **Запись** = набор полей; поле = числовая **метка** (tag) + значение, делящееся на **подполя** разделителем `^x`; поля **повторяемы**. Идентификатор записи — **MFN**.
- **Поле 920** — селектор типа/уровня записи (= имя рабочего листа): `PAZK`/`PVK` (книги), `SPEC` (сводное многотомное), `J`/`NJ`/`NJP`/`NJK` (журнал/номер/подшивка), `ASP`/`AUNTD` (аналитика-статьи), `MUSP` (музейный предмет) и др. Управляет выбором РЛ (`WS.OPT`) и формата (`PFTW.OPT`).
- **Статусы записи**: логически удалена, физически удалена, заблокирована, неактуализирована (+ номер версии). Файлы БД: `MST` (записи переменной длины: лидер+справочник+поля), `XRF` (MFN→позиция+флаги), `L01/N01` (дерево словаря), `IFP` (постинги), `CNT` (счётчики). Изменение = дозапись в конец с сохранением обратной ссылки (откат).
- **Формат элементов данных** ЭК и комплектования (метки/подполя/обязательность/UNIMARC «(mf)») — формализован в [FINDINGS_13](FINDINGS_13_irbisprl.md) (две таблицы: ЭК и комплектование).
- **Связи записей**: иерархия журнал↔номер↔статья и многотомники (поля 461/46/463/963/481, поиск по связи); каталог↔комплектование (поле 66, 901/938); каталог↔книговыдача (910, 999); авторитетный контроль (подполе «3» → ATHR*); сводный каталог (902).

## 5. Поиск ([FINDINGS_02 §3–4](FINDINGS_02_config_and_db_profiles.md), [FINDINGS_03](FINDINGS_03_protocol.md))
- **Виды поиска** задаются в `irbis*.ini [SEARCH]` (подпись→префикс→тип словаря→справочник→авторитет). Пример Читателя — 23 вида (K=,A=,T=,M=,S=,R=,V=,HD=,U=,B=,G=,J=,C=,DP=,DR=,VUZ=,P=,N=,X= …).
- **Префиксы** физически создаются инвертированием `.FST` (литерал перед `=`). IBIS — 109 префиксов; полный профиль по всем 49 БД — [FINDINGS_02 §8](FINDINGS_02_config_and_db_profiles.md).
- **Язык запроса** (прямой): `"<преф><терм>$"` ($=правое усечение), квалификация `/(метки)`, операторы `+`(ИЛИ) `*`(И) `^`(НЕ) `(G)`(в поле) `(F)`(в повторении) `.`(подряд) с приоритетами. **Последовательный** поиск — формат-условие `if … then '1' else '0' fi`.
- **Доп. механизмы**: морфология (БД MORPH, `MORPHOLOGY=1`), фасеты (`.cells`, [FINDINGS_14](FINDINGS_14_db_descriptions_misc.md)), поиск по связи (`Scnt`), навигаторы (УДК/ББК/ГРНТИ/тезаурус/MeSH/URUB), авторитетные файлы (ATHRA/C/S/G/U/B), квалификатор по группам полей.

## 6. Форматирование/показ ([FINDINGS_02 §5](FINDINGS_02_config_and_db_profiles.md), [FINDINGS_10](FINDINGS_10_overview_system.md))
- **Язык PFT** (на базе CDS/ISIS): `v<tag>^<подполе>`, режимы `MP/MH/MD`, условные `if…then…else…fi` (ветвление по 920), повторяющиеся группы `(…)`, функции `&unifor`/`&uf`, `val`, `rsum`, `ref`, `s`; вывод RTF/HTML.
- **Выбор формата**: `.OPT` по полю 920 (`PFTW.OPT` экран, `PFTW_H.OPT` web); меню `PFTW.MNU` (десятки форматов: БО книг/журнала/аналитики, BRIEF, UNIMARC/USMARC/RUSMARC, Dublin Core, комплекты каталожных карточек `!KKK*`, формуляры, штрих-коды).
- **Экранные vs веб**: HTML-форматы — суффикс `_H`; web-PFT содержат команды шлюза (`<!FORMAT=>`, `<!FILE=>`, подстановки) — [FINDINGS_09](FINDINGS_09_web_reader.md).

## 7. Ввод данных ([FINDINGS_02 §6](FINDINGS_02_config_and_db_profiles.md), [FINDINGS_05](FINDINGS_05_arm_cataloger.md))
- **Рабочий лист `.WS`**: многостраничная форма (страницы «Основное БО», «Экземпляры», «Систематизация», «КО»…); на поле — метка/подпись/тип-редактора/связанный `.WSS`/ФЛК-формат `@…`/подсказка.
- **Подполя `.WSS`**: код подполя/подпись/тип-редактора/средство ввода (`,A=,@!amovu` словарь+контроль; `gor.mnu\\1` меню-колонка; `df.mnu`)/подсказка.
- **Типы рабочих листов IBIS**: PAZK/PVK/SPEC/!RPJ(J)/!NJ/ASP/AUNTD/MUSP/IBIS (+варианты 31/42/71); вложенные РЛ (Bouni/Index/Content/KO/RD/Techn…). ФЛК (форматы логического контроля) валидируют поля.

## 8. Реестр баз данных и роли ([FINDINGS_02 §8](FINDINGS_02_config_and_db_profiles.md))
Реестры: каталогизатор — `DBNAM1.MNU` (44 БД), читатель — `DBNAM3.MNU` (IBIS+IMAGE), МБА — `DBNAMMBA`, заказ — `DBNAMZAK`. Ключевые БД:

| Группа | БД | Назначение |
|---|---|---|
| Каталог | **IBIS** (ядро), **IMAGE** (имидж), **VKR** (ВКР), **SK**+Sk_* (сводный) | библиографические записи |
| Циркуляция | **RDR**/**RDRV** (читатели), **RQST** (заказы), **RDR_ARH** (архив) | книговыдача |
| Комплектование | **CMPL**, **PODB** (заказ книг), **POST** (подписка) | заказ/поступление/КСУ |
| Книгообеспеч. | **VUZ** (дисциплины/контингент) | расчёт ККО |
| МБА | **MBA**, **MBA_ARH** | межбибл. абонемент |
| Авторитетные | **ATHRA/ATHRC/ATHRS/ATHRG/ATHRU/ATHRB** | нормативный контроль |
| Классификация | **RSUDC** (УДК), **RSBBK** (ББК), **HELP** (ГРНТИ), **MESH**, **TEZ**, **URUB** | рубрикаторы/тезаурусы |
| Полные тексты | **RIGHT** (права), **LICH** (личные данные), **ZAPR** (запросы) | ЭБ |
| Спец. | **GUAR/GURF/GuSk** (гум.архив), **EVENT** (мероприятия), **KZD**, **PAY**, **IRI** | предметные задачи |
| Служебные | **LOGB/LOGC/LOGP/LOGDB** (протоколы), **COUNT**, **MORPH**, **WORK**, **BLANK**, **SPRAV** | инфраструктура |

## 9. АРМы (краткие карты → детальные FINDINGS)
- **Читатель / Web** ([FINDINGS_09](FINDINGS_09_web_reader.md)): поиск ЭК (словарь/полнотекст/связь/навигаторы), заказ, личный кабинет (литература на руках, история, закладки/LICH, корзина/RQST, запросы/ZAPR, загрузка ВКR), доступ к ПТ по правам. Десктоп-АРМ — конфиг `irbisr.ini` (#0901).
- **Каталогизатор** ([FINDINGS_05](FINDINGS_05_arm_cataloger.md)): ввод (НОВЫЙ/по MFN), типы РЛ, поиск, корректура (по MFN/поиску/словарю), глобальная корректировка, импорт/экспорт (ISO2709 RUSMARC/UNIMARC/USMARC, ЛИБНЕТ, Z39.50, Web-ИРБИС), печать (списки/КК/таблицы), дублетность, авторитетные БД.
- **Комплектатор** ([FINDINGS_06](FINDINGS_06_arm_acquisition.md)): ЗАКАЗ→ПОСТУПЛЕНИЕ→ВЫБЫТИЕ→ПОДПИСКА; БД CMPL/PODB/POST; записи КСУ; мастера поступления/списания/заказов; «Комплектатор в Каталогизаторе».
- **Книгообеспеченность** ([FINDINGS_07](FINDINGS_07_arm_bookprovision.md)): БД VUZ (дисциплины/контингент-«связки»), связь IBIS↔RDR, расчёт коэффициента книгообеспеченности (ККО, поле 693), учебный план, отчёты/лицензирование.
- **Книговыдача / МБА** ([FINDINGS_08](FINDINGS_08_arm_lending_mba.md)): циркуляция (заказ/выдача/возврат/продление; статусы экземпляра 0/6/C/U; штрих-код/RFID), списание (КСУ выбытия); МБА (MBA/MBA_ARH, заказы между библиотеками). ⚠ подлинное руководство «Книговыдача» отсутствует (#0801) — модель выводима из конфигов RDR/RQST.
- **Администратор** ([FINDINGS_04](FINDINGS_04_server_admin.md), [FINDINGS_03 гр.10](FINDINGS_03_protocol.md)): управление БД (создание/опустошение/удаление, реорганизация мастер-файла и словаря, разблокировка), список клиентов/прав, мониторинг процессов, рестарт сервера, полнотекстовая подсистема.

## 10. Права и гейтинг по АРМам ([FINDINGS_04](FINDINGS_04_server_admin.md))
Список клиентов `Datai\CLIENT_M.MNU` — тройки `логин / пароль / строка-прав`, где строка = `C=<ini>; B=<ini>; P=<ini>; K=<ini>; A=<ini>; M=<ini>;`: `C`=Каталогизатор, `B`=Книговыдача, `P`=Комплектатор, `K`=Книгообеспеченность, `A`=Администратор, `M`=Комплектатор(прот.). Пустое/отсутствующее значение = АРМ запрещён. Код АРМа при регистрации (`IC_reg`): `IRBIS_READER/CATALOG/COMPLECT/BOOKLAND/ADMINISTRATOR/BOOKPROVD`. Опц. шифрование профилей `ENCRYPT_PASSWORDS`. (Значения логинов/паролей в карту НЕ переносились.)

## 11. Полные тексты, обмен, интеграции
- **Полные тексты**: путь — 11-я строка `<DB>.PAR` (IBIS: `Texts\`); поля 951 (ПТ/URL), 950 (имидж), 955; извлечение/кэш PDF — `irbis_server.ini [FullText]` (`ExtractPDFPageMode`, кэш страниц), `pdftotext`/QuickPDF; права — БД RIGHT/LICH, web-параметр `Show_ed`.
- **Форматы обмена**: текстовый ИРБИС (`#МММ`/`*****`), ISO 2709, MARC — RUSMARC/UNIMARC/USMARC/MARC21 (двусторонняя конверсия: `rmarcw/umarcw/usmrcw.pft` + `.fst`). 
- **Протоколы/периферия**: Z39.50 (клиентский импорт, `yaz.dll`; серверный `Z39_50_SERVER=0` выкл.), OAI/ЛИБНЕТ (импорт), штрих-коды/QR (генераторы), RFID (`IRBIS_RFID.exe`).

## 12. Ориентир для веб-реализации (выводы для будущей разработки)
- **Сервер**: воспроизвести нативный TCP-протокол порт 6666 (⚠ проводной формат не задокументирован — #R02; ориентир — открытые клиенты) ИЛИ обёртку над `irbis64_client.dll`. Семантика — 80 функций FINDINGS_03 (это контракт API).
- **Данные**: парсер MST/XRF/L01/N01/IFP (структура — FINDINGS_12) либо доступ через протокол. Поле 920 — ключ маршрутизации формы/формата.
- **Поиск**: реализовать язык запросов (префиксы+усечение+операторы); словарные префиксы и FST-инвертирование — из `.FST` каждой БД.
- **Показ/ввод**: интерпретатор PFT (показ) и движок рабочих листов `.WS/.WSS` (ввод) с ФЛК; меню/справочники `.MNU/.TRE`.
- **Права/АРМы**: модель `CLIENT_M.MNU` + коды АРМов + per-DB видимость (`DBNAM*.MNU`).
